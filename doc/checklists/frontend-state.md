# 前端状态联动检查清单

每次修改前端组件的状态逻辑时，逐项检查。

## watch 联动

- [ ] 切换筛选条件（selectedCode / selectedMetric / activeTab）时，相关数据是否都重新加载？
- [ ] 列表数据变化后，依赖它的 computed 是否会自动更新？
- [ ] 弹窗关闭后，相关状态是否重置？（避免下次打开残留旧数据）

## 数据加载

- [ ] 初始加载和手动刷新是否走同一个函数？（避免逻辑重复但行为不一致）
- [ ] 加载失败时，旧数据是否清除？（避免显示过期数据）
- [ ] 快速切换时是否有竞态保护？（后发请求先返回时不应覆盖新数据）

## Tab / 页面切换

- [ ] 切换 Tab 时，该 Tab 的数据是否自动加载？
- [ ] 切换 Tab 后再切回来，数据是否还在？（应该保留缓存）
- [ ] 外层 Tab 切换时，内层子组件状态是否需要重置？

## 常见 bug 模式

| 场景 | 检查点 |
|------|--------|
| 下拉选择器 change | watch 里是否加载了所有关联数据？ |
| 路由 / Tab 切换 | 新页面的数据是否在 onMounted 或 watch 里加载？ |
| 删除操作 | 删除后列表是否刷新？选中项是否清除？ |
| 表单提交成功 | 关闭弹窗 + 刷新列表 + 重置表单状态？ |
