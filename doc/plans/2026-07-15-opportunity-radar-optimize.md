# 机会雷达 — 关注机会 & 事件雷达 优化设计稿

> 日期：2026-07-15
> 状态：已确认
> 目标：提升"关注机会"Tab 的信息清晰度和决策效率，增强"事件雷达"与"关注机会"的联动

## 一、现状诊断

### 1.1 "关注机会" Tab 当前页面结构

```
┌─ 组合智能面板 ───────────────────────┐  ← 首屏 30%
│  风险度量(8指标) + 7维体检 + 大师矩阵 + 持仓明细 │
├─ 大师决策回测面板 ────────────────────┤  ← 首屏 30%
│  总体统计 + 大师胜率排名 + 历史决策列表        │
├─ 统计卡片（6类）──────────────────────┤
├─ 排序工具栏 ─────────────────────────┤
├─ 关注基金卡片（逐个滚动）─────────────────┤  ← 仅占首屏 40%
└────────────────────────────────────┘
```

### 1.2 核心问题

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 主题偏离 | 高 | 组合智能+大师回测回答的是"组合健康吗"，但用户打开此Tab想回答的是"哪只该买" |
| 决策信息不前置 | 高 | 上车信号灯+建议价埋在卡片中下部，需逐个滚动，无"一眼可扫"的排序 |
| 卡片信息过载 | 中 | 每张卡片展开体检报告+大师矩阵，5只基金=信息爆炸，看不清重点 |
| 统计卡无动态 | 中 | 只有计数，无"较昨日变化"、"估值分位趋势"等对比信息 |
| 事件雷达联动弱 | 中 | 事件雷达发现的"关注机会"事件和关注列表中基金割裂，需来回切换 |
| 缺少行动建议 | 高 | 用户打开页面最核心需求是"我该做什么"，当前需逐个翻看卡片才能决策 |

### 1.3 已有能力（无需改动）

| 能力 | 状态 |
|------|------|
| 巡检自动刷新净值+估值分位 | ✅ patrol API |
| 三级估值查询兜底（本地DB → 天天基金 → akshare） | ✅ |
| 自动推算建议上车价 | ✅ |
| 信号灯状态（green/yellow/red/gray） | ✅ |
| 买入评分（buy-score） | ✅ |
| 基金六维体检报告 | ✅ |
| 大师理念矩阵（单基金版） | ✅ |
| 事件雷达扫描+关联 | ✅ |

## 二、设计方案

### 2.1 核心思路

**"关注机会" Tab 重新定位为决策中心而非信息中心**：用户打开此页面，第一眼看到的是"今日操作建议"和按优先级排序的基金卡片，组合智能/大师回测等"组合层面"内容下移到独立区块或可折叠。

### 2.2 页面结构改造

```
┌──────────────────────────────────────┐
│  💡 今日操作建议（新增）               │  ← 一句话决策 + 信号分布
│  "2只可上车，优先关注中证医疗ETF"       │
├──────────────────────────────────────┤
│  🥇 可上车基金（绿色边框+高亮）         │
│  · 估值分位 · 距上车价 · 建议买入金额    │  ← 3列关键指标
│  · 上车理由（1句话）                   │
│  · 📋 展开详情（体检+大师+走势）        │  ← 默认折叠
├──────────────────────────────────────┤
│  🥡 接近上车（橙色边框）               │
│  · 同样3列关键指标                    │
├──────────────────────────────────────┤
│  🥉 等待中（灰色，降低视觉权重）         │
├──────────────────────────────────────┤
│  ⚪ 数据不足（提示巡检）               │
├──────────────────────────────────────┤
│  📊 组合智能（可折叠面板）              │  ← 下移到底部
│  📊 大师决策回测（可折叠面板）          │
└──────────────────────────────────────┘
```

### 2.3 各模块详细设计

#### 2.3.1 今日操作建议（新增）

顶部横幅，绿色渐变背景，一眼可扫：

```html
💡 今日操作建议
2只基金可上车，建议优先关注「中证医疗ETF」（估值分位12%，距上车价仅+3.9%）
[🟢可上车 2] [🟡接近 1] [🔴等待 1] [⚪数据不足 1]
```

数据来源：直接复用 `watchlistStats` 计算属性 + 取 `sortedWatchlist` 第一条可上车基金。

#### 2.3.2 基金卡片重构

**关键改动**：
1. 卡片按信号优先级排序（可上车 > 接近 > 等待 > 数据不足），默认排序
2. 关键指标改为3列网格布局（`grid-template-columns: 1fr 1fr 1fr`）
3. 体检报告+大师矩阵默认折叠，点击展开
4. 可上车卡片绿色边框+阴影突出，等待中卡片降低透明度
5. 每张卡片加1句话"上车理由"（从 signal_reason 提取）

**卡片结构**：
```
┌──────────────────────────────────────┐
│ 🥇 基金名称          [立即上车] 标签   │
│ ┌─────────┬─────────┬─────────────┐ │
│ │ 估值分位 │ 距上车价 │ 建议买入     │ │
│ │ 12% 低估 │ +3.9%   │ ¥1,000~2,000│ │
│ └─────────┴─────────┴─────────────┘ │
│ 理由：估值历史低位+今日跌幅扩大，适合分批建仓│
│ 📋 展开详情：体检报告 · 大师矩阵 · 走势  │
│ ┌─ 展开后 ─────────────────────────┐ │
│ │ 净值 0.8520 · 目标价 0.8000       │ │
│ │ 跟踪指数：中证医疗                │ │
│ │ [六维体检80分] [大师矩阵]         │ │
│ └──────────────────────────────────┘ │
└──────────────────────────────────────┘
```

#### 2.3.3 统计卡片优化

- 6项统计卡片保持不变，但"可上车"卡片点击后自动滚动到第一个可上车基金
- 新增：统计卡片上方显示"较上次巡检"的时间戳

#### 2.3.4 组合智能面板下移

- 整个组合智能面板从顶部移到卡片列表下方
- 默认折叠，标题栏显示"📊 组合智能 · 72分 良好"
- 大师决策回测同样下移+折叠，标题栏显示"🎯 大师决策回测 · 胜率 61.1%"

#### 2.3.5 事件雷达 Tab 联动增强

事件雷达中"关注机会"类型的事件卡片，增加醒目提示：
- 关联的关注基金名称高亮显示
- 点击可跳转到"关注机会" Tab 并定位到对应基金卡片

## 四、风险

1. **卡片折叠后体检报告加载**：点击展开时再触发体检报告加载（此前是批量加载），首次展开可能稍慢。可接受（用户主动操作）。
2. **卡片排序逻辑**：默认按信号灯排序，用户仍可手动切换排序方式。排序功能保留。
3. **向后兼容**：纯前端改动，不影响任何数据或API。
