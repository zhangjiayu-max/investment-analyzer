## 后端设计

### 新增 `backend/routers/capabilities.py`

聚合 API，3 个端点：

```
GET /api/capabilities/overview
  → 总览（各来源能力数、已暴露/未暴露统计）

GET /api/capabilities/tools
  → 完整能力清单（含分类、参数、来源、暴露状态、启用状态）

GET /api/capabilities/mcp/unexposed
  → 仅未暴露的 MCP 能力（用于扩展面板）
```

### MCP 扩展机制设计

#### 未暴露能力的接入成本分级

| 成本等级 | 条件 | 接入工作量 | 数量 |
|---------|------|-----------|------|
| none | 已暴露 | 0 | 53 |
| low | 有 Python 方法 + 有参数 schema | 加 TOOLS 定义 + execute_tool case | 10 |
| medium | 有 Python 方法 + 无参数 schema | 补 schema + 加定义 + case | 0 |
| high | 无 Python 方法 | 新增 Client 方法 + 定义 + case | 6 |

#### low 成本（10 个，可快速接入）

**ttfund（5 个）：**
- fund_manager（基金经理信息）
- fund_condition（条件选基）
- fund_holding（重仓股查询）
- fund_nav（净值历史）
- fund_gold（黄金行情）

**eastmoney（5 个）：**
- screener（智能选股）
- finance_data（金融数据查询）
- macro_data（宏观数据查询）
- comparable（可比公司分析）
- industry（行业研究，注意耗时约 20 分钟需异步）

#### high 成本（6 个，需新增封装）

**ttfund（4 个）：** fund_tg_strategy / fund_portfolio / fund_favor / fund_backtest
**eastmoney（2 个）：** earnings（业绩点评）/ topic（主题研究）

### 扩展机制实现

在 `routers/capabilities.py` 中新增：
```
POST /api/capabilities/mcp/integrate
  → 一键接入指定 MCP 能力（仅 low 成本）
  → 自动在 TOOLS 数组追加定义 + execute_tool 追加 case
  → 返回接入结果
```

> 注：一键接入涉及代码生成，初版先做"接入指引"（展示需要添加的代码片段），用户手动粘贴。后续可做自动代码注入。
