# LLM 调用审计与降本设计稿

> 日期：2026-06-26  
> 背景：MIMO 套餐到期，切换到 DeepSeek 按量付费，需要精确控制每日 LLM 消耗  
> 原则：核心分析功能保留，自动触发类精简约 70%

## 一、LLM 调用全景审计

### 1.1 调用分类统计

| 类别 | 调用点 | 触发方式 | 日均消耗估算 | 必要性 |
|------|--------|---------|------------|--------|
| **用户主动对话** | orchestrator.py 多Agent编排 | 用户询问 | 50万 token | 🟢 必留 |
| **用户主动分析** | 持仓诊断/费率/相关性/滚动收益 | 用户点击 | 20万 token | 🟢 按需 |
| **自动市场日报** | `_auto_daily_report()` | 启动时 + 检查今日无日报 | 1.2万 token | 🔴 可停 |
| **自动日报评估** | `_auto_eval_report()` | 日报生成后 | 0.5万 token | 🔴 可停 |
| **每日评测Pipeline** | `_auto_daily_eval()` | 凌晨4:00 | 0.5万 token | 🔴 可停 |
| **自动Shadow晋升** | `auto_promote_shadows()` | 凌晨4:00 | 0 token | 🟢 无LLM |
| **自动Eval生成** | `auto_generate_eval_cases()` | 凌晨4:00 | 0 token | 🟢 无LLM |
| **对话自动评测** | `_schedule_auto_evaluation()` | 每次对话结束 | 0.3万 token/次 | 🔴 可停 |
| **页面LLM总结** | health_score/four_pots/fee/corr/rolling | 页面加载自动 | 0.5万 token/次 | 🟡 可缓存 |
| **交叉审阅** | orchestrator.py Phase B | 多专家分歧时 | 2万 token/次 | 🟢 质量必要 |
| **仲裁** | orchestrator.py Phase C | complex场景 | 0.5万 token/次 | 🟢 质量必要 |
| **文章解读** | article_reader.py | 粘贴链接时 | 0.3万 token/次 | 🟢 按需 |
| **RAG查询重写** | rag.py query rewrite | RAG检索时 | 0.1万 token/次 | 🟢 评测必要 |
| **每日持仓提示引擎** | daily_position_advisor.py | 净值刷新后 | 0 token (纯规则) | 🟢 无LLM |

### 1.2 日均消耗构成（145万 token 分析）

```
用户主动对话:      50万  (34%)  ← 必须保留
用户主动分析:      20万  (14%)  ← 按需触发，保留API
自动市场日报:      12万  (8%)   ← 可停
日报质量评估:       5万  (3%)   ← 可停
每日评测Pipeline:   5万  (3%)   ← 可停
对话自动评测:      15万  (10%)  ← 可停（刚实现的胶水）
页面LLM总结:       20万  (14%)  ← 改缓存/手动触发
交叉审阅+仲裁:     10万  (7%)   ← 必须保留
其他(RAG/文章等):   8万  (5%)   ← 保留
```

**结论：约 40% 的日消耗是非必要的自动触发，可以停掉。**

## 二、降本方案：三级开关

### 2.1 设计思路

不要删代码，用**配置开关**控制。需要时可以一键恢复。

```python
# db/config.py 新增
('llm_cost.auto_daily_report', 'false', '自动市场日报', 'llm_cost'),
('llm_cost.auto_daily_eval', 'false', '每日评测Pipeline', 'llm_cost'),
('llm_cost.auto_conversation_eval', 'false', '对话结束自动评测', 'llm_cost'),
('llm_cost.auto_shadow_mode', 'false', 'Shadow Mode 静默运行', 'llm_cost'),
('llm_cost.page_llm_summary', 'false', '页面加载自动LLM总结', 'llm_cost'),
('llm_cost.llm_judge_eval', 'false', 'LLM-as-Judge 评分', 'llm_cost'),
('llm_cost.root_cause_analyzer', 'false', '自动根因分析', 'llm_cost'),
```

### 2.2 具体改动

#### 开关 1：自动市场日报（8% 节省）

```python
# app.py startup() 中
if get_config("llm_cost.auto_daily_report", "false") == "true":
    asyncio.create_task(_auto_daily_report())
    asyncio.create_task(_auto_eval_report())
else:
    logging.info("自动市场日报已关闭（llm_cost.auto_daily_report=false）")
```

#### 开关 2：每日评测 Pipeline（3% 节省）

```python
# app.py startup() 中
if get_config("llm_cost.auto_daily_eval", "false") == "true":
    asyncio.create_task(_auto_daily_eval())
else:
    logging.info("每日评测 Pipeline 已关闭（llm_cost.auto_daily_eval=false）")
```

#### 开关 3：对话结束自动评测（10% 节省）

```python
# orchestrator.py _schedule_auto_evaluation() 中
def _schedule_auto_evaluation(conv_id, msg_id, result):
    if get_config("llm_cost.auto_conversation_eval", "false") != "true":
        return  # 关闭
    _eval_executor.submit(...)
```

#### 开关 4：Shadow Mode（按需节省）

```python
# shadow_mode.py run_shadow() 中
def run_shadow(config_id, input_data, production_output, agent_type=""):
    if get_config("llm_cost.auto_shadow_mode", "false") != "true":
        logger.debug(f"Shadow Mode 已关闭，跳过 {agent_type}")
        return None
    ...  # 正常执行
```

#### 开关 5：页面 LLM 总结 → 改缓存（14% 节省）

不是完全停掉，而是**只在首次生成时调 LLM，之后读缓存**：

```python
# routers/analysis/health_score.py calc_health_score()
async def calc_health_score():
    ...
    # 🆕 检查缓存
    cache_key = f"health_score_summary_{today}"
    cached = get_analysis_cache(cache_key)
    if cached:
        summary = cached["data"]["summary"]
    else:
        # 首次调 LLM
        summary = await _generate_llm_summary(...)
        save_analysis_cache(cache_key, {"summary": summary})
    
    # 🆕 同一天不再重新生成
```

同样逻辑应用到 `four_pots.py`、`fee_analyzer.py`、`correlation.py`、`rolling_return.py` 的 LLM 总结。

#### 开关 6：LLM-as-Judge 评分（3% 节省）

```python
# eval_system.py run_llm_judge() 中
async def run_llm_judge(case_type, expected_behavior, input_params, agent_output):

            "score_data_accuracy": 0, "score_actionability": 0,
            "score_risk_warning": 0, "score_hallucination": 0,
            "score_format": 0, "score_total": 0,
            "judge_comments": "LLM Judge 已关闭",
            "issues_found": [], "improvement_suggestions": [],
        }
    ...  # 正常执行
```

## 五、保留下来的核心功能

| 功能 | 为什么保留 |
|------|-----------|
| 用户对话 (Multi-Agent) | 核心价值，没有它系统就没用了 |
| 每日净值刷新 | 必须，数据新鲜度是分析基础 |
| 每日持仓提示引擎 | 纯规则计算，零 LLM 消耗 |
| 交叉审阅 + 仲裁 | 分析质量保障，Pro/Flash 分派降低费用 |
| 用户手动触发的分析 | 用户主动发起才消耗 |
