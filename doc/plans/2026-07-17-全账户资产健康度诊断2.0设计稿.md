# 全账户资产健康度诊断 2.0 设计稿

**日期**：2026-07-17  
**方向**：A — 全账户资产健康度诊断  
**目标**：把现有分散的健康分、四笔钱、组合体检、智能补仓、预警、决策候选等能力整合为"诊断→归因→行动→追踪"闭环，让用户每天一眼看清"我的钱健康吗？今天该做什么？"

---

## 1. 背景与问题

系统已具备多个健康度相关模块：

| 模块 | 位置 | 现状 |
|---|---|---|
| 投资健康分 | `backend/routers/analysis/health_score.py` + `frontend/src/components/analysis/HealthScore.vue` | 五维度 1000 分，参考蚂蚁财富"理财分"，但评分依赖 akshare 实时抓取，且只有分数展示，缺少归因与行动 |
| 四笔钱 | `backend/routers/analysis/four_pots.py` | 活钱/稳健/长期/保险归类 + 简单建议，未与目标配置对比 |
| 组合体检 | `backend/services/portfolio/portfolio_intelligence.py` | 7 维体检 + 风险度量 + 大师矩阵，但独立页面，未与健康分打通 |
| 智能补仓 | `backend/services/advisor/smart_add_planner.py` | 多引擎补仓计划，但建议与健康分/组合体检是割裂的 |
| 预警系统 | `backend/db/portfolio.py` 等 | 有 alert 表，但未在健康度视图中统一呈现 |
| 决策候选 | `backend/db/decisions.py` | 可沉淀建议，但健康诊断未直接生成候选 |

**核心问题**：功能点都在，但用户看到的是"多个孤立的数据面板"，而非"一个统一的理财体检报告 + 今日行动清单"。

---

## 2. 设计目标

借鉴成熟产品（蚂蚁理财分、且慢四笔钱、有知有行温度计、Empower 净资产视图、Betterment 目标驱动），实现：

1. **一页看清**：资产全景 + 健康总分 + 五维得分 + 今日待办
2. **归因到具体标的**：健康分低，能定位到哪只基金、哪个维度、哪项纪律
3. **可执行的行动清单**：按"影响力 × 紧迫性"排序，直接跳转补仓/调仓/决策
4. **目标配置对比**：用户设定风险等级/目标日期后，系统给出"当前配置 vs 目标配置"偏离度
5. **效果追踪闭环**：建议是否被采纳、采纳后的收益变化，反哺评分与推荐

|---|---|---|

## 4. 核心设计概念

### 4.1 三层诊断模型

```
┌─────────────────────────────────────────┐
│  L1 资产全景（Net Worth View）            │
│  总资产 = 持仓市值 + 现金 - 负债           │
├─────────────────────────────────────────┤
│  L2 健康度评分（Health Score 2.0）        │
│  总分 1000 = 选品 + 配置 + 估值 + 行为 + 风控 │
├─────────────────────────────────────────┤
│  L3 行动清单（Action Feed）               │
│  今天最值得做的 N 件事                     │
└─────────────────────────────────────────┘
```

### 4.2 健康分 2.0 升级点

| 维度 | 旧逻辑 | 新逻辑 | 数据来源 |
|---|---|---|---|
| 选品质量 | 同类排名 + 费率 + 估值 | 增加基金经理稳定性、规模健康度、费率分位、夏普比率 | `fund_analysis` + `fund_manager` + `smart_add_metrics` |
| 分散配置 | 持仓数量 + 集中度 + 股债比 | 增加行业分散度、地区分散度、标的间相关性、与目标配置偏离度 | `portfolio_intelligence` + 四笔钱 |
| 估值合理 | 持仓整体估值百分位 | 按市值加权的组合估值分位 + PB/PE 替代警告 + 周期股修正 | `db.valuations` + `smart_add_planner` |
| 持有行为 | 交易频率 + 追涨杀跌 + 持有时长 | 增加智能补仓执行情况、止损纪律、再平衡纪律、行为偏差识别 | `portfolio_transactions` + `behavior` |
| 风控纪律 | 最大回撤 + 单只仓位 | 增加 VaR/CVaR、组合波动率、现金占比、预警触发与处理率 | `portfolio_intelligence` + `alerts` |

### 4.3 行动清单排序算法

每个行动项计算两个分数：

- **影响力分数（Impact）**：对总资产或健康分的潜在提升（0-100）
- **紧迫性分数（Urgency）**：时间窗口（止盈止损 > 补仓机会 > 再平衡 > 长期优化）

```
Action Priority = Impact × Urgency × 可执行系数
```

可执行系数：用户是否有可用现金/是否已有该标的持仓/是否需要二次确认。

```
└──────────────────────┬───────────────────────────────────────┘
                       │ GET /api/health/v2/dashboard
                       ▼
┌──────────────────────────────────────────────────────────────┐
│              后端：HealthV2Service（新建服务层）               │
│  1. 聚合持仓、现金、估值、预警、补仓、决策候选                  │
│  2. 调用既有引擎计算五维得分                                    │
│  3. 生成行动清单并写入 decision_candidates                     │
└──────────────────────┬───────────────────────────────────────┘
                       │
       ┌───────────────┼───────────────┬───────────────┐
       ▼               ▼               ▼               ▼
  portfolio      valuations      smart_add       alerts
  four_pots    intelligence     decisions
```

**数据共享原则**：HealthV2Service 只做聚合与编排，不重复实现计算逻辑；评分与行动所需原始数据全部来自现有模块。


## 9. 算法设计

### 9.1 五维得分计算（加权复用）

```python
def calc_health_score_v2(user_id="default") -> dict:
    holdings = list_holdings(user_id)
    profile = get_user_profile(user_id)

    quality_score = calc_quality_v2(holdings)            # 复用 fund_analysis / smart_add_metrics
    diversification_score = calc_diversification_v2(holdings, profile)  # 复用 four_pots + intelligence
    valuation_score = calc_valuation_v2(holdings)        # 复用 valuations + smart_add_planner
    behavior_score = calc_behavior_v2(user_id)           # 复用 behavior + transactions
    risk_score = calc_risk_v2(holdings)                  # 复用 portfolio_intelligence

    return {
        "total_score": sum([quality, diversification, valuation, behavior, risk]),
        "dimensions": {...},
        "asset_snapshot": {...},
        "actions": generate_actions(...),
    }
```

### 9.2 行动优先级排序

```python
def priority_score(action: dict) -> float:
    impact = action.get("impact", 50)
    urgency = action.get("urgency", 50)
    executable = 1.0 if action.get("executable", True) else 0.3
    # 已拒绝的行动 7 天内不再推荐
    if recently_rejected(action["action_id"]):
        executable *= 0.1
    return impact * urgency * executable
```

### 9.3 四笔钱偏离度

```python
def pot_drift(actual: dict, target: dict) -> dict:
    result = {}
    for pot in ["cash", "steady", "long_term", "insurance"]:
        drift = actual.get(pot, 0) - target.get(pot, 0)
        result[pot] = {
            "actual_pct": actual[pot],
            "target_pct": target[pot],
            "drift_pct": drift,
            "status": "ok" if abs(drift) <= 5 else "warning" if abs(drift) <= 10 else "alert"
        }
    return result
```

## 13. 风险与边界

1. **数据一致性**：HealthV2Service 必须复用现有数据源，避免引入独立计算导致数值不一致。
2. **性能**：聚合多个模块可能慢；对 dashboard 接口增加 5 分钟内存缓存，支持 force_refresh。
3. **用户画像缺失**：新用户未设置风险等级时，默认使用"稳健"目标配置。
4. **负债数据**：当前系统无负债表，净资产初期只展示"总资产 - 0"，后续可扩展。
5. **开关控制**：所有新功能通过 `health_v2.enabled` 开关控制，默认开启，旧版 `/api/health/*` 保持兼容。
