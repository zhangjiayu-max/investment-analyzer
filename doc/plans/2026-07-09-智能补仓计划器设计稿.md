# 智能补仓计划器设计稿

**日期**：2026-07-09
**目标**：为深套持仓（如白酒消费 -35%）提供"估值分位加权定投 + 金字塔补仓"双引擎，生成前瞻性补仓计划表，明确"何时补、补多少、补到什么程度"。
**定位**：**增强**现有 `daily_position_advisor` DCA 引擎，不替代。现有引擎产出"今日信号"，本模块产出"多档位前瞻计划表"，两者数据源相同但产出形态互补。

---

## 一、问题诊断

| 现状 | 痛点 |
|------|------|
| 4% 等额定投 | 深套时摊薄力度不足，白酒 -35% 仍只投 4% 基础额 |
| 现有 dca_add 信号 | 倍率 1.0/1.5/2.0，对极端亏损不够激进 |
| 信号触发即建议 | 缺少"再跌 X% 就补 Y 元"的前瞻计划，用户心里没底 |
| 无资金池概念 | 补仓无总额上限，容易越补越深 |
| 单标的逐个评估 | 多个深套标的时，缺少"有限资金如何分配"的组合视角 |

---

## 二、双引擎设计

### 引擎1：估值 z-score 加权定投（日常，替代等额定投）

替换现有 4% 等额定投，改为 z-score 驱动的连续倍数定投：

```
实际月投 = 基础定投额(4%×总资产/12) × 估值倍数
估值倍数 = clamp(1.0 + (-z_score) × 0.5, 0, 3.0)
```

**z-score**：当前估值偏离历史均值的标准化程度（系统 `index_valuations.zscore` 字段已有），-2 = 比均值低 2 个标准差。

| z-score | 含义 | 倍数 | 场景 |
|---------|------|------|------|
| -3 | 极度低估（3σ）| ×2.5 | 罕见深度低估 |
| -2 | 深度低估（2σ）| ×2.0 | 白酒当前（z≈-2）|
| -1 | 低估（1σ）| ×1.5 | 普通低估 |
| 0 | 中位数 | ×1.0 | 合理区间 |
| +1 | 高估（1σ）| ×0.5 | 相对高估 |
| +2 | 深度高估（2σ）| ×0 | 暂停定投 |

**优势**：连续而非跳档，避免分位边界突变；有统计学依据；数据已有。

**数据来源**：复用 `db.valuations.get_best_valuation(index_code, metric_type)`，已含智能降级 + PB 替代 + 过期标记，返回值含 `zscore` 字段。

### 引擎2：金字塔补仓（极端下跌触发，新增）

针对亏损超过阈值的标的，从"补仓资金池"分档释放：

```
补仓资金池 = 总资产 × 15%（可配置）
```

| 亏损幅度 | 释放比例 | 示例（资金池¥30000）| 累计释放 |
|----------|----------|---------------------|----------|
| -10% | 15% | ¥4500 | ¥4500 |
| -20% | 25% | ¥7500 | ¥12000 |
| -30% | 30% | ¥9000 | ¥21000 |
| -40% | 20% | ¥6000 | ¥27000 |
| -50% | 10% | ¥3000 | ¥30000（耗尽）|

**特点**：
- 跌得越多，单档释放越多（中段 -30% 释放最大）
- 总额封顶，避免越补越深
- 每档有明确触发条件和金额

### 双引擎协同
- 日常：引擎1（估值加权定投）按月执行
- 极端：引擎2（金字塔补仓）在跌幅达标时触发，与引擎1**叠加**
- 示例：白酒 -35%、分位8% → 月投¥2500（引擎1）+ 已释放¥21000（引擎2累计）

## 三、核心产出：补仓计划表

每个深套标的生成一张前瞻性计划表：

```
┌─────────────────────────────────────────────────┐
│ 招商中证白酒 (161725)          亏损 -35%        │
│ PE分位 8%（深度低估）  持仓占比 18%              │
├─────────────────────────────────────────────────┤
│ 引擎1：估值加权定投                              │
│   基础额 ¥1000 × 倍数 2.5 = 月投 ¥2500          │
├─────────────────────────────────────────────────┤
│ 引擎2：金字塔补仓计划                            │
│   资金池 ¥30000（总资产15%）  已用 ¥21000       │
│   ─────────────────────────────────────         │
│   档位1 亏损-10%  释放¥4500   ✅ 已触发          │
│   档位2 亏损-20%  释放¥7500   ✅ 已触发          │
│   档位3 亏损-30%  释放¥9000   ✅ 已触发          │
│   档位4 亏损-40%  释放¥6000   ⏳ 待触发          │
│   档位5 亏损-50%  释放¥3000   ⏳ 待触发          │
├─────────────────────────────────────────────────┤
│ 预估摊薄效果                                     │
│   当前成本 -35% → 全触发后 -28%（改善7个百分点）│
│   下次触发：亏损达 -40% 时补 ¥6000              │
├─────────────────────────────────────────────────┤
│ 安全阀                                           │
│   单标的累计上限：总仓位 25%（当前18%，可继续）  │
│   资金池剩余：¥9000                             │
│   估值回升到 60%+ 自动暂停引擎2                  │
└─────────────────────────────────────────────────┘
```

**组合视角**（多深套标的时）：
```
补仓资金池总览（总资产 ¥200000）
├─ 白酒    -35%  已用¥21000  剩余¥9000   优先级★★★
├─ 消费    -28%  已用¥12000  剩余¥18000  优先级★★☆
├─ 医药    -22%  已用¥4500   剩余¥25500  优先级★☆☆
└─ 合计          已用¥37500  剩余¥52500

优先级规则：亏损幅度大 + 估值分位低 + 持仓占比低 → 优先补
```

## 五、与现有模块的关系

| 现有模块 | 关系 | 说明 |
|----------|------|------|
| `daily_position_advisor` | **并存互补** | 现有产出"今日信号"（触发即建议），本模块产出"前瞻计划表"（多档位预规划）|
| `get_dca_suggestion` | **降级兜底** | 本模块不可用时回退到简单 4% 定投法 |
| `rebalancer` | **数据互通** | 调仓建议的 `buy_index` 可参考本模块的补仓优先级 |
| `four_pots.dca_optimization` | **整合** | 恐贪指数+PE定投优化可并入引擎1的倍数修正 |

**关键原则**：不修改 `daily_position_advisor` 和 `get_dca_suggestion` 的现有逻辑，避免影响已稳定的 DCA 信号。新模块独立计算，前端独立展示，用户可对比参考。

## 六、安全阀（硬约束）

1. **单标的累计补仓 ≤ 总仓位 25%**：达到上限后停止该标的的所有补仓
2. **资金池总额封顶**：耗尽则停止引擎2，引擎1（定投）仍可执行
3. **估值回升暂停**：PE/PB 分位回升到 60%+ 时暂停引擎2（金字塔），引擎1倍数降为 ×1.0
4. **数据过期保护**：估值数据 >14 天时降级为 PB，PB 也过期则该标的不生成计划
5. **冷却期**：单标的 7 天内已补仓则跳过（复用 `daily_advice.recent_buy_cooldown_days`）
6. **亏损不足不触发**：亏损 <10% 不启动引擎2，只走引擎1定投

---

## 七、不做的事

- **不替代现有 DCA 引擎**：`daily_position_advisor` 保持不变，避免信号冲突
- **不做自动下单**：只生成计划和建议，用户手动确认执行（合规要求）
- **不做个股补仓**：仅限基金/指数，个股风险敞口不同
- **不做融资补仓**：资金池仅来自现有现金，不引入杠杆
- **不做回测引擎**：摊薄效果是"预估"而非历史回测（回测是另一个模块的事）

## 九、设计稿确认点

1. **定位**：增强现有 DCA，不替代 `daily_position_advisor`（两者并存互补）
2. **双引擎**：估值分位加权定投（日常）+ 金字塔补仓（极端）
3. **核心产出**：前瞻性补仓计划表（每标的一张 + 组合视角）
4. **安全阀**：单标的25%上限 + 资金池封顶 + 估值回升暂停 + 数据过期保护
5. **合规**：只生成计划，不自动下单
6. **配置默认值**：倍率保守（分位<20% ×2.5 而非 ×3），资金池15%，可调
