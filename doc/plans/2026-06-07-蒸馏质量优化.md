# 统一蒸馏脚本设计文档

## 1. 背景与目标

### 问题
- 原有 **11 个蒸馏脚本**，功能大量重叠、质量参差不齐
- 蒸馏质量差：176 个知识点中混入大量低质量内容
- 去重粗糙（仅标题前 15 字符），重复率高
- 扫描版 PDF 需要多模态 OCR 才能正确识别

### 目标
1. 统一为一个 `distill.py` 脚本
2. 支持完整流程：PDF → OCR（多模态）→ Markdown → 蒸馏 → 入库
3. 三阶段蒸馏保证质量：提取 → 去重 → 审核
4. OCR 结果保存为 Markdown 文件，便于复用

## 2. 脚本结构

```
backend/scripts/distill.py
├── OCR 模块（PDF → Markdown）
│   ├── detect_pdf_type()       # 检测文字版/扫描版
│   ├── extract_text_from_pdf() # 文字版直接提取
│   ├── ocr_scanned_pdf()       # 扫描版多模态 OCR
│   └── _merge_pages()          # 合并整理为 Markdown
│
├── 蒸馏模块（Markdown → 知识点）
│   ├── split_into_chunks()     # 按章节分块
│   ├── extract_knowledge()     # LLM 提取
│   ├── deduplicate()           # 基础去重
│   ├── deduplicate_semantic()  # 语义去重
│   └── review_knowledge()      # LLM 质量审核
│
├── 入库模块
│   └── distill_from_text()     # 存入 knowledge_base
│
└── CLI 命令
    ├── ocr     # 只做 OCR
    ├── distill # 只做蒸馏
    ├── full    # 完整流程
    ├── clean   # 清理旧数据
    └── list    # 查看已蒸馏书籍
```

## 3. 命令行用法

```bash
# 完整流程：PDF → OCR → Markdown → 蒸馏 → 入库
python3 scripts/distill.py full /path/to/book.pdf --name 书名

# 只做 OCR：PDF → Markdown
python3 scripts/distill.py ocr /path/to/book.pdf --name 书名

# 只做蒸馏：从已有 Markdown 文件
python3 scripts/distill.py distill /path/to/book.md

# 清理某书的旧数据
python3 scripts/distill.py clean --name 书名

# 查看已蒸馏书籍列表
python3 scripts/distill.py list

# 试运行（不写入数据库）
python3 scripts/distill.py full /path/to/book.pdf --name 书名 --dry-run

# 跳过质量审核
python3 scripts/distill.py distill /path/to/book.md --skip-review

# 指定页码范围
python3 scripts/distill.py ocr /path/to/book.pdf --name 书名 --start 100 --end 200
```

## 4. OCR 流程（PDF → Markdown）

### 输出路径
```
/Users/xiaoyuer/projects/investment-analyzer/data/books/
├── 债券投资实战.md
├── 股市长线法宝.md
├── 以交易为生.md
└── ...
```

### 流程
1. 检测 PDF 类型（文字版 vs 扫描版）
2. 文字版：PyPDF2 直接提取
3. 扫描版：
   - 每页转为 JPEG 图片（150 DPI）
   - 用多模态 LLM 逐页识别文字
   - 每 10 页合并整理一次
4. 输出 Markdown 文件到 `data/books/书名.md`

## 5. 蒸馏流程（三阶段）

### 第一阶段：分块提取
- 按章节标题（Markdown # 或 中文"第X章"）分块
- 每块最大 3000 字
- 用 LLM 提取高质量知识点

### 第二阶段：去重
- 基础去重：标题前 20 字符 + 关键词
- 语义去重：当知识点 > 20 条时，用 LLM 识别重复组

### 第三阶段：LLM 质量审核
- 将所有知识点一次性发给 LLM 进行全局审核
- 删除率应达 30-50%
- 只保留重要性 >= 7 分的内容
- 合并相似知识点

## 6. 完整 Prompt 列表

### 6.1 OCR 逐页识别 Prompt

```
请完整提取这张书籍页面中的所有文字内容。
要求：
1. 保持原文段落结构
2. 忽略页码、页眉页脚
3. 保留标题层级（用 # 标记）
4. 公式用 LaTeX 格式
5. 只输出文字内容，无其他说明
```

### 6.2 OCR 合并 Prompt（每 10 页）

```
请将以下多页书籍《{book_title}》的内容整理成连贯的 Markdown 文本。

要求：
1. 合并跨页的段落，去除断句
2. 保留所有重要内容（概念、公式、数据、案例）
3. 用 Markdown 格式组织（# 标题、## 子标题、- 列表等）
4. 去除重复内容
5. 保留原文含义，不要添加或修改内容
6. 如果有表格，用 Markdown 表格格式

## 原始内容：
{combined}

请输出整理后的 Markdown 文本：
```

### 6.3 知识提取 Prompt

```
你是金融投资知识提取专家。请从以下书籍内容中提取**最核心、最有实战价值**的投资知识点。

## 书名：{book_title}
## 章节：{chapter_title}

## 提取原则（严格遵守）

**宁缺毋滥**：宁可少提，不可滥提。每个知识点必须有**独特的实战价值**。

### 必须提取的内容（至少满足一项）
- 有具体步骤的投资策略（如"当XX指标超过YY时，分3批买入"）
- 量化的公式/阈值/参数（如"久期缺口控制在±0.5年以内"）
- 有数据支撑的规律（如"历史上10年期国债收益率低于3%时..."）
- 可直接执行的交易规则（如"止损线设为成本价的-5%"）

### 必须排除的内容（发现即删除）
- ❌ 概念定义（如"什么是久期"、"债券的定义"）
- ❌ 常识性内容（如"投资有风险"、"要分散投资"）
- ❌ 笼统建议（如"关注市场动态"、"做好风险管理"）
- ❌ 历史背景（如"我国债券市场发展历程"）
- ❌ 与其他知识点重复的内容

### 内容质量要求
- 每个知识点 200-500 字，**必须有具体数字、公式、步骤或案例**
- 独立完整，不依赖上下文
- 用通俗易懂的语言，但保留专业术语

### 分类标签
- strategy: 具体的买卖策略、仓位管理、择时方法
- indicator: 技术指标、估值指标、市场情绪指标
- principle: 经过验证的投资原则、交易规则
- case: 历史案例、经典交易、数据规律
- psychology: 交易心理、行为金融学洞见
- formula: 计算公式、量化模型

### 输出格式（JSON 数组）
[
  {"title": "精炼的标题（10-20字）", "category": "strategy|indicator|principle|case|psychology|formula", "content": "200-500字的详细内容，必须包含具体数字/公式/步骤", "keywords": ["关键词1", "关键词2", "关键词3"], "importance": 1-10, "reason": "一句话说明为什么这个知识点值得提取"},
  ...
]

**重要**：如果本段内容没有值得提取的高质量知识点，直接返回空数组 []。

## 文本内容：
{chunk}

只输出 JSON 数组，无其他文字。
```

### 6.4 语义去重 Prompt

```
请识别以下知识点中的**重复或高度相似**的内容，返回需要删除的索引号。

## 知识点列表：
{summaries}

## 判断标准
- 两个知识点讨论同一个概念/策略/公式，只是表述不同 → 标记为重复
- 两个知识点虽然关键词相似，但内容侧重点不同 → 不算重复
- 保留**内容最详细、最完整**的版本，删除简略版本

## 输出格式（JSON 数组）
只返回需要删除的索引号数组，如 [2, 5, 8]
如果没有重复，返回 []

只输出 JSON 数组，无其他文字。
```

### 6.5 质量审核 Prompt

```
你是金融投资知识质量审核专家。请对以下从《{book_title}》中提取的知识点进行**最终审核和优化**。

## 审核任务

1. **删除低质量内容**（删除率应达 30-50%）：
   - 删除：纯概念解释、常识性内容、笼统建议
   - 删除：与其他知识点高度重复的内容
   - 删除：缺乏具体数字/公式/步骤/案例的内容
   - 删除：重要性低于 7 分的内容

2. **合并相似内容**：
   - 如果多个知识点讨论同一概念，合并为一个最完整的版本
   - 合并后保留最详细的内容，补充其他版本的独特信息

3. **重新评分**：
   - importance 重新评估（1-10），7分以下的删除
   - 确保保留的都是真正有实战价值的核心知识

4. **优化内容**：
   - 如果某个知识点内容不够具体，补充关键细节
   - 确保每个知识点都包含具体数字、公式、步骤或案例

## 知识点列表（共 {count} 条）：

{items}

## 输出格式（JSON 数组）
[
  {"title": "优化后的标题", "category": "strategy|indicator|principle|case|psychology|formula", "content": "优化后的内容（200-500字）", "keywords": ["关键词1", "关键词2", "关键词3"], "importance": 7-10},
  ...
]

**重要**：
- 只输出审核通过的知识点，不通过的直接删除
- 每个保留的知识点必须有 7 分以上的重要性
- 总数应控制在原数量的 50-70%

只输出 JSON 数组，无其他文字。
```
