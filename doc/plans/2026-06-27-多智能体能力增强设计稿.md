## 二、对标成熟产品的 8 项缺失

### 2.1 估值分析师：缺行业横向对比

**现状：** 只看单指数 PE/PB 分位，判断"低估/高估"。

**成熟产品（韭圈儿）：** 同时展示同行业多指数的估值水位。比如"白酒 PE 分位 60%，但食品饮料行业平均 PE 分位 45%，白酒相对偏高"。

**增强方案：**
```
新增工具: query_peer_valuation(index_code)
  → 返回同类指数的 PE/PB 分位对比表
  → 标注"行业中位数"和"该指数排名"

Prompt 增强:
  "如果用户问'XX估值高不高'，必须横向对比同行业指数。
   单一 PE 分位不能说明问题——60% 分位在科技行业可能正常，在消费行业可能偏高。"
```

---

### 2.2 市场分析师：缺资金流向+板块轮动

**现状：** 主要分析热点新闻和宏观政策，缺少量化信号。

**成熟产品（同花顺/东方财富）：** 北向资金、主力资金、板块轮动地图。

**增强方案：**
```
新增工具:
  query_northbound_flow() → 北向资金净流入/流出
  query_sector_rotation() → 近5/20/60日板块涨跌幅排名
  query_market_breadth()  → 涨跌家数、涨停/跌停

Prompt 增强:
  "市场分析必须包含三个维度：
   1. 资金面：北向/主力资金流向
   2. 结构面：板块轮动状态（领涨/领跌板块）
   3. 情绪面：恐贪指数、成交量、涨停家数
   三者交叉验证，不只看单一信号。"
```

---

### 2.3 风险评估师：缺情景模拟

**现状：** 量化回撤、波动率，回答"风险多大"。

**成熟产品（Bloomberg/RiskMetrics）：** "如果利率上升 2%，组合会亏多少？"

**增强方案：**
```
新增工具: run_what_if_scenario(scenario, portfolio)
  → 预设场景：利率上行/市场下跌20%/风格切换/流动性冲击
  → 输出每只持仓的预估影响

Prompt 增强:
  "评估风险后，至少跑一个预设情景并给出结论：
   '在 XX 情景下，你的组合最大可能亏损 YY%。'
   不要只说'风险较高'，要量化。"
```

---

### 2.4 资产配置师：缺现金流+税务意识

**现状：** 给出股债配比、定投建议。

**成熟产品（Betterment/且慢）：** 考虑月度收支、短期大额支出、税务影响。

**增强方案：**
```
增强工具: query_portfolio() 返回时增加:
  - 月结余（从KYC画像）
  - 近期大额支出预告（从goal_buckets）
  - 红利税预估（A股持股>1月免税）

Prompt 增强:
  "配置建议必须考虑三个约束：
   1. 现金流：月结余能否支撑定投金额？
   2. 近期目标：3个月内有用款计划吗？
   3. 税务：赎回是否触发额外费用？
   如果分红再投资，提醒红利税影响。"
```

---

### 2.5 基金分析师：缺深度对比+经理画像

**现状：** 看排名、费率、持仓。

**成熟产品（天天基金/晨星）：** A vs B 全方位对比、基金经理历史业绩回撤分析。

**增强方案：**
```
新增能力: compare_funds(fund_a, fund_b)
  → 输出：收益对比(1/3/5年)、最大回撤对比、波动率对比、
           费率对比、基金经理稳定性对比、持仓重叠度

新增工具: query_fund_manager(fund_code)
  → 基金经理任职年限、管理规模、历史业绩、最大回撤、
    离职风险（管理多只基金+任期满5年=高风险）

Prompt 增强:
  "当用户问'A和B哪个好'时，必须从以下维度逐一对比：
   收益/风险/费率/经理/持仓/规模，缺一不可。
   最后给出明确偏好，不允许说'各有优劣'。"
```

---

### 2.6 理财顾问：缺生命周期+目标追踪

**现状：** KYC 适配，讲"适合你的"建议。

**成熟产品（Betterment、Vanguard）：** 基于年龄自动调整风险、追踪退休/教育/购房目标进度。

**增强方案：**
```
利用现有 goal_buckets 表，增强理财顾问的上下文注入:

提示词增强:
  "## 🎯 目标追踪（必读）
  你的目标：
  - 退休储备：XX万，当前进度 YY%，预计达成时间 ZZ
  - 子女教育：XX万，距离目标时间 AA 年
  
  在给出建议时，结合目标进度：
  - 距目标时间 < 3年 → 偏保守
  - 目标进度 < 50% → 可能需增加投入
  - 多目标冲突 → 按优先级排序"

新增工具: get_goal_progress()
  → 返回每个goal_bucket的进度、预计达成时间、建议月投入
```

---

### 2.7 行为教练：缺交易日志分析

**现状：** 识别行为偏差（追涨杀跌、损失厌恶），但判断依据来自对话内容。

**成熟产品（Robinhood/富途）：** 分析真实交易记录，量化行为模式。

**增强方案：**
```
新增工具: analyze_trade_patterns(user_id)
  → 从 portfolio_transactions 表分析:
    - 追涨倾向：买入时的PE分位中位数
    - 杀跌倾向：卖出时的亏损幅度中位数
    - 持有耐心：平均持仓天数
    - 频繁交易度：月均交易次数
    - 胜率：盈利交易占比

Prompt 增强:
  "分析用户行为前，先调用 analyze_trade_patterns 获取真实数据。
   不要凭对话猜测用户行为——用数据说话。
   比如：'你过去12个月共交易35次，买入时的平均PE分位是68%（偏高），
   其中8次在PE>80%时买入。建议下次买入前先查看估值分位。'"
```

---

### 2.8 反方观点：缺历史案例库

**现状：** 找"为什么不该做"的反面理由，靠 LLM 推理。

**成熟产品（Dalbar研究/GMO季报）：** 引用历史案例——"上次类似情况发生了什么"。

**增强方案：**
```
增强 RAG 知识库:
  从投资经典书籍中提取"反例"，建立结构化索引：
  - 2000年互联网泡沫：PE>100还追入的案例
  - 2008年金融危机：低位割肉的案例
  - 2015年A股股灾：杠杆爆仓的案例
  - 2020年新冠暴跌割肉后踏空的案例

Prompt 增强:
  "找到反例后，必须从知识库中搜索一个相似的历史案例并引用：
   '2008年金融危机期间，许多投资者在标普500跌了40%后清仓，
   但此后3年反弹了80%。你当前的情况有相似之处...'
   使用 search_knowledge 工具主动检索。"
```

## 四、Agent 间协作增强

### 4.1 当前问题：Agent 之间只有交叉审阅，没有主动协作

```
当前流程：
  用户提问 → Router选Agent → 各Agent独立分析 → 交叉审阅(分歧时) → 仲裁

缺失:
  Agent A对Agent B说: "你的分析用了PE分位25%，但这是两周前的数据了，请重新查"
  Agent B对Agent A说: "你说的风险我可以补充——这个行业还有政策风险你没提到"
```

### 4.2 增强：Agent 间的"追问"能力

```python
# 在 run_specialist 中，如果某个专家使用了过期数据，
# 自动触发一次补充查询并追加到上下文

def _check_data_freshness_in_specialist_result(result: dict) -> list[str]:
    """检测专家结果中的数据过期问题，生成追问列表。"""
    followups = []
    if result.get("valuation_date") and _days_old(result["valuation_date"]) > 7:
        followups.append("估值数据超过7天，需要的指数代码是: ...")
    return followups

# 追问自动追加到 llm_messages，LLM 判断是否需要重新查询
```

## 五、对话质量增强

### 5.1 当前问题：多轮对话中 Agent 可能"失忆"

```
用户: 沪深300怎么样
Agent: PE 25%, 低估 ← 正常

用户: 那中证500呢
Agent: PE 35%, 合理 ← 分析了，但用户上一轮想要的 "结论先行/操作建议" 格式丢了

用户: 两个哪个好
Agent: ... ← 丢失了前两轮的估值数据，重新查
```

### 5.2 增强：对话态追踪

```python
# 在 orchestrate() 的多轮循环中维护 conversation_state
conversation_state = {
    "asked_indexes": ["沪深300", "中证500"],  # 本轮问过的指数
    "last_analysis_type": "valuation",          # 上一轮分析类型
    "pending_comparison": True,                 # 是否还在对比中
    "user_preferred_format": "conclusion_first",# 用户偏好的格式
}
```

注入到每轮 LLM 调用的 system message 前：  
`"本轮上下文：你已分析了沪深300(PE 25%)和中证500(PE 35%)，用户正在做对比，请直接给出偏好判断。"`

## 六、总结

| 类别 | 数量 | 核心交付 |
|------|------|---------|
| Agent 能力增强 | 8 项 | 每个 Agent 补 1-2 个成熟产品有的能力 |
| Agent 协作增强 | 1 项 | 数据新鲜度追问 + 对话态追踪 |
| 对话质量增强 | 1 项 | 多轮对话不丢上下文 |

**P0 三个增强（估值对比 + 基金对比 + 资金流向）补上后，Agent 的分析深度直接对标专业理财顾问，用户不需要再去天天基金/韭圈儿手动查数据。**
