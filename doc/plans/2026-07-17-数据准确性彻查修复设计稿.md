## 二、P0 修复方案

### P0-1+2：基本面健康检查死代码修复

**现状分析**：
- `smart_add_planner.py:529` 调用 `from db.fund_info import get_fund_info` → 模块不存在 → ImportError 被 `except: pass` 吞掉
- 即使修复导入路径，`fund_info` 返回的字段中也没有 `fund_size`/`manager_change_date`/`management_fee`

**可用数据源**：
- `db/portfolio.py:1691` `lookup_fund_info(fund_code)` → 返回 `fund_manager`、`scale`（净资产规模）、`established`（成立日期）
- `services/fund/fund_analysis.py:348` `_fetch_fund_fee(fund_code)` → 通过 `ak.fund_open_fund_info_em(indicator="费率")` 获取管理费+托管费
- `portfolio_holdings` 表已有 `manager_name` 字段（`db/__init__.py:592`），可作为经理变更检测的基线

**修复方案**：

1. 重写 `_check_fund_health` 函数的数据获取逻辑：
   - 从 `lookup_fund_info` 获取基金规模（`scale` 字段）和基金经理（`fund_manager` 字段）
   - 从 `_fetch_fund_fee` 获取综合费率（管理费+托管费）
   - 经理变更检测：对比 `lookup_fund_info` 返回的 `fund_manager` 与 `portfolio_holdings.manager_name`，不一致则预警

2. 规模检查：`scale` 字段为字符串（如"45.23亿元"），需解析数值
   - >100 亿标记"规模较大"
   - 规模数据不可靠时不预警，标记 `data_available=False`

3. 费率检查：`_fetch_fund_fee` 返回综合费率（管理费+托管费）
   - >1.5% 标记"费率偏高"
   - akshare 调用失败时不预警

4. 经理变更检查：
   - 从 `lookup_fund_info` 获取当前基金经理名
   - 从 `portfolio_holdings.manager_name` 获取已存储的经理名
   - 不一致则标记"基金经理变更"
   - 若 `manager_name` 为空（首次），跳过检查但写入当前经理名

**代码位置**：`backend/services/advisor/smart_add_planner.py:502-581`

### P0-3：基金费率数据丢失修复

**现状分析**：
- `fund_data_service.py:145-147` 硬编码 `management_fee=None, custody_fee=None, subscription_fee=None`
- 原因：上游 `_fetch_fund_metadata_from_akshare` 用 `fund_overview_em` 接口，该接口不返回费率
- 已有可用的费率接口：`ak.fund_open_fund_info_em(symbol=..., indicator="费率")`（`fund_analysis.py:348` 已验证可用）

**修复方案**：

1. 在 `_fetch_fund_metadata_from_akshare` 中增加费率获取：
   - 调用 `ak.fund_open_fund_info_em(symbol=fund_code, indicator="费率")`
   - 解析 DataFrame 提取"管理费率"和"托管费率"（参考 `_fetch_fund_fee` 的解析逻辑）
   - 失败时保持 None（不影响元数据其他字段）

2. `_metadata_from_akshare_dict` 保留 `raw` 中的费率字段（不再硬编码 None）：
   ```python
   "management_fee": raw.get("management_fee"),
   "custody_fee": raw.get("custody_fee"),
   "subscription_fee": raw.get("subscription_fee"),  # akshare 该接口无此字段，保持 None
   ```

3. 缓存策略：费率数据不变频繁，复用 `fund_metadata` 表的 30 天刷新周期即可

**代码位置**：`backend/services/fund/fund_data_service.py:82-150`

## 三、P1 修复方案

### P1-4：止损信号逻辑反转修复

**现状**（`smart_add_planner.py:658`）：
```python
if val_pct is None or val_pct > stop_loss_valuation_pct:
    # 触发止损
```
估值数据缺失（None）时 → `val_pct is None` 为 True → 触发止损。**逻辑反转**。

**修复**：
```python
if val_pct is not None and val_pct > stop_loss_valuation_pct:
    # 估值非低估，建议止损
elif val_pct is None:
    # 估值数据缺失，无法判断，不触发止损但标记警告
    pass
```
数据缺失时**不触发止损**，避免错误建议。

**代码位置**：`backend/services/advisor/smart_add_planner.py:655-666`

### P1-5：网格交易放行 bug 修复

**现状**（`smart_add_planner.py:420-423`）：
```python
val_pct = valuation.get("percentile") if valuation else None
if val_pct is not None and (val_pct < 30 or val_pct > 70):
    return None
# percentile 为 None 时不 return，继续生成网格 ← BUG
```

**修复**：
```python
if val_pct is None or val_pct < 30 or val_pct > 70:
    return None
```
估值数据缺失时不生成网格计划。

**代码位置**：`backend/services/advisor/smart_add_planner.py:420-423`
