## 二、P1-A 信号灯规则增强

### 2.1 现状

[`watchlist_multidim.py:249-294 adjust_signal_by_multidim`](../../backend/services/advisor/watchlist_multidim.py) 当前仅 2 条规则:
- `green + tech=bear → yellow`
- `yellow + tech=bull → green`

缺陷:
1. **单一维度看空不够强**:tech=bear 但 capital 也=outflow 时,只降一档 green→yellow 偏弱
2. **无三重共振识别**:tech+capital+sentiment 三重同向是强信号,当前完全忽略
3. **无反向升级**:red 信号即使 tech 转 bull 也无法反弹为 yellow

### 2.2 改造方案

#### 2.2.1 复合规则(双重看空强制降级)

```python
# green + 双重看空 → 直接降 red
if signal_status == "green":
    bearish_count = sum([tech == "bear", capital == "outflow", sentiment == "greed"])
    if bearish_count >= 2:
        adjusted = "red"
        extra_notes.append("⚠️ 多维双重看空,强制降级为观望")
        return adjusted, final_reason, reasons

# yellow + 双重看空 → 降 red
if signal_status == "yellow":
    bearish_count = sum([tech == "bear", capital == "outflow", sentiment == "greed"])
    if bearish_count >= 2:
        adjusted = "red"
        extra_notes.append("⚠️ 多维双重看空,降级为暂不介入")
```

#### 2.2.2 三重共振(强信号识别)

```python
# 三重看多 → 强制升一档
bullish_count = sum([tech == "bull", capital == "inflow", sentiment == "fear"])
if bullish_count == 3:
    if signal_status == "red":
        adjusted = "yellow"  # 反弹信号

    elif signal_status == "yellow":
        adjusted = "green"
        extra_notes.append("✓ 三重共振看多,强信号确认")
    elif signal_status == "gray":
        adjusted = "yellow"
        extra_notes.append("✓ 三重共振看多,值得关注")

# 三重看空 → 强制降一档
bearish_count = sum([tech == "bear", capital == "outflow", sentiment == "greed"])
if bearish_count == 3:
    if signal_status == "green":
        adjusted = "red"
        extra_notes.append("⚠️ 三重共振看空,清仓观望")
    elif signal_status == "yellow":
        adjusted = "red"
        extra_notes.append("⚠️ 三重共振看空,暂不介入")
```

#### 2.2.3 反向升级(red → yellow)


## 三、P1-B 命中率反哺 + 分桶统计 + 交易日精度

### 3.1 现状

[`watchlist_multidim.py:297-324 compute_signal_confidence`](../../backend/services/advisor/watchlist_multidim.py) 当前权重:
- 估值偏离度 40% + 技术 20% + 资金 20% + 情绪 20%

缺陷:
1. **无历史命中率反哺**:回测数据已积累,但 confidence 未使用
2. **统计不分桶**:无法验证"高 confidence 是否真更准"
3. **review_date 不精准**:+21 自然日与 15 交易日偏差 3-5 天

### 3.2 改造方案

#### 3.2.1 历史命中率反哺 confidence

权重重分配(总和 100%):
- 估值偏离度 **35%**(原 40%)
- 技术面 **20%**(不变)
- 资金面 **15%**(原 20%)
- 情绪面 **15%**(原 20%)
- **历史命中率 15%**(新增)

```python
def compute_signal_confidence(signal_status, multidim, fund_code=None):
    base_map = {"green": 80, "yellow": 55, "red": 20, "gray": 0}
    base = base_map.get(signal_status, 50)

    # 历史命中率反哺(需 fund_code)
    history_bonus = 0
    if fund_code:
        from db.watchlist import get_signal_backtest_stats
        stats = get_signal_backtest_stats(fund_code=fund_code)
        if stats.get("reviewed", 0) >= 3:  # 样本量 >= 3 才反哺
            hit_rate = stats.get("hit_rate", 0) or 0
            if hit_rate >= 70:
                history_bonus = 10
            elif hit_rate < 40:
                history_bonus = -15
            # 40-70 不调整

    if not multidim:
        # 无多维数据,confidence 上限 60(扣减 history_bonus 后)
        return max(0, min(60, int(base) + history_bonus))

    # 5 维加权
    confidence = int(
        base * 0.35
        + tech_score * 0.20
        + capital_score * 0.15
        + sentiment_score * 0.15
        + (50 + history_bonus * 2) * 0.15  # 历史维度:50 为中性,bonus 翻倍后归一
    )
    return max(0, min(100, confidence))
```

#### 3.2.3 review_date 交易日精度

新增工具函数 `_calc_review_date(signal_date, trade_days=15) -> str`:

```python
def _calc_review_date(signal_date_str, trade_days=15):
    """计算 N 交易日后的日期。

    优先用 akshare 交易日历;失败兜底 +21 自然日。
    """
    try:
        import akshare as ak
        df = ak.tool_trade_date_hist_sina()
        trade_dates = set(df['trade_date'].astype(str).tolist())
        # 从 signal_date 之后开始数 trade_days 个交易日
        from datetime import datetime, timedelta
        current = datetime.strptime(signal_date_str, "%Y-%m-%d")
        count = 0
        while count < trade_days:
            current += timedelta(days=1)
            if current.strftime("%Y-%m-%d") in trade_dates:
                count += 1
        return current.strftime("%Y-%m-%d")
    except Exception:
        # 兜底:+21 自然日
        from datetime import datetime, timedelta
        dt = datetime.strptime(signal_date_str, "%Y-%m-%d") + timedelta(days=21)
        return dt.strftime("%Y-%m-%d")
```

## 四、P1-C 退出信号动态化

### 4.1 现状

[`watchlist.py:397-430`](../../backend/routers/portfolio/watchlist.py) 退出信号:
- `target_profit_pct` 固定止盈
- `stop_loss_pct` 固定止损

缺陷:
1. **无移动止盈**:涨超 +20% 后回撤 5% 才止盈,锁定更多利润
2. **无保本止损**:涨超 +10% 后仍按原 -10% 止损,可能从盈利转亏损
3. **无时间止损**:持有 30 天未达 +3% 提示减仓,避免资金沉淀

### 4.2 改造方案

#### 4.2.1 watchlist 表新增字段

- `high_water_mark` REAL:历史最高价(用于移动止盈)
- `entry_date` TEXT:已有,用于时间止损计算

#### 4.2.2 移动止盈(activated when pnl_pct >= 20%)

```python
# 巡检时更新 high_water_mark
if current_nav and (not item.get("high_water_mark") or current_nav > item["high_water_mark"]):
    update_watchlist_item(item["id"], high_water_mark=current_nav)

# 移动止盈判定
if pnl_pct >= 20 and item.get("high_water_mark"):
    hwm = item["high_water_mark"]
    drawdown_from_peak = (current_nav - hwm) / hwm * 100
    if drawdown_from_peak <= -5:  # 从最高点回撤 5%
        exit_signal = "moving_profit_target"
        exit_signal_reason = f"已从最高点 {hwm:.3f} 回撤 {abs(drawdown_from_peak):.1f}%,移动止盈"
```

#### 4.2.3 保本止损(activated when pnl_pct >= 10%)

```python
if pnl_pct >= 10:
    # 止损线从 -stop_loss_pct 提升至 +5%
    breakeven_threshold = 5
    if pnl_pct <= breakeven_threshold:
        exit_signal = "breakeven_stop_loss"
        exit_signal_reason = f"已涨超 10% 后回落至 {pnl_pct:.1f}%,触发保本止损(>5%)"
```

#### 4.2.4 时间止损(activated when holding_days >= 30)

```python
entry_date = item.get("entry_date")
if entry_date:
    holding_days = (datetime.now() - datetime.strptime(entry_date, "%Y-%m-%d")).days
    if holding_days >= 30 and pnl_pct < 3:
        exit_signal = "time_stop_loss"
        exit_signal_reason = f"持有 {holding_days} 天涨幅仅 {pnl_pct:.1f}%,时间止损提示减仓"
```

### 4.3 退出信号类型扩展

原有 `profit_target` / `stop_loss`,新增:
- `moving_profit_target`:移动止盈(severity=info)
- `breakeven_stop_loss`:保本止损(severity=warning)
- `time_stop_loss`:时间止损(severity=warning)

### 4.4 开关

- `watchlist.moving_profit_target_enabled` 默认 **true**
- `watchlist.breakeven_stop_loss_enabled` 默认 **true**
- `watchlist.time_stop_loss_enabled` 默认 **true**

## 五、P1-D 信号有效期 + 缓存分层

### 5.1 现状

1. [`watchlist.py:317-360`](../../backend/routers/portfolio/watchlist.py) green 信号一旦触发持续 green,即使 5 天后估值已涨 20% 仍是 green。
2. [`watchlist_multidim.py:22-26`](../../backend/services/advisor/watchlist_multidim.py) 5 分钟全局缓存,但情绪指标是全局共享的,每只基金都查一次。

### 5.2 改造方案

#### 5.2.1 信号有效期

watchlist 表新增 `signal_triggered_at` TEXT。

巡检逻辑:

```python
# green 信号有效期检查
if signal_status == "green" and item.get("signal_triggered_at"):
    triggered_at = datetime.strptime(item["signal_triggered_at"], "%Y-%m-%d %H:%M:%S")
    days_since = (datetime.now() - triggered_at).days

    # 规则 1:5 天前触发 + 估值已涨 20% → 降 yellow
    if days_since >= 5:
        # 查 5 天前估值,与当前对比
        # 简化:用 current_pct 与触发时 entry_percentile 对比
        entry_pct = item.get("entry_percentile")  # 信号触发时百分位
        if entry_pct and current_pct and current_pct - entry_pct >= 20:
            signal_status = "yellow"
            signal_reason += "(信号超 5 天且估值已涨 20%,自动降级)"

    # 规则 2:10 天前触发 + 多维转 bear → 降 yellow
    if days_since >= 10 and tech_signal == "bear":
        signal_status = "yellow"
        signal_reason += "(信号超 10 天且技术面转空,自动降级)"

# green 首次触发时写入 signal_triggered_at
if signal_status == "green" and last_status != "green":
    update_fields["signal_triggered_at"] = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
elif signal_status != "green":
    # 信号消失时清空
    update_fields["signal_triggered_at"] = None
```

#### 5.2.2 缓存分层

`watchlist_multidim.py` 重构缓存:

```python
# 全局指标缓存(情绪/北向资金)— 30 分钟,所有基金共享
_global_cache = {}
_global_cache_lock = threading.Lock()
_GLOBAL_CACHE_TTL = 30 * 60

# 主题级缓存(技术指标)— 5 分钟,按指数代码
_theme_cache = {}
_theme_cache_lock = threading.Lock()
_THEME_CACHE_TTL = 5 * 60


def _get_sentiment_score_cached():
    """情绪指标(全局共享,30 分钟缓存)。"""
    with _global_cache_lock:
        cached = _global_cache.get("sentiment")
        if cached and time.time() - cached[0] < _GLOBAL_CACHE_TTL:
            return cached[1]
    from services.advisor.opportunity_engine import _get_sentiment_score
    score, signal = _get_sentiment_score()
    result = {"score": score, "signal": signal}
    with _global_cache_lock:
        _global_cache["sentiment"] = (time.time(), result)
    return result


def _get_capital_flow_score_cached(theme_rule):
    """资金流向(全局共享,30 分钟缓存)。"""
    with _global_cache_lock:
        cached = _global_cache.get("capital")
        if cached and time.time() - cached[0] < _GLOBAL_CACHE_TTL:
            return cached[1]
    from services.advisor.opportunity_engine import _get_capital_flow_score
    score, signal = _get_capital_flow_score(theme_rule)
    result = {"score": score, "signal": signal}
    with _global_cache_lock:
        _global_cache["capital"] = (time.time(), result)
    return result
```

`_compute_multidim_signals` 改用上述缓存函数,主题级缓存(技术指标)保持 5 分钟。

## 八、风险与降级

1. **历史命中率反哺样本不足**:`reviewed < 3` 时不反哺,confidence 维持原计算
2. **akshare 交易日历失败**:兜底 +21 自然日,不影响回测触发
3. **移动止盈 high_water_mark 历史数据缺失**:已有 entry_price 但无 hwm 的,首次巡检时用 current_nav 初始化
4. **三重共振误判**:三重共振需 3 个维度都有非 neutral 数据,任一缺失不触发
5. **信号有效期 5/10 天阈值**:`signal_triggered_at` 字段对历史 green 项为空,首次巡检时初始化,不立即触发降级
