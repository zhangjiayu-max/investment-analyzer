# 对话质量增强：失败检测 + Agent 调度 + 新闻关键词 + 前端样式

- 日期：2026-07-08
- 范围：backend `agent/multi_agent.py`、`agent/orchestrator.py`、`agent/pipeline.py`、`db/agents.py`、`routers/conversations.py`、`tools/__init__.py`；frontend `chat/ChatMessage.vue`、`chat/StreamIndicator.vue`
- 触发：conv 95 暴露的三个问题——macro_strategist 失败被掩盖、新闻搜索关键词差、补仓类问题缺 allocation_advisor

---

## 1. 问题与目标

### 1.1 conv 95 暴露的问题

| 问题 | 根因 | 影响 |
|------|------|------|
| macro_strategist 5 次 LLM 调用后产出"分析过程遇到问题，请重试" | `run_specialist` ReAct 3 轮不收敛 + 收口失败 | 自然灾害利好问题完全没回答 |
| `specialist_completion=1.0` 显示完成 | `_calculate_quality_metrics` 只判 `analysis` 非空，兜底文本也算完成 | 质量指标失真，用户以为成功 |
| 补仓类问题没选 allocation_advisor | LLM 路由 prompt 没强制"持仓操作必选 allocation_advisor" | 没看用户实际持仓就给补仓建议 |
| 新闻搜索"自然灾害 洪涝 地震 市场影响"返回 0 条 | macro_strategist 的 system_prompt 没有搜索策略指引 | LLM 用事件词而非板块词搜索 |

### 1.2 目标

- 专家失败时 `status=failed`，质量指标真实反映失败
- 前端专家失败态有视觉区分（红色边框 + 重试按钮）
- 补仓/加仓/减仓类问题必选 allocation_advisor
- macro_strategist 搜索新闻用板块名而非事件本身
- 工具无结果时前端显示"未找到相关数据"
- 移动端样式适配（≤768px 纵向堆叠，≤480px 反馈按钮常显）

### 2.2 `specialist_completion` 质量门槛加严

**文件**：`backend/routers/conversations.py:2156-2162`

```python
_FALLBACK_BLACKLIST = {"分析过程遇到问题，请重试。", "交叉审阅完成，请参考其他专家分析。"}

if specialist_results:
    completed = len([s for s in specialist_results
                    if s.get("analysis")
                    and s.get("status", "success") == "success"
                    and s["analysis"].strip() not in _FALLBACK_BLACKLIST
                    and len(s["analysis"].strip()) >= 200])
    specialist_completion = completed / len(specialist_results)
else:
    specialist_completion = 0.0
```

## 3. 后端：新闻关键词策略

### 3.1 macro_strategist system_prompt 增加搜索策略

**文件**：`backend/db/agents.py:445-471`

在"思维链"之后追加：

```
## 搜索策略（重要）
调用 yingmi_search_news 时：
1. 关键词用【板块/行业名】而非事件本身。如"自然灾害"→搜"基建/水利/农产品/水泥"
2. 单次关键词不超过 6 个字，避免长句
3. 一次分析最多搜 3 个不同板块关键词
4. 搜不到结果时换板块名重试，不要重复搜同一关键词
```

### 3.2 数据库迁移

macro_strategist 的 system_prompt 存在 `agents` 表中，需要 UPDATE 语句更新已有记录。`init_db` 中的种子数据用 `INSERT OR IGNORE`，已存在的记录不会被覆盖，需要单独的迁移逻辑。

在 `init_db()` 末尾添加迁移：

```python
# 迁移：更新 macro_strategist 的 system_prompt（增加搜索策略）
try:
    conn.execute("""
        UPDATE agents SET system_prompt = system_prompt || 
        '\n\n## 搜索策略（重要）\n调用 yingmi_search_news 时：\n1. 关键词用【板块/行业名】而非事件本身。如"自然灾害"→搜"基建/水利/农产品/水泥"\n2. 单次关键词不超过 6 个字，避免长句\n3. 一次分析最多搜 3 个不同板块关键词\n4. 搜不到结果时换板块名重试，不要重复搜同一关键词'
        WHERE agent_key = 'macro_strategist'
        AND system_prompt NOT LIKE '%搜索策略%'
    """)
    conn.commit()
except Exception as e:
    print(f"[db] macro_strategist prompt 迁移失败: {e}")
```
