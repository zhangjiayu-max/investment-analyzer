## 问题分析

1. **数据来源单一**：主要来自公众号文章，缺乏系统性知识
2. **知识深度不足**：缺少经典投资理论、策略框架
3. **更新频率依赖人工**：需要手动抓取文章

### 三、知识蒸馏方案

#### 方案 A: LLM 辅助蒸馏（推荐）

```python
# 使用 LLM 从书籍/文章中提取结构化知识
def distill_knowledge(source_text: str, category: str) -> dict:
    """
    从原始文本中蒸馏出结构化知识

    输入: 原始文本（书籍章节、文章）
    输出: {
        "concept": "概念名称",
        "definition": "定义",
        "formula": "公式（如有）",
        "application": "应用场景",
        "examples": ["示例1", "示例2"],
        "related_concepts": ["相关概念1", "相关概念2"]
    }
    """
    prompt = f"""
    从以下投资/理财内容中提取核心知识，以 JSON 格式返回：

    分类: {category}
    原文: {source_text}

    要求:
    1. 提取核心概念和定义
    2. 如有公式，保留原始公式
    3. 说明应用场景
    4. 提供 1-2 个示例
    5. 列出相关概念
    """

    return call_llm(prompt)
```

### 四、实现步骤

#### Phase 1: 基础知识库建设（1-2 周）

1. **创建知识分类表**
```sql
CREATE TABLE IF NOT EXISTS knowledge_base (
    id INTEGER PRIMARY KEY,
    category TEXT NOT NULL,        -- 'theory' | 'strategy' | 'concept' | 'article'
    subcategory TEXT,              -- 'valuation' | 'allocation' | 'timing'
    title TEXT NOT NULL,
    content TEXT NOT NULL,
    source TEXT,                   -- 来源（书籍名、文章URL）
    keywords TEXT,                 -- 关键词（JSON 数组）
    embedding_id TEXT,             -- ChromaDB ID
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

2. **导入基础概念库**
   - 估值指标（PE/PB/PS/PEG/股息率）
   - 风险指标（波动率/夏普/回撤）
   - 基金类型（指数/主动/ETF/QDII）
   - 宏观指标（GDP/CPI/PMI）

3. **导入策略框架**
   - 资产配置策略（全天候/60-40/风险平价）
   - 估值策略（百分位/均值回归）
   - 定投策略（普通/估值/均线）

#### Phase 2: 书籍知识蒸馏（2-3 周）

1. **准备书籍素材**
   - 购买/下载电子版书籍
   - 提取关键章节（不用全书）
   - 人工标注重点段落

2. **LLM 蒸馏处理**
   - 分章节喂给 LLM
   - 提取核心概念、公式、策略
   - 结构化存储到知识库

3. **质量审核**
   - 人工抽检蒸馏结果
   - 修正错误/补充缺失
   - 添加实际案例

#### Phase 3: 智能检索升级（1 周）

1. **知识图谱构建**
   - 概念关联关系
   - 策略适用场景
   - 指标计算公式

2. **RAG 检索增强**
   - 多层级检索（理论 + 策略 + 数据 + 文章）
   - 概念扩展（查询 "PE" 时关联 "估值"、"百分位"）
   - 上下文感知（根据用户持仓调整检索结果）
