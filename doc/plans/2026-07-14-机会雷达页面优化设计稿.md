# 机会雷达页面优化设计稿

> 日期：2026-07-14
> 组件：`frontend/src/components/market/EventRadarPage.vue`
> 目标：去重精简、关注机会数据增强、新增交互、统一前后端信号逻辑

## 一、去重与精简

### 1.1 统计卡片去冗余（6→4个）

**问题**：「信号触发」⊇「目标到位」+「低估数量」，3个卡片有包含关系。

**方案**：保留4个更有信息量的卡片：

| 卡片 | 计算 | 可点击筛选 |
|------|------|-----------|
| 关注总数 | `watchlist.length` | 否（全量） |
| 信号触发 | `signal_status === 'green'` 数量 | 是→只看绿灯 |
| 已设目标 | `target_price \|\| target_percentile` | 是→只看已设 |
| 近7日新增 | `created_at` 7天内 | 是→只看新增 |

移除：「低估数量」（被信号触发覆盖）、「目标到位」（被信号触发覆盖）

### 1.2 大师矩阵UI去重

**问题**：组合版(行1325-1416)与单基金版(行1766-1868)渲染相同结构。

**方案**：提取为子组件 `WatchlistMasterMatrix.vue`，接收 `data` prop，两处复用。不新建文件，在 EventRadarPage 内用 `<script setup>` 的局部组件方式定义。

### 1.3 7维体检UI去重

**问题**：组合7维(行1303-1321)与单基金体检(行1742-1760) UI相同。

**方案**：同上，提取为局部组件 `HealthDimensions`，接收 `dimensions` prop。

### 1.4 Tab3落地验证精简

**问题**：Tab3是Tab1已验证事件的子集，重复展示。

**方案**：Tab3保留准确率统计面板（有价值），移除已验证事件列表（Tab1已有），改为一句引导文案"已验证事件请在事件雷达中查看，筛选'已落地'状态"。

## 二、统一前后端信号逻辑

**问题**：前端 `watchlistSignalState`（行789，3态）与后端 `_compute_signal_status`（4态）逻辑重复且不一致。后端 patrol 返回 `signal_status`/`signal_reason`/`distance_to_target` 但前端丢弃后重算。

**方案**：
1. `autoPatrol` 取数据时保存 `signal_status`、`signal_reason`、`distance_to_target` 到 watchlist item
2. 移除前端 `watchlistSignalState` 函数
3. 信号标签直接用后端返回的 `signal_status`：

| signal_status | 标签 | 样式 |
|---------------|------|------|
| green | 信号触发 | 绿色 |
| yellow | 接近目标 | 橙色 |
| red | 估值仍高 | 红色 |
| gray | 数据缺失 | 灰色 |

## 三、关注机会数据增强

### 3.1 接入后端已返回字段

| 字段 | 展示位置 | 说明 |
|------|---------|------|
| `signal_status` | 卡片头部信号标签 | 替换前端重算 |
| `signal_reason` | 信号标签 tooltip | 后端计算的触发原因 |
| `distance_to_target` | 目标价行 | 显示"距目标 X%" |
| `pe` / `pb` | 估值分位行旁 | 显示 PE/PB 值 |
| `change_pct` | 净值行 | 显示"今日 ±X%" |
| `source` | 净值行 tooltip | 数据来源(本地/天天基金) |

### 3.2 priority 优先级展示

卡片头部基金名旁加优先级标签：高/中/低（priority ≥ 2 高，=1 中，=0 低）

### 3.3 fund_category 筛选

筛选栏增加"类型"下拉：全部/股票/混合/债券/指数/QDII（从 watchlist 数据动态提取）

## 四、新增交互功能

### 4.1 标记已买入

卡片操作区增加"已买入"按钮，调用 `markWatchlistBought(id)`，成功后 item.status 变为 bought，卡片标记"已买入"角标，不再参与信号统计。

### 4.2 添加关注入口

Tab2 顶部增加"添加基金"按钮，弹出输入框：基金代码 + 可选目标价/目标分位，调用 `addWatchlist` API。

### 4.3 notes事件标题可跳转

notes 文本中"来自事件「title」"的 title 部分改为可点击链接，点击切换到 events Tab 并滚动到对应事件。

## 五、改动范围

仅改动 `EventRadarPage.vue` 单文件：
- 统计卡片 6→4 + 可点击筛选
- 移除 `watchlistSignalState`，用后端 signal_status
- autoPatrol 保存完整 patrol 返回字段
- 卡片新增字段展示（distance/pe/pb/change_pct/priority）
- 提取局部组件 MasterMatrix + HealthDimensions
- Tab3 精简
- 新增：标记已买入 / 添加关注 / notes 跳转

无后端改动（所有数据后端已返回）。
