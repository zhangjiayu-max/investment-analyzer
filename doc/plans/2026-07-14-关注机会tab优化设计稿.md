# 关注机会 Tab 优化设计稿

> 日期：2026-07-14
> 组件：`frontend/src/components/market/EventRadarPage.vue` — 关注机会 Tab
> 目标：丰富统计维度、标注关键时间、支持多维度排序

## 一、现状分析

### 统计卡片（仅3个，信息量不足）
| 卡片 | 取值 |
|------|------|
| 关注总数 | `watchlist.length` |
| 已设目标 | `target_price \|\| target_percentile` 的数量 |
| 目标价到位 | `current_nav <= target_price` 的数量 |

### 时间标注（缺失）
- 卡片底部仅显示 `nav_updated_at`（净值更新时间）
- **未显示** `created_at`（加入关注时间）— 用户不知道关注了多久
- **未显示** `updated_at`（目标设置更新时间）— 用户不知道目标何时设的

### 排序（无）
- 固定按后端返回顺序（priority DESC, created_at DESC）
- 无法按估值分位、最近更新等维度查看

## 二、优化方案

### 1. 统计卡片扩展（3 → 6个）

布局从 `stats-row-3` 改为 `stats-row-6`（移动端自适应换行）。

| 卡片 | 取值 | 颜色 | 业务含义 |
|------|------|------|----------|
| 关注总数 | `watchlist.length` | 默认(蓝) | 总览 |
| 信号触发 | `watchlistSignalState(w).cls === 'sig-hit'` 数量 | 橙 | 目标价到位或低估值区，可行动 |
| 低估数量 | `current_percentile <= 20` 数量 | 绿 | 估值分位≤20%，低估机会 |
| 已设目标 | `target_price \|\| target_percentile` 数量 | 默认 | 设置过目标的基金 |
| 目标到位 | `current_nav <= target_price` 数量 | 橙 | 净值已跌到目标价 |
| 近7日新增 | `created_at` 在7天内数量 | 默认 | 近期关注热度 |

### 2. 卡片时间标注

在卡片中增加3处时间标注：

**a. 加入时间**（卡片头部，基金名旁）
- 数据源：`item.created_at`
- 显示：`加入于 07-10`（MM-DD 格式，超过1年显示 MM-DD-YY）
- 样式：小字 muted，与基金名同行右侧

**b. 目标设置时间**（目标价数据行）
- 数据源：`item.updated_at`
- 显示：在目标价值后追加 `（07-12设）`，仅当设置过目标时显示
- 样式：小字 muted

**c. 净值更新时间**（卡片底部，保留现有）
- 数据源：`item.nav_updated_at`
- 显示：`更新于 07-14 09:30`（保留现有逻辑）

### 3. 排序功能

在统计卡片下方、卡片列表上方增加排序工具栏。

**排序选项**（单选，按钮组样式）：
| 选项 | 排序逻辑 |
|------|----------|
| 默认 | 后端返回顺序（priority DESC, created_at DESC） |
| 估值低→高 | `current_percentile` 升序（null 排最后） |
| 最近加入 | `created_at` 降序（最新在前） |
| 最近更新 | `nav_updated_at` 降序（null 排最后） |

**实现**：
- 新增 `watchlistSort` ref，默认 `'default'`
- 新增 `sortedWatchlist` computed，在 `watchlist` 基础上排序
- 模板 `v-for="item in watchlist"` 改为 `v-for="item in sortedWatchlist"`

## 三、改动范围

仅改动 `EventRadarPage.vue` 单文件：
- `watchlistStats` computed 扩展（+3个维度）
- 新增 `watchlistSort` ref + `sortedWatchlist` computed
- 新增时间格式化辅助函数 `formatShortDate`
- 模板：统计卡片 3→6、卡片头部加加入时间、目标价行加设置时间、列表上方加排序工具栏
- 样式：`stats-row-6` 响应式、排序按钮组样式、时间标注样式

无后端改动（时间字段 `created_at`/`updated_at`/`nav_updated_at` 后端已返回，前端未用）。
