## 优化方案

### 方案 A：修复数据采集（统计准确）

#### A1. 修复 `analyze_article_stream` 漏报

**位置**：[llm_service.py:292-321](file:///Users/xiaoyuer/projects/investment-analyzer/backend/services/llm_service.py#L292)

**改动**：在流式 yield 过程中收集 content，结束后用 tiktoken 估算并记录。

```python
def analyze_article_stream(title: str, content: str, market_data: dict = None):
    user_msg = f"## 文章标题\n{title}\n\n## 文章正文\n{content[:8000]}"
    if market_data:
        user_msg += f"\n\n## 相关行情数据\n{market_data}"

    @_llm_retry
    def _stream():
        return client.chat.completions.create(
            model=MODEL,
            messages=[
                {"role": "system", "content": SYSTEM_PROMPT},
                {"role": "user", "content": user_msg},
            ],
            temperature=get_config_float('llm.temperature_default', 0.3),
            max_tokens=get_config_int('llm.max_tokens_chat', 8000),
            stream=True,
        )

    stream = _stream()
    _collected = []
    _usage = None
    for chunk in stream:
        if chunk.choices and chunk.choices[0].delta.content:
            _collected.append(chunk.choices[0].delta.content)
            yield chunk.choices[0].delta.content
        # 取末包 usage（部分模型流式会返回）
        if hasattr(chunk, "usage") and chunk.usage:
            _usage = chunk.usage

    # 记录 token：优先用 API 返回的 usage，否则 tiktoken 估算
    if not _usage:
        try:
            import tiktoken
            enc = tiktoken.encoding_for_model("gpt-4")
            prompt_tokens = len(enc.encode(SYSTEM_PROMPT + user_msg))
            completion_tokens = len(enc.encode("".join(_collected)))
            _usage = _UsageStub(prompt_tokens, completion_tokens, prompt_tokens + completion_tokens)
        except Exception:
            _usage = _UsageStub(0, 0, 0)
    _record_token_usage(_usage, MODEL, caller="article_analysis_stream")
```

**效果**：文章分析调用不再漏报，预计补全约 100-200 条/月的 token 记录。

#### A2. 修复 `hallucination_guard` 污染

**位置**：[hallucination_guard.py:250-272](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/hallucination_guard.py#L250)

**改动**：不再写入 token_usage 表，改用独立的 `hallucination_events` 表（如已存在则复用，否则在 db/__init__.py 新建）。

```python
def _record_hallucination(conversation_id: int, message_id: int, hallucinations: list[dict]):
    """将幻觉检测结果持久化到独立表，不污染 token_usage。"""
    try:
        from db._conn import _get_conn
        conn = _get_conn()
        # 确保表存在
        conn.execute("""
            CREATE TABLE IF NOT EXISTS hallucination_events (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                conversation_id INTEGER,
                message_id INTEGER,
                fund_code TEXT,
                claimed_name TEXT,
                detected_at TEXT DEFAULT (datetime('now','localtime'))
            )
        """)
        for h in hallucinations:
            conn.execute(
                """INSERT INTO hallucination_events
                   (conversation_id, message_id, fund_code, claimed_name)
                   VALUES (?, ?, ?, ?)""",
                (conversation_id, message_id, h.get('code', ''), h.get('claimed', '')[:200]),
            )
        conn.commit()
        conn.close()
        logger.info(f"幻觉记录已写入 hallucination_events: {len(hallucinations)}条")
    except Exception as e:
        logger.debug(f"幻觉记录写入失败（不影响主流程）: {e}")
```

**效果**：token_usage 表不再有 token=0 的污染记录，avg_per_call 恢复准确。

```js
// 归组逻辑
function groupSpecialistCalls(records) {
  const groups = {}
  for (const r of records) {
    const baseCaller = r.caller.replace(/#turn\d+/, '').replace(/#summary/, '')
    const traceKey = r.trace_id || 'no-trace'
    const key = `${traceKey}|${baseCaller}`
    if (!groups[key]) groups[key] = { ...r, caller: baseCaller, turns: [], total_tokens: 0 }
    groups[key].turns.push({ turn: r.caller.match(/#turn(\d+)/)?.[1] || 's', tokens: r.total_tokens })
    groups[key].total_tokens += r.total_tokens
  }
  return Object.values(groups)
}
```

**效果**：用户不再误以为"切换页面导致重复执行"，呼应上一个对话的 #turnN 优化。
