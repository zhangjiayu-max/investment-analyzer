## 二、修复方案

### F-1：基准获取修复（沪深300本地化）

**问题**：`opportunity_engine.py:1234` `_get_benchmark_return` 直接 `ak.index_zh_a_hist`，无超时、无本地兜底。本地 `index_price_history` 表无沪深300数据（只有 10 个行业指数）。

**方案**：
1. **启动时回填沪深300历史**：在 `app.py` 启动回填指数历史处，新增沪深300（000300）到回填列表
2. **`_get_benchmark_return` 改本地优先 + akshare 兜底**：
   - 优先从 `index_price_history` 查 000300 的 entry_date 和 review_date 收盘价
   - 本地查不到时，用 `call_akshare_with_timeout` 包装 akshare 调用（超时 20s）
   - 仍查不到返回 None（回退绝对涨幅逻辑）

**改动文件**：
- `backend/services/advisor/opportunity_engine.py`：改造 `_get_benchmark_return`
- `backend/app.py`：启动回填列表加 000300（或复用现有 `index_history_fetcher`）

**关键代码**（`_get_benchmark_return` 改造）：
```python
def _get_benchmark_return(entry_date: str, review_date: str) -> float | None:
    """L3 回测基准化：获取沪深300涨幅（本地优先 + akshare 兜底）。"""
    if not entry_date or not review_date:
        return None
    # 1. 本地 index_price_history 优先
    try:
        from db._conn import _get_conn
        conn = _get_conn()
        try:
            entry_row = conn.execute(
                "SELECT close FROM index_price_history WHERE index_code='000300' "
                "AND trade_date <= ? ORDER BY trade_date DESC LIMIT 1",
                (entry_date,)
            ).fetchone()
            review_row = conn.execute(
                "SELECT close FROM index_price_history WHERE index_code='000300' "
                "AND trade_date <= ? ORDER BY trade_date DESC LIMIT 1",
                (review_date,)
            ).fetchone()
        finally:
            conn.close()
        if entry_row and review_row and entry_row[0] and review_row[0] and entry_row[0] > 0:
            return (review_row[0] - entry_row[0]) / entry_row[0] * 100
    except Exception as e:
        logger.debug(f"[opportunity] 本地基准查询失败: {e}")
    # 2. akshare 兜底（带超时保护）
    try:
        import akshare as ak
        from services.market.leading_indicators.akshare_utils import call_akshare_with_timeout
        df = call_akshare_with_timeout(
            ak.index_zh_a_hist, symbol="000300", period="daily",
            start_date=entry_date.replace("-", ""),
            end_date=review_date.replace("-", ""),
            timeout=20,
        )
        if df is None or df.empty or len(df) < 2:
            return None
        first_close = float(df.iloc[0]["收盘"])
        last_close = float(df.iloc[-1]["收盘"])
        if first_close <= 0:
            return None
        return (last_close - first_close) / first_close * 100
    except Exception as e:
        logger.debug(f"[opportunity] akshare 基准获取失败: {e}")
        return None
```

### F-3：watchlist 估值高位否决

**问题**：`watchlist.py:287` `_compute_signal_status` 用 adaptive_target 判定，target 被自适应调高后 current 95% 仍判 green。与机会雷达 P0-A「估值>80% 强制 avoid」矛盾。

**方案**：在 `_compute_signal_status` 返回后、插入回测前，加一道估值高位否决：
- 若 `current_pct > 80`：强制降为 `red`，reason 标注"估值高位否决（{current_pct}% > 80%阈值）"
- 若 `current_pct` 为 None 但 `nav_percentile > 80`：同样否决（净值高位）
- 开关 `watchlist.high_valuation_veto_enabled`（默认 true）

**改动文件**：`backend/routers/portfolio/watchlist.py`

**改动文件**：`backend/services/advisor/opportunity_engine.py` 新增 `backfill_miss_reason()`

**miss_reason 拼接规则**：
```python
def _build_miss_reason(change_pct, benchmark_pct, excess_return, entry_percentile):
    """拼接 miss 原因，用于反哺分析。"""
    reasons = []
    if excess_return is not None:
        reasons.append(f"超额收益{excess_return:+.1f}%（未达+2%阈值）")
    else:
        reasons.append(f"绝对涨幅{change_pct:+.1f}%（未达+3%阈值）")
    if benchmark_pct is not None:
        reasons.append(f"沪深300同期{benchmark_pct:+.1f}%")
    if entry_percentile is not None:
        if entry_percentile > 60:
            reasons.append(f"入场估值偏高({entry_percentile:.0f}%)")
        elif entry_percentile < 30:
            reasons.append(f"入场估值偏低({entry_percentile:.0f}%)")
    return "；".join(reasons)
```

## 四、闭环机制图

```
信号生成（10 维评分，含领先指标 F-2）
    ↓
机会卡落库 + backtest 记录写入（entry_price + signal_source）
    ↓
每日 09:30 回测到期记录（基准化 F-1 修复）
    ↓
hit/miss 判定 + miss_reason 拼接（F-4 回填）
    ↓
_apply_hit_rate_feedback() 连续 3 miss → 降权 20%
    ↓
下次评分用新权重 → 低命中率信号源自动降权
    ↓
watchlist 信号生成（F-3 高位否决）→ 不再高位误导
```

**闭环要点**：
- 信号源维度（F-2）→ 回测分组（signal_source）→ 降权反哺（_apply_hit_rate_feedback）→ 下次评分降权 → 闭环
- 基准准确（F-1）→ hit/miss 判定准确 → 降权决策准确 → 闭环质量提升
- miss_reason（F-4）→ 人工分析 miss 原因 → 调整评分维度权重 → 持续优化
