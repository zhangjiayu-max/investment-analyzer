# 基金代码幻觉全面修复设计稿

## 日期：2026-07-22

## 一、问题现象

conv#133 用户问"博时恒乐债券之前中仓了锂，怎么现在追科技啦呀"，基金分析师：
1. 调用 `ttfund_fund_condition("博时恒乐债券")` → `{"error": "无数据"}`
2. 调用 `query_portfolio({"query_type": "detail"})` → 返回 22 只持仓（**包含博时恒乐债券A 014846 / C 014847**）
3. 收口时调用 `query_fund_info("007679")` → **幻觉了错误代码**（007679 是泰达宏利品牌升级混合C）
4. 还调了 `query_fund_info("016181")` 和 `query_fund_info("016182")`（华安添祥、华安安进）— 同样是幻觉代码
5. answer 为空，用工具结果兜底，最终输出 `{"error": "无数据"}`

**真实代码**：014846（A类）、014847（C类），在持仓列表第 14/15 位。

## 二、根因分析

### 2.1 LLM 幻觉基金代码

基金分析师 prompt 要求"必须调用 query_fund_info"，但：
- prompt **没有要求"从 query_portfolio 结果中提取基金名称对应的 fund_code"**
- `query_fund_info` 工具**只接受 fund_code，不支持基金名称查询**
- LLM 拿到 22 只持仓的完整列表后，没有逐条匹配"博时恒乐债券"，而是凭记忆编造了代码

### 2.2 工具无名称匹配能力

`query_fund_info` 的 `fund_code` 参数 description 只写了"基金代码，如 '161725'"，没有提供"如果只知道名称怎么办"的指引。`lookup_fund_info` 也只接受 fund_code，无名称查找能力。

### 2.3 收口逻辑无代码校验

收口时（answer 为空或 <200 字）：
- 单次收口调用 LLM，但 LLM 可能再次幻觉代码
- 兜底 `_fallback_from_tool_results` 只是拼接工具结果，不检查工具是否查了正确的基金
- 没有检测"用户问基金A，但工具查了基金B"的矛盾

### 2.4 数据源局限（非本设计稿范围）

即使代码正确，akshare 对 014846 的股票/债券持仓解析失败（`Can not decode value starting with character ';'`），只有行业配置成功。这是 akshare 数据源问题，不在本设计稿修复范围，但需要在分析中如实告知用户。

## 三、修复方案（四层防御）

### 3.1 L1 工具层：query_fund_info 支持基金名称模糊匹配

**文件**：`tools/__init__.py` L2526-2569

**修改**：`_query_fund_info` 函数增加名称匹配能力。当 `fund_code` 不是 6 位数字时，从用户持仓列表中模糊匹配基金名称，返回正确代码。

```python
def _query_fund_info(args: dict) -> str:
    """查询基金详细信息。支持基金代码或名称（自动从持仓匹配）。"""
    fund_code = args.get("fund_code", "").strip()
    fund_name_hint = args.get("fund_name", "").strip()  # 新增：可选名称提示
    detail_type = args.get("detail_type", "all")

    if not fund_code and not fund_name_hint:
        return json.dumps({"error": "请提供基金代码或基金名称"}, ensure_ascii=False)

    # L1 防幻觉：如果 fund_code 不是纯数字，或提供了 fund_name，尝试从持仓列表匹配
    _match_note = ""
    if fund_code and not fund_code.isdigit():
        # fund_code 实际是名称，做名称匹配
        fund_name_hint = fund_code
        fund_code = ""
    if fund_name_hint and not fund_code:
        matched_code = _match_fund_code_by_name(fund_name_hint)
        if matched_code:
            _match_note = f"通过名称'{fund_name_hint}'匹配到基金代码 {matched_code}"
            fund_code = matched_code
        else:
            return json.dumps({
                "error": f"未在持仓中找到名称包含'{fund_name_hint}'的基金",
                "hint": "请先调用 query_portfolio 查看持仓列表，使用其中的 fund_code 调用本工具",
            }, ensure_ascii=False)

    if not fund_code:
        return json.dumps({"error": "请提供基金代码"}, ensure_ascii=False)

    result = {"fund_code": fund_code}
    if _match_note:
        result["_match_note"] = _match_note
    # ... 后续逻辑不变
```

**新增辅助函数** `_match_fund_code_by_name`：

```python
def _match_fund_code_by_name(name_hint: str) -> str | None:
    """从用户持仓列表中模糊匹配基金名称，返回 fund_code。
    
    匹配策略：
    1. 完全包含匹配（持仓 fund_name 包含 name_hint）
    2. 去空格后包含匹配
    3. 关键词匹配（取名称中>=2字的核心词）
    返回第一个匹配项的 fund_code，无匹配返回 None。
    """
    try:
        holdings = list_holdings()
    except Exception:
        return None
    name_hint = name_hint.strip()
    # 1. 完全包含
    for h in holdings:
        fn = h.get("fund_name") or ""
        if name_hint in fn:
            return h.get("fund_code")
    # 2. 去空格包含
    name_clean = name_hint.replace(" ", "")
    for h in holdings:
        fn = (h.get("fund_name") or "").replace(" ", "")
        if name_clean in fn:
            return h.get("fund_code")
    # 3. 关键词匹配（取>=3字的子串）
    if len(name_hint) >= 3:
        for h in holdings:
            fn = h.get("fund_name") or ""
            # 取 hint 的前3字做匹配
            if name_hint[:3] in fn:
                return h.get("fund_code")
    return None
```

## 基金代码获取规范（防幻觉强制要求）
**禁止凭记忆猜测基金代码**，基金代码必须来自以下可信源之一：
1. **用户持仓列表**：调用 query_portfolio(query_type="detail")，从返回的 holdings 数组中，根据 fund_name 字段匹配用户提到的基金名称，取对应的 fund_code

- 禁止使用未在 query_portfolio 结果中出现的基金代码
- 禁止根据基金名称"猜测"代码（如"博时"开头不一定是 016xxx）

**新增函数** `_detect_fund_code_hallucination`：

```python
def _detect_fund_code_hallucination(tool_calls_log: list, agent: dict) -> str:
    """检测基金分析师是否幻觉了基金代码。
    
    检查 query_fund_info 调用的 fund_code 是否在用户持仓中存在。
    如果不存在，返回纠正提示；否则返回空字符串。
    """
    if agent.get("agent_key") != "fund_analyst":
        return ""
    # 收集所有 query_fund_info 调用的 fund_code
    queried_codes = []
    for tc in tool_calls_log:
        if tc.get("name") == "query_fund_info":
            try:
                args = json.loads(tc.get("arguments", "{}"))
                code = args.get("fund_code", "")
                if code and code.isdigit():
                    queried_codes.append(code)
            except Exception:
                pass
    if not queried_codes:
        return ""
    # 检查这些代码是否在持仓中
    try:
        holdings = list_holdings()
        holding_codes = {h.get("fund_code") for h in holdings}
    except Exception:
        return ""
    wrong_codes = [c for c in queried_codes if c not in holding_codes]
    if not wrong_codes:
        return ""
    # 返回纠正提示
    # 同时提供正确的代码建议
    code_suggestions = []
    for wc in wrong_codes:
        # 尝试从工具结果中找到查到的基金名称，帮 LLM 对应到正确代码
        for tc in tool_calls_log:
            if tc.get("name") == "query_fund_info":
                try:
                    args = json.loads(tc.get("arguments", "{}"))
                    if args.get("fund_code") == wc:
                        result = json.loads(tc.get("result", "{}"))
                        wrong_name = result.get("basic_info", {}).get("fund_name", "")
                        if wrong_name:
                            code_suggestions.append(f"代码 {wc} 实际是'{wrong_name}'，不是您要查的基金")
                        break
                except Exception:
                    pass
    return (
        f"⚠️ 检测到基金代码错误：您查询的代码 {wrong_codes} 不在用户持仓列表中。"
        f"{'；'.join(code_suggestions) if code_suggestions else ''}。"
        f"请重新调用 query_portfolio(query_type='detail') 获取持仓列表，"
        f"从中找到用户提到的基金名称对应的正确 fund_code，再调用 query_fund_info。"
        f"禁止使用持仓列表中不存在的基金代码。"
    )
```
