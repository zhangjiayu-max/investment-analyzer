## 一、问题诊断

### 1.1 蒸馏质量两极分化（已用数据库验证）

| 书籍 | 总条数 | <200字比例 | 平均字数 | 问题定性 |
|------|--------|-----------|---------|---------|
| **资产配置的艺术** | 196 | **98%** | **109字** | ❌ 源文本似目录，提取成摘要 |
| **共同基金常识** | 167 | **42%** | 201字 | ⚠️ 碎片严重，最短30字 |
| **聪明的投资者** | 51 | **37%** | 214字 | ⚠️ 内容空洞，像读书笔记 |
| 债券投资实战 | 176 | 23% | 243字 | ⚠️ 标题模糊，碎片化 |
| 漫步华尔街 | 61 | 25% | 257字 | ⚠️ 部分内容过短 |
| 黄金投资新时代 | 40 | **0%** | **339字** | ✅ 质量好，有公式有数据 |
| 投资最重要的事 | 63 | **0%** | **357字** | ✅ 质量好 |
| 周期 | 68 | **0%** | **374字** | ✅ 质量好 |
| 手把手教你读财报 | 212 | **0%** | **402字** | ✅ 质量好 |

**根因**：
1. **单 prompt 适配所有书籍**：`extract_knowledge` 的 prompt 要求"具体步骤、交易规则、量化公式"，对《彼得·林奇》等实战书有效，但对《聪明的投资者》《资产配置的艺术》等**理念型/框架型**书籍完全不适用。LLM 面对格雷厄姆的投资哲学只能勉强提取空泛概念。
2. **chunk 切分过度**：`max_chars=3000`，经典著作章节很长，切分后 LLM 只见局部不见整体，只能提取碎片。
3. **审核过于激进**：要求"删除率30-50%、重要性<7删除"，把理念型书籍中重要但不够"实战"的内容（如安全边际的数学推导）误删。
4. **源文本质量问题**：《资产配置的艺术》98%低于200字，大概率 OCR/源文本本身就是目录级别。
5. **LLM 调用硬编码**：`distill_period.py` 写死了 MIMO 套餐 key，不兼容其他模型，无 fallback。

### 1.2 RAG 检索链路的瓶颈

1. **同义词词典和知识图谱空白**："黄金"未关联"实际利率/美元/避险"，"美联储"未关联"加息/降息/黄金/美股"。
2. **m3e-base 对英文金融术语弱**："S&P 500"、"Treasury Yield"、"Federal Reserve" 向量编码不准确。
3. **低质量内容淹没高质量内容**：《资产配置的艺术》196条占 book 总量的 11%，但几乎都是噪声。

## 二、优化方案

### 2.1 蒸馏脚本优化（`distill_period.py`）

#### 改动 1：LLM 调用兼容化（高优先级）

- 不再硬编码 API key/base_url/model
- 统一使用 `config.get_llm_config()` / `get_llm_fallback_config()`
- 支持 `--model` 参数覆盖，方便用 DeepSeek 等模型做蒸馏
- 调用失败自动 fallback

#### 改动 2：书籍类型感知蒸馏（核心优化）

按书籍类型使用不同的提取策略和 prompt：

| 类型 | 代表书籍 | 提取重点 | 内容长度要求 |
|------|---------|---------|-------------|
| **实战操作型** | 彼得·林奇、手把手教你读财报 | 具体步骤、量化规则、选股标准、案例 | 300-600字 |
| **理念框架型** | 聪明的投资者、投资最重要的事 | 核心原则、数学推导、历史验证、反直觉结论 | 300-500字 |
| **资产配置型** | 资产配置的艺术、漫步华尔街 | 配置比例、资产特性、再平衡规则、长期数据 | 300-500字 |
| **市场周期型** | 周期 | 周期指标、阶段判断、应对策略、历史案例 | 300-500字 |
| **行为心理型** | 思考，快与慢 | 认知偏差、决策陷阱、实证数据、实验案例 | 250-400字 |
| **基金科普型** | 共同基金常识、指数基金投资指南 | 费率对比、历史收益数据、产品结构 | 250-400字 |

**理念框架型 prompt 的关键差异**：
- 不要求"具体买卖步骤"，而要求"核心原则 + 历史验证 + 适用范围"
- 允许抽象概念，但必须附带**具体数据或案例支撑**
- 示例："安全边际"必须包含"价格低于内在价值的幅度、历史回测数据、失败案例"

#### 改动 3：chunk 策略优化

- 实战操作型：`max_chars=3000`（保持现状）
- 理念框架型/资产配置型：`max_chars=6000-8000`（保留章节完整性）
- 增加"章节完整性保护"：如果检测到 `# 第X章`，不跨章节切分

#### 改动 4：审核策略按类型调整

- 实战操作型：删除率 30-50%，重要性<7删除（保持现状）
- 理念框架型：删除率 20-40%，重要性<6删除（保留更多原则性内容）
- 资产配置型：删除率 20-40%，但强制要求包含"具体配置比例或历史数据"

#### 改动 5：源文本质量预检

在提取前扫描 Markdown：
- 如果一段内容全是标题/列表项（无段落文字），标记为"疑似目录"
- 如果平均段落长度 <50字，提示"源文本可能过于碎片化"

#### 改动 6：提取阶段严格过滤

当前只在审核阶段过滤 <200 字，改到**提取阶段就过滤 <250 字**，减少无效 token 消耗。

### 2.2 RAG 检索链路优化

#### 改动 1：扩展同义词词典

```python
# SYNONYMS 新增跨资产术语
"黄金": ["gold", "XAU", "贵金属", "避险资产", "抗通胀", "实际利率"],
"美联储": ["Fed", "FOMC", "鲍威尔", "加息", "降息", "联邦基金利率"],
"美股": ["S&P 500", "纳斯达克", "NYSE", "美国股票", "标普500", "道琼斯"],
"美债": ["美国国债", "Treasury", "收益率曲线", "期限利差"],
"实际利率": ["real interest rate", "名义利率-通胀", "TIPS收益率"],
"美元": ["USD", "美元指数", "DXY", "汇率"],
"通胀": ["CPI", "PPI", "物价", "通货膨胀", "deflation"],
```

#### 改动 2：扩展知识图谱

```python
"黄金": {
    "related": ["实际利率", "美元", "通胀", "CPI", "避险", "央行购金", "美元指数"],
    "parent": "大宗商品",
},
"美联储": {
    "related": ["加息", "降息", "美元", "美债收益率", "美股", "黄金", "北向资金", "A股外资"],
    "parent": "宏观政策",
},
"美债收益率": {
    "related": ["收益率曲线", "倒挂", "美联储", "美元", "全球流动性", "中美利差", "A股估值"],
    "parent": "债券市场",
},
```

#### 改动 3：RAG 参数调优

- `SCORE_THRESHOLD` 从 0.012 提高到 **0.018**，过滤更多噪声
- 同书多样性衰减：超过 **2 条**就开始衰减（当前是 3 条）
- 增加"内容长度加权"：对 <200 字的结果在 RRF 排序中额外 ×0.9 降权

#### 改动 4：查询改写增强

增加跨资产查询示例：
```
"黄金还能买吗" → "黄金 价格 投资策略 实际利率"
"Fed 降息对 A 股有什么影响" → "美联储 降息 中国 A股 影响"
"美债收益率倒挂意味着什么" → "美债 收益率曲线 倒挂 经济衰退"
```
