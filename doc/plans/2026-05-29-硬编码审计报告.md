# 硬编码审计报告

## Context
本项目是理财投资分析系统，涉及用户资金决策。根据项目规范「金融逻辑禁止硬编码」，需全面排查代码中所有硬编码的配置值、金融阈值、LLM 提示词等，确保数据有来源、逻辑可追溯。

## 审计范围
覆盖 `backend/` 目录下所有 Python 文件，重点检查：
- 金融相关阈值（估值百分位、仓位比例、温度计等）
- LLM 系统提示词中的金融逻辑
- Agent 配置中的硬编码规则
- 技术参数（token 限制、重试策略等）

## 一、金融阈值硬编码（最高优先级）

### 1.1 估值百分位分界线 — 多处冲突

同一套阈值在 4+ 个位置独立定义，且存在不一致：

| 位置 | 低估线 | 高估线 | 说明 |
|------|--------|--------|------|
| `config.py:131` | 30% | 70% | 可通过 env 配置 |
| `portfolio_context.py:189-198` | 20%/30% | 70%/80% | **硬编码，不读 config** |
| `db.py:1245-1249` 估值分析师 prompt | 20% | 80% | prompt 内硬编码 |
| `db.py:2918-2919` 市场日报 prompt | 20% | 70% | prompt 内硬编码 |
| `valuation.py:135,137` | 30% | 70% | 代码内硬编码 |
| `llm_service.py:360-362` 编排 prompt | 30% | 70% | prompt 内硬编码 |
| `image_parser.py:12` DD 解析 prompt | 30% | 70% | prompt 内硬编码 |
| `tools/__init__.py:111` 工具描述 | 30% | 70% | 工具描述内硬编码 |

**风险**：`portfolio_context.py` 的分界线(20/80)与 `config.py`(30/70) 不一致，用户修改配置后不会生效。

### 1.2 PE/PB 估值比较阈值

**文件**: `valuation.py:118-151`
- `ratio < 0.8` → 相对低估
- `ratio > 1.2` → 相对高估
- `ratio < 1.0` → PE/PB 合理
- 无来源依据，全硬编码

### 1.3 股票评分体系

**文件**: `valuation.py:329-393` `_score_stock()`
- PE 阈值: 15/25/50（分 4 档，各档加减分）
- PB 阈值: 1.0/2.0/5.0
- 52 周高点偏离: -30%/-5%
- 评分→建议映射: 30/45/60/75
- **无任何来源依据**

### 1.4 基金评分体系

**文件**: `valuation.py:399-436` `_score_fund()`
- 净值偏离高点: -20%/-10%/-3%
- 百分位阈值: 30%/70%
- **无来源依据**

### 1.5 集中度分析阈值

**文件**: `db.py:2970-2980` 分散度分析师 prompt
- 单基金集中度: >25% 高度集中, >15% 适度集中
- 行业集中度: >35% 过高, >20% 偏高
- 相关性: ≥0.85 强同向, 0.7-0.85 中等
- 重仓定义: >10% 仓位
- **全部硬编码在 prompt 字符串中**

### 1.6 全景诊断评分规则

**文件**: `db.py:3050-3079` 全景诊断分析师 prompt
- 前 3 大持仓: <50% 优, 50-65% 中, >65% 差
- 单基金占比: >20% 扣 2 分, >30% 扣 4 分
- 同类型占比: >80% 扣 2 分
- PE 百分位: <30% 优, >60% 差
- 相关对: >3 对扣 2 分
- 总分等级: 85+/70-84/55-69/40-54/<40
- **无来源依据**

### 1.7 债券配置温度阈值

**文件**: `db.py:3444-3454` + `app.py:5531-5577`
- 温度 <30° 冷 → 配 60-80%
- 温度 30-50° 偏冷 → 配 50-60%
- 温度 50-70° 适中 → 配 30-50%
- 温度 >70° 热 → 配 15-30%
- 单基金不超过零钱 30%
- 推荐 2-4 只基金
- **两处独立定义，可能不一致**

### 1.8 4% 定投法

**文件**: `orchestrator.py:586-592` + `multi_agent.py:109-113,178-183` + `db.py:4966-4978`
- 下跌 4% 买 1 份, 8% 买 2 份, 12% 买 3 份
- 单品种仓位 20-30%
- 深度低估(<10%) 份数翻倍
- 保留 20-30% 现金
- **3 处重复定义，无外部来源**

### 1.9 风险评估阈值

**文件**: `multi_agent.py:82` 风险评估师 prompt
- CV <0.15 低风险, 0.15-0.30 中等, >0.30 高风险
- z-score >2 极度高估, <-2 极度低估
- **无来源依据**

### 1.10 基金深度分析

**文件**: `db.py:3174-3182`
- 买入时百分位 <30% → 好, 30-60% → 中, >60% → 差
- 卖出时百分位 >70% → 好, <30% → 差
- **无来源依据**

### 1.11 热点分析

**文件**: `db.py:3385,3400`
- 高置信度要求: 百分位<10% + 政策利好
- 单一资产集中警报: >20%
- **无来源依据**

### 1.12 调仓建议金额比例

**文件**: `rebalancer.py:186,194,219,228`
- 现金调拨: 50%-100% 的差额
- 买卖建议: 30%-70% 的差额
- **无来源依据**

### 1.13 年龄法则公式

- 权益最低 10%
- 债券占非权益 75%, 货币 8%, QDII 17%

### 1.14 推荐验证

## 六、改进建议

### 立即修复（一致性风险）
1. `portfolio_context.py` 的百分位分界线应读取 `config.py` 配置
2. 统一所有位置的百分位分界线为同一套值
3. 温度阈值统一（db prompt vs app.py 代码）

### 中期改进（金融逻辑外部化）
1. 将 14 个 Agent prompt 中的金融阈值提取为配置变量
2. 评分体系（stock/fund scoring）的权重和阈值应可配置
3. 集中度阈值、相关性阈值应存入配置表
4. 4% 定投法参数应统一来源

### 长期改进（系统参数外部化）
1. LLM temperature/max_tokens 存入配置表
2. RAG 参数（chunk_size, overlap, freshness）存入配置表
3. 内容截断限制统一管理
