# 2026-07-06 trace_id 日志完善与刷新卡顿修复

## 一、问题诊断（基于真实数据）

### 1.1 trace_id 日志覆盖现状

| 文件 | logger 总数 | 含 trace_id | 覆盖率 |
|------|-----------|-----------|-------|
| agent/orchestrator.py | 100 | 0 | 0% |
| agent/multi_agent.py | 14 | 0 | 0% |
| routers/conversations.py | 55 | 6 | 11% |
| services/llm_service.py | 12 | 1 | 8% |
| **合计** | **181** | **7** | **3.9%** |

核心矛盾：DB 表（token_usage / agent_runs / tool_audit_logs / rag_logs / execution_traces）trace_id 完整，3 个 trace 查询 API 也已就位（GET /api/conversations/{conv_id}/trace/{trace_id}、GET /api/token-usage/trace/{trace_id}），但日志文件里只有 3.9% 带 trace_id。出问题看日志时无法用 trace_id 关联。

### 1.2 刷新页面卡顿根因

根因在后端：app.py 的 `/api/finance/quote-bar` 端点把同步的 `ak.stock_hot_keyword_em()` 直接放在 `async def` 中，阻塞 FastAPI 事件循环 1-5 秒。

刷新时序：
1. Home.vue onMounted 立即发 GET /api/finance/quote-bar
2. 后端进入 ak.stock_hot_keyword_em() 同步阻塞 1-5 秒
3. 事件循环被占，期间所有请求排队（ChatView 的 GET /api/conversations、消息加载、agents 列表、resume SSE 等）
4. akshare 返回后积压请求一次性涌出，前端串行 await 链开始解锁，体感"卡住"

前端无同步阻塞，对话分析在 daemon 线程中跑也不阻塞主线程。

## 二、设计目标

| 目标 | 度量 |
|------|------|
| 关键路径日志带 trace_id | 异常/兜底/专家开始结束/仲裁/持久化 ≥30 处补齐 |
| trace_id 可串联日志与 DB | 同一 trace_id 能在日志文件和 5 张表中都查到 |
| 刷新页面不卡顿 | quote-bar 响应 <500ms（命中缓存 <50ms），不阻塞事件循环 |
| akshare 失败不影响主流程 | 超时 3s 自动降级返回空 hot_keywords |

### P2：akshare 异步化 + 缓存 + 超时降级

#### P2-1：/api/finance/quote-bar 异步化

策略：用 `asyncio.to_thread` 包装 akshare 同步调用 + 30s 内存缓存 + 3s 超时降级。

修改 app.py L1300-1350 的 `finance_quote_bar()`：

```python
import asyncio
from functools import lru_cache
from datetime import datetime, timedelta

_quote_bar_cache = {"data": None, "expires_at": datetime.min}

@app.get("/api/finance/quote-bar")
async def finance_quote_bar():
    # 1. 命中缓存直接返回
    now = datetime.now()
    if _quote_bar_cache["data"] and _quote_bar_cache["expires_at"] > now:
        return _quote_bar_cache["data"]

    # 2. 异步获取 hot_keywords（不阻塞事件循环）
    hot_keywords = []
    try:
        import akshare as ak
        # 用 asyncio.to_thread 包装同步调用 + 3s 超时
        df = await asyncio.wait_for(
            asyncio.to_thread(ak.stock_hot_keyword_em),
            timeout=3.0,
        )
        if df is not None and len(df) > 0:
            hot_keywords = df["概念名称"].tolist()[:5]
    except asyncio.TimeoutError:
        logger.warning("[quote-bar] akshare 超时 3s，降级返回空 hot_keywords")
    except Exception as e:
        logger.warning(f"[quote-bar] akshare 失败: {e}")

    # 3. 构建响应（其余逻辑不变）
    result = {...}
    # 4. 写缓存（30s TTL）
    _quote_bar_cache["data"] = result
    _quote_bar_cache["expires_at"] = now + timedelta(seconds=30)
    return result
```
