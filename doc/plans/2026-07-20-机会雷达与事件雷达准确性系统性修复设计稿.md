### 2.2 P0-B：修复 entry_price 获取（sina→本地表）

**文件**：`backend/services/advisor/opportunity_engine.py:497-510`

**当前**：用 `ak.stock_zh_index_daily(symbol=index_code)` 调用 sina API → 404

**修复后**：先查本地 `valuations` 表的最新 close，本地无再降级 akshare `index_zh_a_hist`，最后降级返回 None：

```python
def _get_theme_index_current_price(theme_rule: dict) -> float | None:
    index_code = _get_theme_index_code(theme_rule)
    if not index_code:
        return None
    # 1. 优先查本地 valuations 表的最新 close
    try:
        from db.valuations import get_latest_valuation
        v = get_latest_valuation(index_code)
        if v and v.get("close"):
            return float(v["close"])
    except Exception as e:
        logger.debug(f"[opportunity] 本地估值表查询失败: {e}")
    # 2. 降级 akshare index_zh_a_hist
    try:
        import akshare as ak
        df = ak.index_zh_a_hist(symbol=index_code, period="daily", count=1)
        if df is not None and len(df) > 0:
            return float(df['close'].values[-1])
    except Exception as e:
        logger.debug(f"[opportunity] akshare 获取指数价格失败: {e}")
    return None
```

### 2.4 P0-D：事件 radar sources 按主题过滤

**文件**：`backend/services/market/event_radar.py`

**当前问题**：抓取后所有事件塞同一批 sources（"长鑫科技中签"、"陆家嘴财经早餐"、"优秀的理财经理"）

**修复方案**：事件创建时，按 `affected_sectors + affected_themes + title 关键词` 过滤新闻列表，每个事件只保留 3 条真正相关的 sources：

```python
def _filter_sources_by_relevance(event: dict, all_news: list[dict]) -> list[dict]:
    """按事件主题过滤相关新闻。"""
    sectors = event.get("affected_sectors", [])
    themes = event.get("affected_themes", [])
    title = event.get("title", "")
    keywords = set(sectors + themes + _extract_title_keywords(title))
    
    scored = []
    for news in all_news:
        text = f"{news.get('title','')} {news.get('summary','')}"
        score = sum(1 for kw in keywords if kw in text)
        if score > 0:
            scored.append((score, news))
    scored.sort(key=lambda x: -x[0])
    return [n for _, n in scored[:3]]
```

### 2.5 P0-E：matched_holdings 用 index_code 精确匹配

**文件**：`backend/services/market/event_radar.py`

**当前问题**：用名称包含匹配，"机器人"匹配到"人工智能"，"科技"匹配到医药

**修复方案**：优先用持仓的 `index_code` 与事件 `affected_index_codes` 精确匹配，名称匹配仅作为兜底且要求多关键词命中：

```python
def _match_holdings(event: dict, holdings: list) -> list:
    affected_sectors = event.get("affected_sectors", [])
    affected_themes = event.get("affected_themes", [])
    # 建立 sector→index_code 精确映射
    sector_index_map = {
        "人工智能": ["931638"], "半导体": ["931638"], "新能源": ["931140"],
        "机器人": ["H30590"], "红利": ["000922"], "白酒": ["399997"],
    }
    expected_codes = []
    for s in affected_sectors + affected_themes:
        expected_codes.extend(sector_index_map.get(s, []))
    
    matched = []
    for h in holdings:
        # 1. 优先 index_code 精确匹配
        if h.get("index_code") and h["index_code"] in expected_codes:
            matched.append({...})
            continue
        # 2. 名称匹配要求 ≥2 个关键词命中（避免单"科技"误匹配）
        # 比如事件 affected_themes 含 ["AI上市","产业融资"]，与"机器人"名称只匹配 0 个，不命中
    return matched
```

### 2.6 P1-A：summary / risk_note 动态生成

**文件**：`backend/services/advisor/opportunity_engine.py:442-494` (`_build_item`)

**当前**：summary 全部是「{theme}出现热点线索，当前结论为可小仓试投」

**修复后**：基于 evidence（新闻+估值+技术面）动态拼接：

```python
def _build_summary(theme: str, news_hits: list, valuation: dict | None, 
                   verdict: str, score: int) -> str:
    parts = []
    # 1. 新闻维度
    if news_hits:
        top_news = news_hits[0].get("title", "")[:30]
        parts.append(f"新闻线索：{top_news}")
    # 2. 估值维度
    if valuation:
        pct = valuation.get("percentile")
        if pct is not None:
            if pct > 80:
                parts.append(f"估值偏高（{pct}%分位）")
            elif pct < 30:
                parts.append(f"估值偏低（{pct}%分位）")
            else:
                parts.append(f"估值合理（{pct}%分位）")
    # 3. verdict 总结
    verdict_text = {
        "can_buy": "综合多维信号，可小仓试投",
        "watch": "信号分歧，建议观察",
        "avoid": "风险较高，不建议追",
    }.get(verdict, "")
    if verdict_text:
        parts.append(verdict_text)
    return "；".join(parts) if parts else f"{theme}暂无明显信号"
```

risk_note 同理动态拼接：估值过高→"估值已处历史高位，追高风险大"；技术看空→"技术指标偏空"；场外基金→"场外基金不适合少于7天交易"。

### 2.7 P1-B：entry_plan amount 与估值挂钩

**文件**：`backend/services/advisor/opportunity_engine.py:476-481`

**当前**：所有 can_buy 的 amount 都是 portfolio_fit.suggested_budget（现金*10% 或 总资产*1%）

**修复后**：基于估值分位动态调整 amount：

```python
def _calc_entry_amount(verdict: str, valuation: dict | None, 
                      base_budget: float) -> float:
    if verdict != "can_buy":
        return 0
    if not valuation or valuation.get("percentile") is None:
        return base_budget * 0.5  # 无估值数据减半
    pct = valuation["percentile"]
    if pct < 20:
        return base_budget * 1.5  # 深度低估加仓
    elif pct < 40:
        return base_budget * 1.0  # 合理偏低
    elif pct < 60:
        return base_budget * 0.7  # 合理
    elif pct < 80:
        return base_budget * 0.4  # 偏高减仓
    else:
        return 0  # 高估不应入场（理论上 verdict 已 avoid）
```
