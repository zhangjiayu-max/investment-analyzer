# 估值百分位优化设计

> 参考文章：《估值百分位怎么用？这4个风险要注意》（银行螺丝钉）
> 日期：2026-06-04

## 背景

螺丝钉文章指出估值百分位的 4 个使用风险，我们的系统目前只展示原始百分位数字，缺乏风险提示和使用指导，可能导致用户误判。

## 优化目标

让用户看到百分位时，同时看到**风险提示**和**使用建议**，避免盲目跟数字投资。

---

## 一、PE/PB 背离预警（风险4：盈利波动失效）

### 问题
当盈利大幅波动时，PE 百分位会失真：
- 盈利大涨 → PE 被动降低 → 百分位看起来很低（假低估）
- 盈利大跌 → PE 被动升高 → 百分位看起来很高（假高估）

文章案例：中证1000 PE百分位50%，PB百分位20%，差异30%，说明PE已失效。

### 方案

在 `enhanced-strategy` 接口返回数据中新增 `valuation_warnings` 字段：

```json
{
  "index_name": "中证1000",
  "valuation_warnings": [
    {
      "type": "pe_pb_diverge",
      "level": "warning",
      "message": "PE百分位(50%)与PB百分位(20%)差异较大，盈利波动可能导致PE失真，建议参考PB",
      "pe_percentile": 50,
      "pb_percentile": 20,
      "diverge_pct": 30
    }
  ],
  "recommended_metric": "PB"
}
```

### 判断规则
| 条件 | 级别 | 提示 |
|------|------|------|
| \|PE百分位 - PB百分位\| > 30% | 🔴 强警告 | "PE/PB严重背离，强烈建议参考{更稳定指标}" |
| \|PE百分位 - PB百分位\| > 20% | 🟡 警告 | "PE/PB差异较大，建议参考{更稳定指标}" |
| \|PE百分位 - PB百分位\| > 10% | 🔵 提示 | "PE/PB存在差异，可结合参考" |

**选择更稳定指标的逻辑**：
- 如果 PB 百分位 < PE 百分位 → PB 更稳定（盈利可能虚高），推荐 PB
- 如果 PE 百分位 < PB 百分位 → PE 更稳定（盈利可能虚低），推荐 PE

## 二、数据覆盖年限标注（风险1：历史太短）

### 问题
刚成立2-3年的指数，即使百分位很低，也不一定是真正低估，只是数据不够。

### 方案

在估值数据中新增 `history_years` 字段：

```json
{
  "index_name": "科创50",
  "history_years": 4,
  "history_warning": {
    "level": "info",
    "message": "该指数历史数据仅4年，未覆盖完整牛熊周期，百分位参考价值有限"
  }
}
```

### 判断规则
| 覆盖年限 | 级别 | 提示 |
|----------|------|------|
| < 3年 | 🔴 警告 | "数据不足3年，百分位参考价值很低" |
| 3-5年 | 🟡 提示 | "数据仅覆盖{N}年，未经历完整牛熊周期" |
| 5-7年 | 🔵 备注 | "数据覆盖{N}年，基本可信" |
| ≥ 7年 | ✅ 无提示 | — |

### 实现方式
从 `index_valuations` 表中查询该指数最早和最新的 `snapshot_date`，计算跨度：

```sql
SELECT index_code,
       MIN(snapshot_date) as earliest,
       MAX(snapshot_date) as latest,
       ROUND((julianday(MAX(snapshot_date)) - julianday(MIN(snapshot_date))) / 365.25, 1) as years
FROM index_valuations
GROUP BY index_code
```

## 三、百分位类型说明标签

### 问题
用户不清楚百分位是"时间百分位"还是"空间百分位"，两种算法结果差异大。

### 方案

在估值卡片/详情页增加类型说明标签：

```
[时间百分位] 当前估值在过去N年中，有X%的时间比现在更低
```

当前系统使用的是**空间百分位**（螺丝钉数据源：min~max之间的位置），在展示时加个小标签说明：

```html
<span class="metric-tag">空间百分位</span>
<span class="metric-hint">当前值在历史最低~最高之间的位置</span>
```

## 四、指数规则变更记录（风险2：规则修改）

### 问题
指数规则修改后，历史估值失去参考价值（如中证100→中证A100、H股指数纳入互联网公司）。

### 方案

在 `index_info` 表中新增 `rule_changes` 字段（JSON），存储规则变更记录：

```json
{
  "rule_changes": [
    {
      "date": "2024-10",
      "description": "中证100更名为中证A100，从市值选股改为龙头策略",
      "impact": "估值中枢下移，历史百分位需重新校准"
    }
  ]
}
```

展示时在指数详情页显示：
```
⚠️ 规则变更 (2024-10): 中证100更名为中证A100，从市值选股改为龙头策略
   影响：估值中枢下移，历史百分位需重新校准
```

## 五、实现优先级

| 优先级 | 功能 | 工作量 | 价值 |
|--------|------|--------|------|
| P0 | PE/PB 背离预警 | 中 | 高 — 直接避免误判 |
| P0 | 数据覆盖年限标注 | 小 | 高 — 简单有效 |
| P1 | 百分位类型说明 | 小 | 中 — 提升理解 |
| P2 | 指数规则变更记录 | 中 | 中 — 需要人工维护 |
