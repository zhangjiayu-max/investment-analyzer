# 估值数据利用监测设计稿

## 1. 背景与目标

用户提出需要观测系统中三类估值数据是否真正被 AI 对话和 LLM 分析功能充分利用：

1. **螺丝钉估值**：`dd_valuations` + `dd_parse_tasks`，图片解析后的宽表数据
2. **雷牛牛估值**：`index_valuations` 详细估值表，含 PE/PB/PS/股息率、百分位、z-score
3. **工具查询估值**：`query_valuation` / `query_online_valuation` 工具通过智能降级（库内→akshare→天天基金）返回的数据

当前已有 `valuation_query_logs` 记录工具调用链路，但缺失两个关键观测维度：

- **上下文注入侧**：系统把多少估值数据塞进了 prompt，数据新鲜度和覆盖度如何
- **LLM 消费侧**：专家回答中是否实际引用了估值数据，有没有"给了数据但没用"或"没给数据却编造"的情况

本设计稿通过三步补齐监测能力，实现"数据源 → 工具调用 → 上下文注入 → LLM 引用"的完整可观测链路。

## 2. 现状分析

### 2.1 数据源与降级策略

`db/valuations.py::get_best_valuation()` 已实现四级降级：

```
雷牛牛 7天内(index_valuations) → 螺丝钉 30天内(dd_valuations) → 过期 365天内 → akshare在线 → 天天基金MCP
```

每次查询通过 `_log_valuation_query()` 写入 `valuation_query_logs`。

### 2.2 工具调用链路

`tools/__init__.py::execute_tool()` 执行 `query_valuation` 时：

- 已带 `trace_id`、`conversation_id`、`message_id` 参数
- 已写入 `tool_audit_logs` 审计日志
- 已有工具缓存命中统计

但 `valuation_query_logs` 未记录 `conv_id/message_id/agent_name`，无法把一次估值查询关联回具体对话和专家。

### 2.3 上下文注入链路

`agent/core/orchestrator.py::_stream_build_context()` 会调用 `services/portfolio/portfolio_context.py::build_valuation_summary()`，把当前市场估值表格注入 `system_content` 和 `prebuilt_context`。

`build_valuation_summary()` 内部会遍历所有指数并调用 `get_best_valuation()`，这些调用已记录到 `valuation_query_logs`（`query_source=portfolio`）。

### 2.4 专家消费链路

专家在 function calling 中可主动调用 `query_valuation`，结果通过 `tool_calls` 字段随 `specialist_result` 返回。但目前没有对专家 `analysis` 文本做"是否引用估值"的检测。

## 3. 方案设计（三步）

### Step 1：扩展 `valuation_query_logs`，让工具调用可追踪

新增字段：

```sql
ALTER TABLE valuation_query_logs ADD COLUMN conv_id INTEGER;
ALTER TABLE valuation_query_logs ADD COLUMN message_id INTEGER;
ALTER TABLE valuation_query_logs ADD COLUMN agent_name TEXT;
ALTER TABLE valuation_query_logs ADD COLUMN user_query TEXT;
ALTER TABLE valuation_query_logs ADD COLUMN cache_hit INTEGER DEFAULT 0;
```

修改 `_log_valuation_query()` 签名，接收 `conv_id/message_id/agent_name/user_query/cache_hit`，并在 `get_best_valuation()` 各调用点传入。

### Step 2：新增 `valuation_context_usage`，记录上下文注入

建表：

```sql
CREATE TABLE IF NOT EXISTS valuation_context_usage (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    conv_id INTEGER,
    message_id INTEGER,
    trace_id TEXT,
    query_source TEXT,           -- 'stream_build_context' / 'portfolio_context' / 'analysis_route'
    valuation_summary_length INTEGER,  -- 注入的 markdown 字符数
    index_count INTEGER,         -- 覆盖指数数量
    data_sources TEXT,           -- 实际使用的数据源集合，逗号分隔
    metric_types TEXT,           -- 指标类型集合
    max_days_old INTEGER,        -- 最旧数据天数
    has_expired INTEGER,         -- 是否包含过期数据
    online_fallback_count INTEGER, -- 触发在线兜底的次数
    created_at TEXT DEFAULT (datetime('now','localtime'))
);

CREATE INDEX IF NOT EXISTS idx_vcu_created ON valuation_context_usage(created_at);
CREATE INDEX IF NOT EXISTS idx_vcu_conv ON valuation_context_usage(conv_id);
```

在 `build_valuation_summary()` 返回前，把本次构建的元信息写入该表。

### Step 3：新增 `valuation_reference_check`，检测 LLM 实际引用

建表：

```sql
CREATE TABLE IF NOT EXISTS valuation_reference_check (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    trace_id TEXT,
    conv_id INTEGER,
    message_id INTEGER,
    agent_name TEXT,
    analysis_type TEXT,          -- 'specialist' / 'synthesis' / 'arbitration'
    context_injected INTEGER,    -- 是否注入了估值上下文
    tool_called INTEGER,         -- 是否调用过 query_valuation/query_online_valuation
    tool_call_count INTEGER,     -- 工具调用次数
    refs_found INTEGER,          -- 回答中引用估值的次数
    ref_sources TEXT,            -- 引用到的指数/基金名，JSON 数组
    ref_metrics TEXT,            -- 引用到的指标（PE/PB/分位/百分位等）
    has_hallucination_risk INTEGER, -- 1=未给数据却出现数值 / 给了数据却未引用
    confidence TEXT,             -- 'high' / 'medium' / 'low'
    sample_text TEXT,            -- 引用片段，前 500 字符
    created_at TEXT DEFAULT (datetime('now','localtime'))
);

CREATE INDEX IF NOT EXISTS idx_vrc_created ON valuation_reference_check(created_at);
CREATE INDEX IF NOT EXISTS idx_vrc_trace ON valuation_reference_check(trace_id);
CREATE INDEX IF NOT EXISTS idx_vrc_conv ON valuation_reference_check(conv_id);
```

检测规则：

1. **context_injected**：检查 `valuation_context_usage` 同 trace_id 是否存在
2. **tool_called**：检查 `tool_audit_logs` 同 trace_id 是否有 `query_valuation` / `query_online_valuation`
3. **refs_found**：用正则匹配 `analysis` 中的估值关键词和数值
   - 关键词：`PE|PB|PS|市盈率|市净率|市销率|股息率|分位|百分位|低估|高估|合理|极度低估|极度高估|zscore|z-score`
   - 数值：关键词附近 30 字符内出现数字/百分号
4. **has_hallucination_risk**：
   - `context_injected=1 AND tool_called=1 AND refs_found=0` → 风险等级 medium（给了数据但可能未显式引用）
   - `context_injected=0 AND tool_called=0 AND refs_found>0` → 风险等级 high（未给数据却引用，疑似编造）

## 4. 关键代码修改点

### 4.1 `db/valuations.py`

- 修改 `_log_valuation_query()` 接收 `conv_id`, `message_id`, `agent_name`, `user_query`, `cache_hit`
- 在 `get_best_valuation()` 调用 `_log_valuation_query()` 时透传上下文
- 新增 `get_valuation_query_stats_v2()` 支持按 conv_id / agent_name / 时间范围聚合

### 4.2 `db/__init__.py`

- 在 `init_db()` 中为 `valuation_query_logs` 增加迁移字段
- 新增 `valuation_context_usage` 和 `valuation_reference_check` 建表语句

### 4.3 `services/portfolio/portfolio_context.py`

- 修改 `build_valuation_summary()`：
  - 收集 `data_sources`, `metric_types`, `max_days_old`, `has_expired`, `online_fallback_count`
  - 返回前调用 `log_valuation_context_usage()`

### 4.4 `agent/core/orchestrator.py`

- `_stream_build_context()` 调用 `build_valuation_summary()` 后，把 `conv_id/message_id/trace_id` 写入 `valuation_context_usage`
- 在专家结果收集阶段（specialist_results 生成后），对每个专家调用 `check_valuation_reference()`
- 在最终综合报告生成后，对综合报告也调用一次 `check_valuation_reference()`

### 4.5 新增 `services/valuation/valuation_monitor.py`

集中实现：

- `log_valuation_context_usage()`
- `check_valuation_reference()`
- `get_valuation_usage_report()`
- `detect_valuation_hallucination_risks()`

## 5. 查询接口与观测脚本

### 5.1 后端接口（新增）

- `GET /api/admin/valuation-usage/stats?days=7`：整体统计
  - 总查询次数
  - 各数据源命中占比
  - 上下文注入次数 / 平均覆盖指数数
  - LLM 引用率
  - 幻觉风险数
- `GET /api/admin/valuation-usage/conversations?days=7&limit=50`：按对话统计
- `GET /api/admin/valuation-usage/risks?days=7&limit=20`：幻觉风险列表

### 5.2 观测脚本

新增 `backend/scripts/valuation_usage_report.py`，支持命令行输出日报：

## 6. 观测指标

| 指标 | 来源表 | 用途 |
|------|--------|------|
| 估值查询次数 | valuation_query_logs | 观测工具使用频率 |
| 数据源命中率 | valuation_query_logs.final_source | 评估螺丝钉/雷牛牛/在线兜底贡献 |
| 平均覆盖指数数 | valuation_context_usage.index_count | 评估上下文覆盖广度 |
| 数据新鲜度 | valuation_context_usage.max_days_old | 评估分析时效性 |
| 专家估值引用率 | valuation_reference_check | 评估 LLM 是否消费了估值数据 |
| 幻觉风险率 | valuation_reference_check.has_hallucination_risk | 识别编造或漏用数据 |
| 缓存命中率 | valuation_query_logs.cache_hit | 评估工具去重效果 |

## 8. 开关与性能

- 所有新增写入通过 `system_config` 开关控制，默认开启：
  - `valuation.context_usage_enabled`（默认 true）
  - `valuation.reference_check_enabled`（默认 true）
- 引用检测为正则匹配，零 LLM 成本
- 写入均为异步/后台执行，不阻塞主流程
