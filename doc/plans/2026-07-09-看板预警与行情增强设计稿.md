# 每日看板增强：持仓预警提醒 + 市场行情总览

**日期**：2026-07-09
**目标**：补齐每日看板缺失的两块核心数据——持仓预警（需立即关注）和市场行情总览（市场环境锚点），让用户打开看板就能掌握"今天该关注什么"+"市场环境如何"。

## 一、持仓预警提醒

### 1.1 现状与问题

- `portfolio_alerts` 表已有 **105 条预警 / 8 种类型 / 99 条未读**（截至 2026-07-09）
- `alert_scanner.py` 服务每 30 分钟自动扫描，`routers/portfolio.py:351` 的覆盖版扫描器更生成 7 种类型（含 danger 级）
- 已有 9 个 alerts API 端点（`GET /api/portfolio/alerts` 等）
- **但看板完全没有展示**，用户必须切到持仓页才能看到预警

### 1.2 预警类型与分级

数据库实际出现的 alert_type 及展示分级：

| alert_type | 默认 severity | 展示等级 | 含义 |
|---|---|---|---|
| `buy_drop_alert` | danger / warning | 🔴 紧急 | 大跌机会提示 |
| `concentration_alert` | warning / danger | 🟡 警告 | 集中度过高 |
| `concentration_high` | warning | 🟡 警告 | 单标的占比超阈值 |
| `loss_warning` | warning | 🟡 警告 | 单标的亏损超阈值 |
| `valuation_high` | warning | 🟡 警告 | 估值偏高 |
| `valuation_low` | info | 🔵 提示 | 估值偏低机会 |
| `daily_advice_signal` | info | 🔵 提示 | 每日持仓建议信号 |
| `news_impact` | info | 🔵 提示 | 新闻影响持仓 |
| `recommendation_verified` | info | 🔵 提示 | 推荐验证结果 |
| `watchlist_trigger` | info | 🔵 提示 | 观察列表命中 |

### 1.3 后端改造

**无需新增 API**，复用现有端点：

- `GET /api/portfolio/alerts?unread_only=true&limit=20` — 获取未读预警
- `PUT /api/portfolio/alerts/{alert_id}/read` — 标记已读
- `POST /api/portfolio/alerts/scan` — 手动触发扫描（可选按钮）

**唯一调整**：在 `GET /api/portfolio/alerts` 返回中确认已包含 `holding_id` 字段（用于跳转持仓页定位）。若未包含，在 `routers/portfolio.py` 的响应中补上。

## 二、市场行情总览

### 2.1 现状与问题

- `services/market_data.py:409` 的 `get_market_overview()` 已能返回主要指数涨跌、领涨/领跌板块、涨跌家数
- 5 分钟进程内缓存，3 个 akshare 接口串行调用
- **但该函数没有独立 REST 端点**，只在日报/简报生成时作为内部数据源被调用
- 看板只有文字新闻（hot-topics），缺少量化行情锚点

### 2.2 后端改造

**新增端点**：`GET /api/market/overview`

**文件**：`backend/routers/market_intelligence.py`（已 import 但未使用，正好激活）

```python
@router.get("/market/overview")
async def get_market_overview_api():
    """市场行情总览：主要指数 + 领涨/领跌板块 + 涨跌家数。
    数据 5 分钟缓存，前端可高频调用。
    """
    try:
        data = get_market_overview()

        raise HTTPException(status_code=500, detail=f"行情数据获取失败: {e}")
```
