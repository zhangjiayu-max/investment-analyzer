# 智能补仓多维度触发器 设计稿

> 日期：2026-07-17
> 触发：用户反馈"智能补仓那里没看懂，希望结合持仓亏损+上次补仓数+估值+近期涨势好+连续大跌4%定投等多个维度触发补仓"
> 范围：在现有 smart_add_planner 之上增加"多维度触发器"层，补齐 3 个缺失维度

---

## 一、现状诊断

### 1.1 现有 smart_add_planner（主引擎）只看 2 个维度

[smart_add_planner.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/services/advisor/smart_add_planner.py) L500-505 金字塔触发条件：

```python
if (
    cfg["pyramid_enabled"]                          # 总开关
    and profit_rate_pct <= cfg["loss_threshold"]    # 维度1：亏损率 ≤ -10%
    and valuation_ok                                # 维度2：估值分位 < 60%
    and pyramid_aggressive                          # 基金类型允许
):
```

**问题**：只有"亏损+估值"两个维度，且必须同时满足才触发。用户想要的"涨势好时加仓"和"连续大跌4%定投"完全无法触发。

### 1.2 用户期望的 5 个维度，现状对照

| 维度 | smart_add_planner | daily_position_advisor | 差距 |
|------|------|------|------|
| 1. 持仓亏损 | ✅ 金字塔核心 | ❌ 无 | 无差距 |
| 2. 上次补仓数（冷却期） | ❌ 完全未读交易历史 | ✅ `_count_recent_buys` 已实现 | smart_add 未集成 |
| 3. 指数/基金估值 | ✅ z-score + 暂停阈值 | ❌ 无 | 无差距 |
| 4. 近期涨势好加仓 | ❌ 无趋势维度 | ✅ `_detect_momentum_breakout` 独立信号 | smart_add 未集成 |
| 5. 连续大跌4%定投 | ❌ 引擎1是月度定投 | ✅ `dca_drop_step_pct=4` 独立逻辑 | smart_add 未集成 |

### 1.3 核心问题

**两套独立系统各做各的，用户打开 SmartAddPlan 页面只能看到金字塔补仓（亏损-10%才触发），看不到涨势好/大跌4%的信号**。daily_position_advisor 的信号没有回流到 smart_add_planner 的计划表里。

## 二、设计方案：多维度触发器

### 2.1 核心思路

在 smart_add_planner 中新增"触发信号层"，把 5 个维度整合为 3 类触发信号，**任一信号命中即生成补仓建议**（现有金字塔只是其中之一）：

```
┌─────────────────────────────────────────────────────┐
│  多维度触发器（新增）                                  │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  │
│  │ 信号A: 金字塔 │  │ 信号B: 趋势  │  │ 信号C: 大跌  │  │
│  │ (亏损+估值)   │  │ (涨势好)     │  │ (4%定投)    │  │
│  │  现有逻辑     │  │  新增        │  │  新增        │  │
│  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘  │
│         │                │                │          │
│         └────────────────┴────────────────┘          │
│                          │                            │
│              ┌───────────▼───────────┐                │
│              │  冷却期检查（新增）     │                │
│              │  10天内≤2次 → 通过     │                │
│              └───────────┬───────────┘                │
│                          │                            │
│              ┌───────────▼───────────┐                │
│              │  安全阀（现有）         │                │
│              │  仓位上限+资金池硬约束   │                │
│              └───────────┬───────────┘                │
│                          │                            │
│              ┌───────────▼───────────┐                │
│              │  计划表+决策候选        │                │
│              └───────────────────────┘                │
└─────────────────────────────────────────────────────┘
```

### 2.2 三类触发信号定义

#### 信号 A：金字塔补仓（现有，保留）
- **触发条件**：亏损率 ≤ -10% AND 估值分位 < 60% AND 非债基
- **建议金额**：金字塔档位释放（10:15, 20:25, 30:30, 40:20, 50:10）
- **适用场景**：深度套牢的摊低成本

#### 信号 B：趋势加仓（新增）—— "近期涨势好"
- **触发条件**（三选二）：
  1. 估值分位 35-60%（合理区间，非高估）
  2. 近 20 日基金净值涨幅 > 3%（趋势向上）
  3. 近 5 日有资金净流入信号（可选，数据不可得时降级为"近20日涨幅>5%"）
- **建议金额**：基础月投 × 1.5 倍（轻仓试探，标注"短期波段"）
- **仓位上限**：单标的 ≤ 5% 总资产（小仓位，严格止损 -5%）
- **适用场景**：估值合理+趋势向上时小仓位跟进

#### 信号 C：大跌定投（新增）—— "连续大跌4%定投"
- **触发条件**：
  1. 相对上次买入价（或近 30 日高点）累计跌幅 ≥ 4%
  2. 估值分位 < 60%（非高估，避免接飞刀）
  3. 近 10 天内买入次数 < 2 次（冷却期）
- **建议金额**：基础月投 × 1 倍（标准定投额，不翻倍）
- **触发后下一档**：累计跌幅达 8% → 再触发一次（月投 × 1.5）；达 12% → 月投 × 2
- **适用场景**：分批定投摊低成本，非一次性抄底

### 2.3 冷却期检查（新增，所有信号共享）

在任一信号触发后、生成建议前，检查 `portfolio_transactions` 表：
- 近 `cooldown_days`（默认 10 天）内，同基金的买入次数（含真实+假设交易）
- 若已达 `max_buys_in_cooldown`（默认 2 次）→ 跳过该信号，标注 `blocked_reason="冷却期内已补仓X次"`
- **例外**：信号 C 的大跌定投，若累计跌幅 ≥ 8% 可突破冷却期（分批接刀）

### 2.4 信号优先级与冲突处理

同一基金可能同时命中多个信号（如亏损-15% + 近期大跌8%），处理规则：
1. **信号 A 金字塔优先级最高**（深度套牢场景）
2. 信号 A 与信号 C 大跌定投**互斥**（亏损-10%已触发金字塔，不再触发4%定投，避免重复建议）
3. 信号 B 趋势加仓与信号 A/C **可叠加**（但总建议金额受安全阀约束）
4. 前端计划卡显示所有命中信号，标注主信号
