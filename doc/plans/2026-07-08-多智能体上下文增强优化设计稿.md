# 2026-07-08 多智能体上下文增强优化设计稿

## 背景

调研发现 Pipeline 默认路径下专家看不到完整持仓盈亏和交易记录，影响决策质量。经评估 5 个建议后，采纳 3 个修复 + 2 个替代方案，跳过 1 个不建议改的项。

## 问题清单

### P0: Pipeline 持仓注入不完整（必须修）

**位置**：[pipeline.py:350](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/pipeline.py#L350)

**现状**：
```python
results["portfolio"] = build_user_memory_context("default")[:800]
```

`build_user_memory_context` 只返回基金名列表（[memory.py:186](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/memory.py#L186)）：
```python
parts.append(f"<portfolio>用户当前持有: {'、'.join(holding_names)}</portfolio>")
```

**不含盈亏数字、不含交易记录、不含集中度、不含风险信号。** 专家无法基于持仓实际状况给出建议。

**修复方案**：改为 `build_portfolio_context()`，预构建一次复用给所有专家。

```python
# pipeline.py:347-352 修改
try:
    from services.portfolio_context import build_portfolio_context
    results["portfolio"] = build_portfolio_context("default")
except Exception as e:
    logger.debug(f"[pipeline] 持仓加载失败: {e}")
```

**token 成本控制**：`build_portfolio_context()` 返回约 1500-2000 字，3 个专家共享同一份（通过 Layer 3 注入），不会 3 倍重复。Layer 3 的 `max_tokens=1500` 会自动压缩。

### P1: system_prompt/KYC 重复注入（应该修）

**位置**：[multi_agent.py:481-497](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/multi_agent.py#L481)

**现状**：Pipeline 路径下，`build_specialist_context` 的 Layer 0 已含 system_prompt，Layer 1 已含 KYC。但 `run_specialist` 收到 `prebuilt_context` 后又重新拼接：
```python
system_content = agent["system_prompt"]  # 重复
if context:
    system_content += f"\n\n以下是相关上下文信息...\n{context[:6000]}"  # context 就是 prebuilt_context
# 注入持仓上下文（prebuilt_context 非空时跳过 build_portfolio_context）
if prebuilt_context:
    system_content += f"\n\n{prebuilt_context}"  # 重复注入
system_content = _inject_kyc_profile(system_content, agent)  # KYC 重复
```

**修复方案**：Pipeline 路径传 `prebuilt_context` 时，标记 `from_pipeline=True`，跳过重复拼接。

```python
# multi_agent.py run_specialist 签名增加 from_pipeline 参数
def run_specialist(agent_key, query, context="", prebuilt_context="", 
                   from_pipeline=False, ...):
    if from_pipeline:
        # Pipeline 路径：prebuilt_context 已含 Layer 0-5，直接用作 system
        system_content = prebuilt_context
        # 追加通用约束（不在 Layer 0 中的）
        system_content += _UNIVERSAL_CONTEXT_INSTRUCTION
        system_content += _ACTIONABILITY_CONSTRAINT
        system_content += _RISK_AND_DEPTH_CONSTRAINT
        # 跳过 system_prompt、KYC、持仓的重复注入
    else:
        # ReAct 路径：保持原逻辑
        system_content = agent["system_prompt"]
        ...
```

**效果**：每个专家节省 ~1000 token 的重复注入。

### P2: 专家工具分配不全（应该修）

**修复方案**：给 `risk_assessor` 和 `allocation_advisor` 增加 `query_portfolio` 工具。

# db/agents.py _init_wealth_specialists 中

"tools": ["search_knowledge", "query_valuation", "query_portfolio", "yingmi_latest_quotations"]

### P1 替代: Phase 4 综合注入持仓约束提醒（非完整持仓）

**位置**：[pipeline.py:_phase_synthesis:1032](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/pipeline.py#L1032)

**原建议**：注入完整持仓到综合 prompt。**不合理** — 专家已看过持仓，综合时再看会重复。

**替代方案**：只注入"持仓约束提醒"，让综合做最终校验。

```python
# _phase_synthesis 中，构建综合 prompt 时追加
from services.portfolio_context import build_portfolio_summary_line
portfolio_line = build_portfolio_summary_line()  # 单行摘要，如"持仓: 沪深300(30%), 中证500(20%)"
if portfolio_line and "无持仓" not in portfolio_line:
    parts.append(f"\n## 持仓约束提醒\n当前{portfolio_line}\n综合时请校验：建议是否会导致集中度>25%？是否与近期操作冲突？")
```

**效果**：综合阶段用单行摘要做约束校验，不重复注入完整表格。

### P1 替代: SmartRouter 持仓感知路由（非 plan_generator）

**位置**：[router.py:_rule_route:118](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/router.py#L118)

**原建议**：plan_generator 注入持仓。**不合理** — 选专家不需要看具体盈亏数字，且路由命中时跳过 LLM。

**替代方案**：优化 SmartRouter 的 `_rule_route`，根据持仓状况调整路由。

```python
def _rule_route(self, query: str, portfolio_summary: str = "") -> Optional[dict]:
    specialists = set()
    for keywords, agents in _KEYWORD_ROUTES:
        if any(kw in query for kw in keywords):
            specialists.update(agents)
    
    # 持仓感知路由：根据持仓状况追加专家
    if portfolio_summary:
        # 重仓某板块 → 追加风险专家
        if "占比" in portfolio_summary:
            # 解析占比，若某基金占比>25% → 追加 risk_assessor
            import re
            matches = re.findall(r'([\d.]+)%', portfolio_summary)
            if matches and max(float(m) for m in matches) > 25:
                specialists.add("risk_assessor")
        # 现金占比高 → 追加配置专家
        if "现金" in portfolio_summary:
            cash_match = re.search(r'现金.*?(\d+)%', portfolio_summary)
            if cash_match and int(cash_match.group(1)) > 30:
                specialists.add("allocation_advisor")
    
    if _is_high_risk_action(query):
        specialists.update(["risk_assessor"])
    ...
```

**调用处修改**：`route` 方法把 `portfolio_summary` 传给 `_rule_route`。

```python
# router.py:281
rule_result = self._rule_route(query, portfolio_summary)
```

**效果**：路由阶段感知持仓，重仓时自动追加风险专家，现金多时追加配置专家。

### 不改: LLM 摘要默认关闭

`llm_cost.memory_summarize` 默认 false 是合理的成本控制。开启后每次对话都调 LLM 做摘要，成本高收益低。当前 `_fallback_summary()` 简单截取虽不完美但够用。如需改善记忆压缩，建议优化截取策略（保留首尾、按关键词提取），而非调 LLM。
