# RAG 检索系统设计文档

> 最后更新：2026-06-07
> 状态：已上线

## 一、系统概述

RAG（Retrieval-Augmented Generation）检索系统为投资分析助手提供知识检索能力，支撑 Agent 对话时引用专业知识。系统采用 **FTS5 全文检索 + ChromaDB 向量语义搜索** 双路召回，通过 RRF 融合排序实现高精度检索。

### 核心指标

| 指标 | 值 | 说明 |
|------|-----|------|
| 召回率（测试集） | 80% | 20 个测试用例，覆盖精确/语义/跨书/边界场景 |
| 平均检索延迟 | 160ms | 含 FTS5 + 向量搜索 + 融合排序 |
| 知识库规模 | 2062 条向量 + 2052 条全文 | 13 本书籍 + 122 篇作者文章 + 234 条技能文档 |

## 三、核心组件

### 3.1 FTS5 全文检索

**存储**：SQLite FTS5 虚拟表 `knowledge_fts`

| 字段 | 类型 | 说明 |
|------|------|------|
| content_type | TEXT | book / author_article / skill / valuation |
| title | TEXT | jieba 预分词后的标题 |
| body | TEXT | jieba 预分词后的内容 |
| reference_id | TEXT | 关联 ID |

**检索策略**：
1. jieba 分词 → 过滤停用词 → 清洗特殊字符
2. 多字词（≥2字）用 AND 连接，单字词用 OR 补充
3. AND 结果 < 3 条时，降级为 OR 查询（提高召回）
4. 时效性过滤：估值数据超过 30 天的降权

**代码位置**：`backend/rag.py` → `search_knowledge()`, `_build_fts_query()`

### 3.2 ChromaDB 向量检索

**Embedding 模型**：`moka-ai/m3e-base`

| 参数 | 值 |
|------|-----|
| 模型 | m3e-base (MokaAI) |
| 参数量 | 102M |
| 向量维度 | 768 |
| 模型大小 | 407MB |
| 加载时间 | ~3s（首次） |
| 编码速度 | <10ms/query |

**Collection 配置**：
- 名称：`knowledge`
- 距离度量：cosine
- 存储路径：`data/chroma_db/`

**检索策略**：
- 候选数量 = limit × 2（放大候选提升召回）
- 支持单类型和多类型过滤（`$in` 操作符）
- 向量归一化后计算余弦距离

**代码位置**：`backend/rag.py` → `search_chroma()`, `_ensure_embed_model()`

### 3.3 查询扩展

**同义词词典**（`rag_enhanced.py` → `SYNONYMS`）：

| 类别 | 示例 |
|------|------|
| 估值指标 | PE → 市盈率/估值/valuation |
| 投资策略 | 定投 → 定期定额/DCA/定额投资 |
| 口语化表达 | "什么时候卖" → 卖出时机/离场信号/卖出规则 |
| 人物关联 | 格雷厄姆 → 聪明的投资者/价值投资之父 |
| 市场情绪 | 恐慌 → 恐惧/悲观/抛售潮/踩踏 |

**扩展逻辑**：
- 按 key 长度降序匹配（"指数基金" 优先于 "基金"）
- 支持中文子串匹配（无空格分词）
- 同时用于 FTS5 和 ChromaDB 搜索

**知识图谱**（`KNOWLEDGE_GRAPH`）：
- 节点：投资概念（PE、定投、估值等）
- 边：related 关系（PE ↔ PB ↔ 股息率）
- 扩展方式：查询命中的节点，补充其 related 概念

**代码位置**：`backend/rag_enhanced.py` → `expand_query_with_synonyms()`, `expand_query_with_graph()`

### 3.4 RRF 融合排序

**算法**：Reciprocal Rank Fusion

```
score(doc) = Σ 1/(k + rank_i)

其中：
- k = 60（常数，控制排名衰减速度）
- rank_i = 文档在第 i 路检索中的排名（从 1 开始）
- 两路：FTS5 + ChromaDB
```

**加权机制**：

| 机制 | 权重 | 说明 |
|------|------|------|
| 标题匹配 | +10% × 覆盖率 | 关键词在标题中的覆盖率 |
| 双路命中 | ×1.15 | FTS5 和 ChromaDB 同时命中 |
| 同书多样性 | ×0.85 | 同一本书超过 3 条后逐条衰减 |
| 分数阈值 | 0.012 | 低于阈值的结果视为噪声过滤 |
| 估值时效 | +0.05/+0.02 | 7天内/30天内的估值数据加分 |

**代码位置**：`backend/rag.py` → `build_rag_context_with_details()`

### 3.5 轻量级重排序

**算法**：token 重叠率 + BM25 风格打分

```
score = 0.35 × title_overlap
      + 0.25 × body_overlap
      + 0.25 × title_exact_match
      + 0.15 × body_contains
```

**tokenize**：按空格切分 + 2-gram 切分长词

**代码位置**：`backend/rag_enhanced.py` → `lightweight_rerank()`

## 四、知识入库流程

### 4.1 书籍蒸馏入库

```
PDF/Markdown
  │
  ▼
OCR + LLM 蒸馏（distill.py）
  │
  ▼
SQLite knowledge_base 表（category=book）
  │
  ├──▶ FTS5 knowledge_fts 表（content_type=book）
  │    └── jieba 预分词写入
  │
  └──▶ ChromaDB knowledge collection
       └── m3e-base 编码 → 768 维向量
```

### 4.2 作者文章入库

```
爬取文章 → SQLite author_articles 表
  │
  ├──▶ FTS5 knowledge_fts（content_type=author_article）
  └──▶ ChromaDB（content_type=author_article）
```

### 4.3 估值数据入库

```
akshare 拉取 → SQLite valuation_snapshots 表
  │
  └──▶ FTS5 knowledge_fts（content_type=valuation）
       （ChromaDB 不存估值数据，时效性太强）
```

## 五、Agent 对话集成

### 5.1 知识类型配置

每个 Agent 通过 `knowledge_scope.rag_types` 配置可检索的知识类型：

| Agent | rag_types | 说明 |
|-------|-----------|------|
| 估值分析师 | valuation, analysis, book | 估值 + 分析记录 + 书籍 |
| 投资研究助手 | article, valuation, analysis, book | 全类型 |
| 风险管理师 | valuation, analysis, book | 风险相关 |
| 资产配置师 | valuation, article, book | 配置相关 |

### 5.2 检索流程

```
用户消息
  │
  ▼
解析 Agent knowledge_scope → 获取 rag_types
  │
  ▼
build_rag_context_with_details(query, content_types=rag_types)
  │
  ▼
格式化 RAG 上下文（max 3000 字）
  │
  ▼
注入 Agent system_prompt
  │
  ▼
LLM 生成回答（引用知识来源）
```

## 九、已知限制与优化方向

### 9.1 已知限制

| 问题 | 原因 | 影响 |
|------|------|------|
| 口语化查询召回不足 | embedding 模型对口语→术语的映射有限 | 语义匹配类查询通过率偏低 |
| 无关查询仍有噪声 | FTS5 OR 回退 + 向量搜索总会返回结果 | "量子力学 投资" 仍返回 10 条 |
| 《投资要义》霸榜 | 该书知识点标题关键词密度高 | 跨书检索多样性不足 |

### 9.2 后续优化方向

| 方向 | 成本 | 预期收益 |
|------|------|----------|
| 换 bge-large-zh-v1.5 (1024维) | 模型 1.2GB，需重建索引 | 语义匹配 +5-10% |
| 开启 Cross-Encoder Reranker | +500ms 延迟 | 精度 +3-5% |
| 知识点标题优化 | 人工标注 | 精确匹配 +10% |
| 查询意图分类 | 新增模块 | 区分事实查询 vs 分析查询 |
| 多轮对话上下文感知 | 新增模块 | 利用对话历史优化查询 |
