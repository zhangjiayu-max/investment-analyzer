# 知识库蒸馏质量优化

## 问题

"金融投资经典系列共6册" PDF 蒸馏出 778 条记录，其中 703 条是垃圾数据（平均 58 字，标题为随机文本碎片），严重污染知识库。根因是 PyPDF2 文本提取质量差。

## 方案：两阶段蒸馏

### 阶段一：PDF → 干净文本（LLM 视觉识别）

脚本 `scripts/ocr_book.py`：
1. PDF 每页 → 图片 → base64 → mimo-v2.5 图片理解 → 提取文字
2. 每 10 页合并为一个 chunk，LLM 整理成连贯文本
3. 每本书输出一个干净的 markdown 文件到 `data/books/` 目录

PDF 结构（3126 页）：
| 页码 | 书名 |
|------|------|
| 56-517 | 股市长线法宝 |
| 518-1157 | 专业投机原理 |
| 1158-1494 | 以交易为生 |
| 1495-1827 | 日本蜡烛图技术新解 |
| 1828-2589 | 笑傲股市 |
| 2590-3126 | 通向财务自由之路 |

### 阶段二：干净文本 → 知识库蒸馏

脚本 `scripts/distill_from_text.py`：
1. 读取 `data/books/*.md` 文件
2. 按章节分块，每块 3000 字
3. LLM 提取高质量知识点（200-500 字/条）
4. 质量过滤 + 去重
5. 写入 knowledge_base 表

### 清理旧数据

删除所有 `source LIKE '%金融投资经典系列共6册%'` 的记录（778 条）。

## 文件清单

| 文件 | 操作 |
|------|------|
| `backend/scripts/ocr_book.py` | 新建 — PDF 视觉识别转文本 |
| `backend/scripts/distill_from_text.py` | 新建 — 基于文本蒸馏 |
| `data/books/` | 新建目录 — 存放识别后的文本 |
| `data/valuations.db` | 清理垃圾数据 |

## 使用方式

```bash
# 第一步：识别（每本书约 30-60 分钟，共 6 本）
python3 scripts/ocr_book.py /path/to/pdf --book 以交易为生 --start 1158 --end 1494

# 第二步：蒸馏（每本书约 5-10 分钟）
python3 scripts/distill_from_text.py data/books/以交易为生.md
```
