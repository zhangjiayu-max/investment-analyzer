# 2026-07-09 P0 决策闭环 + 主动提醒 设计稿

## 背景诊断

当前系统"决策闭环断裂"的根因（代码实证）：
- `db/dashboard.py` 的 `save_recommendations()` 完整实现了建议保存（含 baseline/verify_window）
- `auto_verify_pending_recommendations()` 实现了自动验证
- **但 `agent/pipeline.py` 的 Phase 5 只调用 `save_analysis_conclusion`，从不调用 `save_recommendations`**
- 结果：AI 给出"减仓白酒10%"的建议后，建议就消失了——不落库、不追踪、不验证

"缺乏主动提醒"的根因：
- `db/portfolio.py` 的 `create_alert()` 完整实现了预警写入
- `portfolio_alerts` 表有 alert_type/severity/is_read 字段
- **但没有定时任务扫描估值/风险阈值生成 alert**——只有用户主动调用 `generate_portfolio_alert` 工具时才触发
- 结果：沪深300 跌到 20% 分位，用户不打开系统就不知道

## P0-A: 决策闭环

### 目标
AI 建议 → 自动落库 → T+N 自动验证 → 结果反馈给用户和 AI

### 设计

#### 1. Pipeline Phase 5 新增：建议提取与落库

在 `_phase_memory`（Phase 5）中，`save_analysis_conclusion` 之后，新增 `_extract_and_save_recommendations()`：

```python
def _extract_and_save_recommendations(state, query_info, trace_id):
    """从最终回答中提取建议并落库到 recommendations 表。"""
    # 1. 用 LLM 从 state.answer 提取结构化建议
    #    prompt: "从以下分析中提取投资建议，输出 JSON 数组：
    #    [{index_name, index_code, direction(up/down/hold), reason, confidence(high/medium/low)}]"
    # 2. 获取 baseline（当前估值/净值）
    # 3. 调用 save_recommendations(recs, analysis_id, baselines, verify_days=5)
    # 4. 返回保存的 recommendation_ids，写入 state 供前端展示
```

**开关**：`pipeline.save_recommendations_enabled`（默认 true）

**提取 prompt 约束**：
- 只提取明确的方向性建议（加仓/减仓/买入/卖出/持有）
- 模糊建议（"关注""留意"）不提取
- 每条建议必须关联具体标的（指数/基金）
- direction 必须是 up/down/hold 三选一

#### 2. 前端：建议卡片展示 + 采纳标记

在 ChatMessage.vue 中，当 AI 回复包含 recommendations 时，渲染建议卡片：

```

├─────────────────────────────────────┤
│ 沪深300 (000300)                     │
│ 方向: 加仓 ↑  置信度: 高             │
│ 原因: PE 分位 22%，显著低估          │

└─────────────────────────────────────┘
```

**采纳标记**：点击后更新 recommendations.status 为 `adopted` / `rejected` / `pending`

#### 3. T+N 自动验证（已有，需接入定时任务）

`auto_verify_pending_recommendations()` 已实现，需要：
- 在 `app.py` 的 `lifespan` 中启动后台定时任务（每小时检查一次）
- 扫描 `verify_after_date <= today AND status = 'pending'` 的记录
- 调用 `query_valuation` 获取当前值，调用 `verify_recommendation()`
- 验证后生成 alert 通知用户（"你 5 天前的减仓建议已验证：白酒下跌 3.2%，判断正确 ✅"）

#### 4. 决策准确率页面增强

现有 `DecisionAccuracy.vue` 已展示准确率统计。新增：
- **最近验证结果列表**：展示已验证的建议（正确/错误）
- **采纳率统计**：用户采纳了多少建议
- **采纳 vs 未采纳收益对比**：证明采纳建议的收益更高

## P0-B: 主动提醒

### 目标
估值阈值/风险变化/建议验证结果 → 主动生成 alert → 前端展示未读数 → 用户不打开系统也能知道

### 设计

#### 1. 后台扫描任务（app.py lifespan）

新增 `asyncio.create_task()` 后台任务，每 30 分钟执行一次：

```python
async def _periodic_alert_scan():
    """每 30 分钟扫描一次，生成主动提醒。"""
    while True:
        await asyncio.sleep(1800)
        try:
            _scan_valuation_thresholds()   # 估值阈值
            _scan_portfolio_risk()          # 持仓风险
            _scan_recommendation_verifications()  # 建议验证结果
        except Exception as e:
            logger.warning(f"主动提醒扫描失败: {e}")
```

**开关**：`alerts.proactive_scan_enabled`（默认 true）

#### 2. 估值阈值扫描 `_scan_valuation_thresholds()`

```python
def _scan_valuation_thresholds():
    """扫描用户关注指数的估值分位，触达阈值时生成 alert。"""
    # 1. 获取用户持仓涉及的指数（从 portfolio 提取）
    # 2. 对每个指数调用 query_valuation 获取当前分位
    # 3. 阈值规则（可配置）：
    #    - 分位 < 20% → severity=info, "XX 估值进入低估区（分位 18%），可关注加仓机会"
    #    - 分位 > 80% → severity=warning, "XX 估值进入高估区（分位 85%），注意减仓风险"
    # 4. 跨日合并：同指数同类型同日内不重复生成（用 idx_alerts_group 索引）
    # 5. 调用 create_alert() 落库
```

#### 3. 持仓风险扫描 `_scan_portfolio_risk()`

```python
def _scan_portfolio_risk():
    """扫描持仓集中度和亏损，生成风险 alert。"""
    # 1. 获取用户持仓
    # 2. 集中度检测：
    #    - 单标的占比 > 30% → warning "XX 占比 35%，集中度过高"
    #    - 行业集中度 > 50% → warning "白酒行业占比 55%，行业过于集中"
    # 3. 亏损检测：
    #    - 单标的亏损 > 15% → warning "XX 当前亏损 18%，建议关注"
    # 4. 调用 create_alert() 落库
```

- `pipeline.save_recommendations_enabled` = true
- `alerts.proactive_scan_enabled` = true
- `alerts.valuation_low_threshold` = 20（分位 <20% 触发低估提醒）
- `alerts.valuation_high_threshold` = 80（分位 >80% 触发高估提醒）
