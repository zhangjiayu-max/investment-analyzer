# 智能补仓增强算法设计稿（L2-L5 全套）

**日期**：2026-07-09
**前置**：基于已完成的 [2026-07-09-智能补仓计划器设计稿.md](./2026-07-09-智能补仓计划器设计稿.md) 的双引擎方案
**目标**：从"一刀切"升级为"按基金类型+波动率+历史回撤修复+估值胜率"的差异化智能补仓

---

## 一、当前方案局限

| 问题 | 说明 | 影响 |
|------|------|------|
| 一刀切倍数 | 所有基金用同一个 z-score 倍数公式 | 宽基/行业/债基补仓节奏一样，不合理 |
| 拍脑袋上限 | 单标的 25% 上限无数学依据 | 可能过度集中或过于保守 |
| 缺时间维度 | 不知道"跌下去后历史上多久能回本" | 补仓资金可能被困太久 |
| 缺胜率视角 | 不知道"当前估值买入后赚钱概率" | 无法量化补仓信心 |

---

## 二、分层算法体系

### L2：基金类型差异化策略（本地数据即可）

**分类标准**：基于 `fund_category` + `index_name` 关键词

| 类型 | 特征 | 波动率范围 | 补仓策略 |
|------|------|-----------|----------|
| 宽基 | 沪深300/中证500/创业板 | 18-22% | 凯利比例高，金字塔激进，周投 |
| 行业指数 | 白酒/医药/银行/地产 | 25-35% | 金字塔中档，月投，单次金额中等 |
| 主题基金 | 新能源/AI/半导体/军工 | >35% | 严格上限，只在极低估补，双月投 |
| 港股 | 恒生/港股通 | 25-30% | 参考行业指数，但需考虑汇率 |
| 债基 | 纯债/二级债 | <8% | 不用金字塔，只定投，修复快 |

**实施**：新增 `_classify_fund(holding)` 函数，返回类型 + 对应策略参数

### L3：半凯利公式（本地数据即可）

**公式**：
```
f* = (μ - r) / σ²    （预期超额收益 / 方差）
实际配置上限 = f* × 0.5（半凯利，安全边际）
```

**数据来源**：本地 `fund_nav_history` 表（8-13 年基金净值）
- μ：基金历史年化收益率（按日收益均值 × 252）
- r：无风险利率（10年期国债，默认 2.5%）
- σ：年化波动率（日收益标准差 × √252）

**用途**：替代 25% 拍脑袋上限，为每个基金算出科学的"最大配置比例"

**示例**：
- 白酒 161725：μ=12%, σ=28% → f*=(0.12-0.025)/0.28²=1.21 → 半凯利=60%
- 债基：μ=4%, σ=5% → f*=(0.04-0.025)/0.05²=6.0 → 半凯利上限100%（实际用 40% 封顶）

### L4：历史回撤修复时间分析（需前置数据回填）

**算法**：
1. 获取指数 11.5 年日线点位（akshare `stock_zh_index_hist_csindex`）
2. 识别所有"从高点下跌 X% 后回升至前高"的回撤-修复周期
3. 统计 -20%/-30%/-40% 回撤的中位修复月数

**数据来源**：akshare `stock_zh_index_hist_csindex`（已实测可用，白酒 11.5 年 2799 行）

**用途**：决定补仓节奏
- 修复 <12 月 → 激进补仓（月投×1.5）
- 修复 12-24 月 → 正常
- 修复 >24 月 → 拉长周期（双月投，避免资金被困）

### L5：估值分位历史胜率（需前置数据回填）

**算法**：
1. 获取指数 11.5 年"滚动PE + 收盘点位"（同一接口）
2. 计算每个历史日期的 PE 分位（0-100%）
3. 对每个分位区间（如 0-10%, 10-20%），统计"买入后 6/12/24 月正收益概率"

**数据来源**：同 L4，同一接口返回 PE + 点位

**用途**：补仓信心量化
- 胜率 >80% → 激进（倍数×1.3）
- 胜率 60-80% → 正常
- 胜率 <60% → 谨慎（倍数×0.7）

### 4.2 增强 `services/smart_add_planner.py`

在 `_generate_single_plan` 中注入 L2-L5 指标：

```python
# L2：基金类型分类
fund_type, type_strategy = _classify_fund(holding)

# L3：半凯利上限（替代 25% 拍脑袋）
kelly = calc_kelly_limit(fund_code)
max_position_pct = min(kelly["kelly_half"] * 100, type_strategy["hard_cap"])

# L4：修复时间调整节奏
recovery = calc_recovery_time(index_code)
if recovery and recovery["median_recovery_months"] > 24:
    rhythm_adjust = 0.5  # 双月投
elif recovery and recovery["median_recovery_months"] < 12:
    rhythm_adjust = 1.5  # 半月投
else:
    rhythm_adjust = 1.0

# L5：胜率调整倍数
win_rate = calc_valuation_win_rate(index_code, current_percentile)
if win_rate and win_rate["win_rate_12m"] > 0.80:
    confidence_mult = 1.3
elif win_rate and win_rate["win_rate_12m"] < 0.60:
    confidence_mult = 0.7
else:
    confidence_mult = 1.0

# 最终月投 = 基础 × z-score倍数 × 类型倍数 × 胜率倍数 ÷ 节奏调整
final_monthly = base_monthly * zscore_mult * type_strategy["dca_mult"] * confidence_mult / rhythm_adjust
```

## 七、不做的事

- **不做自动下单**：只生成计划，用户手动执行（合规）
- **不做融资补仓**：资金仅来自现金，不引入杠杆
- **不做个股**：仅限基金/指数
- **不做实时回测**：L4/L5 基于历史数据统计，非实时回测引擎
- **不替代现有 DCA**：增强 `smart_add_planner`，不改动 `daily_position_advisor`
