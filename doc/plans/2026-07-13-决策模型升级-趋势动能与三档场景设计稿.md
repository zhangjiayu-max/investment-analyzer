# 决策模型升级 — 趋势动能买入与三档加减仓场景

> 日期：2026-07-13
> 状态：待审核
> 优先级：P0
> 关联：解决"系统过度依赖估值，纯等低估会错过机会"的痛点

---

## 一、背景与问题

### 1.1 用户痛点

> "我感觉我们整个系统推荐很大都基于估值来的。我知道追高不好，但是有些时候可以适当上车，短期的流水也可以吧。如果都等低估肯定会错过更多机会。"

### 1.2 现状证据（调查结论）

| 链路 | 估值依赖度 | 关键证据 |
|------|-----------|---------|
| 对话型建议 | 中等偏重 | `orchestrator.py:3825` 注入"加仓必须≤35% / 减仓必须≥80%"作为"必须遵守"规则 |
| 主动信号扫描 | **极高** | `alert_scanner.py:153` 纯靠 20/80 阈值；`daily_position_advisor.py:911` 行业轮动纯靠 ≤10/≤20 |
| 多维评分 | 较合理 | `_calc_signal_score` 估值 25/100、趋势仅 10/100 |

**核心问题**：
1. DCA 规则用"必须遵守 ≤35%"硬注入，专家没有空间说"这次可以适当追高"
2. 资金流信号被模块设计者主动降级为"辅助确认，不修改 confidence"（`institutional_flow.py:1-12`）
3. 趋势信号在 `_calc_signal_score` 中只占 10/100，且必须先满足"低估+跌"才参与
4. **没有纯动量/趋势突破类买入信号生成器** — 估值不在低估区就永远不会建议买入
5. `orchestrator.py:1542` 存在 `get_watchlist` import bug（`has_watchlist` 永远 False）

### 1.3 设计目标

- **不再纯粹"低估才买"**：在合理估值区间，若趋势止跌回升 + 资金流入 + 行业景气，允许"小仓位试探性建仓"
- **保留估值作为安全边际**：高估区（>60%）仍然只持有/减仓，不追高
- **引入"短期流水"概念**：对趋势确认的标的，明确标注"短期波段，严格止损"
- **复用已有能力**：`_calc_trend_score`、`institutional_flow`、`_calc_signal_score` 都已存在，只需重新组织和加权

---

## 二、核心设计 — 三档场景决策模型

### 2.1 场景定义

将原来的"加仓≤35% / 减仓≥80%"二档硬阈值，升级为**三档场景**：

| 场景 | 估值分位 | 允许的操作 | 触发条件 | 仓位上限 |
|------|---------|-----------|---------|---------|
| **A. 低估区** | ≤35% | 正常加仓 / 定投 | 估值触发（原逻辑保留） | 单标的 15%（原值） |
| **B. 合理区** | 35%-60% | **小仓位试探性建仓**（新增） | 趋势止跌回升 + 资金流入 + 行业景气，三者满足≥2 | 单标的 5%，标注"短期波段" |
| **C. 高估区** | >60% | 仅持有 / 减仓 | 估值触发（原逻辑保留） | 不加仓 |

### 2.2 场景 B 的信号判定逻辑（核心新增）

**"趋势动能买入"信号** — 在合理估值区间允许适当上车的信号：

```
触发条件（三者满足≥2）：
  1. 趋势止跌回升：_calc_trend_score(fund_code) >= 7
     （近5日 +0% 以上，或止跌回升）
  2. 资金流入：institutional_flow.trend == "inflow"
     且 strength in ["moderate", "strong"]
  3. 行业景气：industry_score >= 8（满分15）
     （akshare 5日涨跌为正）

附加约束：
  - 近5日趋势分 <= 3 时强制不触发（持续大跌不接飞刀）
  - 单标的占比已达 15% 时不触发
  - 冷静期内（10天/2次）不触发
```

### 2.3 "短期流水"概念

对场景 B 触发的建议，附加以下字段：

| 字段 | 值 | 说明 |
|------|-----|------|
| `signal_type` | `momentum_breakout` | 区分于低估触发的 `value_dip` |
| `position_strategy` | `probe` | 试探性建仓（非定投） |
| `max_position_pct` | 5% | 严格限制仓位 |
| `stop_loss_pct` | -5% | **严格止损线** |
| `take_profit_pct` | +8% | 短期止盈线 |
| `holding_horizon` | `short_term` | 持有周期标注 |
| `risk_note` | "短期波段，严格止损-5%，不恋战" | 风险提示 |

### 3.2 新增函数：`_detect_momentum_breakout`

**位置**：`backend/services/daily_position_advisor.py`

```python
def _detect_momentum_breakout(
    fund_code: str,
    val_percentile: float | None,
    position_pct: float,
    recent_txs: list,
    config: dict,
) -> dict | None:
    """检测趋势动能买入信号（合理估值区间的适当上车机会）。

    场景 B：估值在 35%-60% 区间，但趋势止跌回升 + 资金流入 + 行业景气，
    允许小仓位试探性建仓。

    返回 None 表示未触发；触发则返回信号字典。
    """
    # 1. 估值必须在合理区（35%-60%）
    add_max_pct = config["add_valuation_max_percentile"]  # 35
    momentum_val_max = get_config_float("daily_advice.momentum_valuation_max_percentile", 60)
    if val_percentile is None or val_percentile <= add_max_pct or val_percentile > momentum_val_max:
        return None

    # 2. 近5日趋势分（复用已有函数）
    trend_score, trend_reason = _calc_trend_score(fund_code, config["trend_lookback_days"])
    if trend_score <= config["trend_danger_threshold"]:  # <=3 强制不触发
        return None

    # 3. 资金流信号（复用 institutional_flow）
    flow_signal = _get_fund_flow_signal(fund_code)  # 新增小封装
    flow_in = flow_signal["trend"] == "inflow" and flow_signal["strength"] in ["moderate", "strong"]

    # 4. 行业景气分（复用已有逻辑）
    industry_score, industry_reason = _calc_industry_score(fund_code)

    # 5. 三选二触发
    signals_passed = sum([trend_score >= 7, flow_in, industry_score >= 8])
    if signals_passed < 2:
        return None

    # 6. 附加约束
    if position_pct >= config["default_single_position_pct"]:  # 已达15%
        return None
    if _count_recent_buys(fund_code, recent_txs, config["recent_buy_cooldown_days"]) >= config["recent_buy_max_count"]:
        return None

    return {
        "signal_type": "momentum_breakout",
        "fund_code": fund_code,
        "val_percentile": val_percentile,
        "trend_score": trend_score,
        "trend_reason": trend_reason,
        "flow_signal": flow_signal,
        "industry_score": industry_score,
        "signals_passed": signals_passed,
        "max_position_pct": get_config_float("daily_advice.momentum_max_position_pct", 5),
        "stop_loss_pct": -5.0,
        "take_profit_pct": 8.0,
        "holding_horizon": "short_term",
        "risk_note": "短期波段，严格止损-5%，不恋战",
    }
```

### 3.3 `_calc_signal_score` 权重调整

**原权重**（7维度，满分100）：
```
1. 跌幅      0-25
2. 估值优势   0-25
3. 仓位空间   0-15
4. 流动性    0-10
5. 盈亏修正  -15~+10
6. 行业前景   0-15
7. 近5日趋势  0-10   ← 调整
```

**新权重**（8维度，满分100）：
```
1. 跌幅      0-20   ← 降5分
2. 估值优势   0-20   ← 降5分
3. 仓位空间   0-15
4. 流动性    0-10
5. 盈亏修正  -15~+10
6. 行业前景   0-15
7. 近5日趋势  0-15   ← 升5分（强化趋势权重）
8. 资金流向   0-5    ← 新增（资金流入加分，流出扣分）
```

### 3.4 DCA 规则注入改造

**位置**：`backend/agent/orchestrator.py:3825`

**原文**：
```
### 加仓条件
- 估值百分位必须 ≤ 35% 才建议加仓
```

**改为**：
```
### 加仓条件（三档场景）

#### 场景A：低估区（估值≤35%）— 正常加仓
- 估值触发，按 4% 定投法执行
- 单标的仓位上限 15%

#### 场景B：合理区（估值35%-60%）— 趋势动能小仓位试探
- 仅当以下三条件满足≥2时，允许小仓位试探性建仓：
  1. 近5日趋势止跌回升（trend_score ≥ 7）
  2. 资金净流入（institutional_flow.trend == inflow，强度≥moderate）
  3. 行业景气（industry_score ≥ 8/15）
- 单标的仓位上限 5%（严格限制）
- 必须标注"短期波段，严格止损-5%，止盈+8%"
- 近5日持续大跌（trend_score ≤ 3）禁止触发

#### 场景C：高估区（估值>60%）— 不加仓
- 仅持有或考虑减仓
```

## 五、风险控制

### 5.1 防止追高

- 场景B严格限制在估值 35%-60% 区间，>60% 永远不触发
- 单标的仓位上限 5%（远低于场景A的15%）
- 必须标注止损-5%

### 5.2 防止接飞刀

- 趋势分≤3（持续大跌）强制不触发
- 三选二机制，避免单一信号误判

### 5.3 防止过度交易

- 复用原有冷静期（10天/2次）
- 资金流信号强度要求≥moderate

## 八、预期效果

改造后，系统对一只"估值在50%分位、但近5日止跌回升+资金流入"的基金，会从原来的"不推荐"变为：

> 📈 趋势机会 · XX基金
> 估值分位 50%（合理区间），近5日 +1.2% 止跌回升，资金净流入（强度 moderate）
> 建议：小仓位试探性建仓（仓位上限 5%）
> ⚠️ 短期波段，严格止损-5%，止盈+8%，不恋战

这解决了"纯等低估错过机会"的痛点，同时通过严格的仓位限制和止损线控制风险。
