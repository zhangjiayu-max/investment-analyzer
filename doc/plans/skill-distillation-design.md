# 作者技能蒸馏方案

## 背景

从 122 篇公众号文章中蒸馏作者（研究员雷牛牛）的认知框架、言行模式、知识边界和经典素材，生成一份 skill 文档，用于增强 AI 对话的专业性和风格一致性。

---

## 一、蒸馏维度

| 维度 | 提取内容 | 示例输出 |
|------|----------|----------|
| **认知框架** | 分析问题的方法论、决策模型 | "先看估值分位，再看资金流向，最后看情绪面" |
| **言行模式** | 表达习惯、口头禅、格式偏好 | "喜欢用短句、imo标记、《》引用经典" |
| **知识边界** | 擅长/不擅长的领域标签 | "擅长：指数基金、量化策略；不擅长：个股深度" |
| **经典素材** | 高频引用的观点、数据、案例 | "恐慌指数买入法、恒生科技30%止损线" |

---

## 二、实现路径

### Phase 1: 批量提取

```
122 篇文章 → LLM 逐篇分析 → 122 份结构化 JSON
```

每篇文章提取：
```json
{
  "article_id": 120,
  "cognitive_framework": ["框架1", "框架2"],
  "behavior_patterns": ["短句为主", "常用imo"],
  "knowledge_strengths": ["指数基金", "量化策略"],
  "knowledge_weaknesses": ["个股深度分析"],
  "classic_quotes": ["原文摘要1", "原文摘要2"]
}
```

### Phase 2: 聚合

```
122 份提取结果 → 聚合算法 → 1 份 skill 文档
```

聚合策略：
- 认知框架：去重 + 频率排序
- 言行模式：提取高频特征
- 知识边界：统计话题出现频率
- 经典素材：按主题分类，保留原文

### Phase 3: 接入对话

skill 文档作为 system prompt 的一部分，或作为 RAG 的增强上下文。

---

## 四、对 AI 对话的提升

### 有效提升

| 场景 | 提升方式 |
|------|----------|
| **风格模仿** | AI 回复会更像作者的表达习惯（短句、imo、《》引用） |
| **知识引用** | 能直接引用作者说过的观点、案例，更有说服力 |
| **分析框架** | 用作者的思维方式分析新问题，保持一致性 |
| **避免越界** | 知道作者不擅长什么，不会在那些领域硬答 |

### 局限性

| 问题 | 说明 |
|------|------|
| **知识时效** | 作者的观点可能过时，AI 需要判断是否适用 |
| **风格僵硬** | 过度模仿可能显得不自然 |
| **冲突调和** | 作者观点 vs LLM 自身知识，需要明确优先级 |

```
## 表达风格
- 短句为主，简洁有力
- 常用 "imo"、"说实话"、"划重点"
- 引用经典时用《》标注

## 知识边界
- 擅长：指数基金、量化策略、市场情绪分析

## 引用规则

```

## 七、验证方式

1. **风格验证**：用蒸馏后的 skill 让 AI 回复问题，对比原作者风格
2. **知识验证**：问作者擅长领域的问题，检查回复质量
3. **边界验证**：问作者不擅长领域的问题，检查是否诚实说明
