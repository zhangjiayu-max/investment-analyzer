# API 路径规范化重构方案

## 一、现状问题

### 1.1 路径命名混乱

当前 API 路径存在以下问题：
- **路径层级不一致**：有的用 `/api/xxx`，有的用 `/api/xxx/yyy`
- **命名风格不统一**：有的用 kebab-case，有的用 camelCase
- **模块划分不清**：功能相关的 API 分散在不同路径下
- **路由顺序问题**：动态参数路由（`/{id}`）和静态路由冲突

## 二、规范化方案

### 2.1 命名规范

1. **路径格式**：`/api/{module}/{resource}[/{action}]`
2. **命名风格**：统一使用 kebab-case（短横线分隔）
3. **资源命名**：使用名词复数形式（如 `/agents` 而不是 `/agent`）
4. **动作命名**：使用动词或动词短语（如 `/create`、`/refresh`）
5. **动态参数**：放在路径末尾，使用 `{resource_id}` 格式

### 2.2 模块划分

| 模块 | 路径前缀 | 说明 |
|------|----------|------|
| **估值** | `/api/valuation` | 指数估值数据管理 |
| **持仓** | `/api/portfolio` | 投资组合管理 |
| **Agent** | `/api/agent` | Agent 系统管理 |
| **对话** | `/api/conversation` | 对话和聊天 |
| **任务** | `/api/task` | 任务管理 |
| **文章** | `/api/article` | 文章管理 |
| **Dashboard** | `/api/dashboard` | 仪表盘和报告 |

### 第一阶段：后端路由重构（2-3天）

1. 创建新的路由模块结构
2. 迁移现有路由到新路径
3. 添加旧路径的重定向（兼容性）
4. 测试所有 API

### 第二阶段：前端 API 更新（1-2天）

1. 更新 `api/index.js` 中的所有 API 调用
2. 测试所有前端功能

### 第三阶段：清理和文档（1天）

1. 删除旧路径的重定向
2. 更新 API 文档
3. 生成 OpenAPI 规范

## 五、兼容性策略

### 5.1 重定向方案

在 `app.py` 中添加旧路径的重定向：

```python
# 旧路径重定向
@app.get("/api/valuations")
async def redirect_valuations():
    return RedirectResponse(url="/api/valuation/indexes", status_code=301)
```

### 5.2 版本化方案

使用 API 版本号：

```
/api/v1/valuations  ->  旧版本（保留）
/api/v2/valuation   ->  新版本
```

### 5.3 渐进式迁移

1. 先实现新路径，旧路径保留
2. 前端逐步迁移到新路径
3. 最后删除旧路径
