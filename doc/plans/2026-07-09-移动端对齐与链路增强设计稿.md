## 二、对话链路增强

### 2.1 建议一：Reflection 自纠错循环

#### 现状
- `_phase_reflection`（[pipeline.py:1457](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/pipeline.py)）默认关闭（`pipeline.reflection_enabled=false`）
- `confidence_adjustment` 算了但只注入 prompt 文本，不触发重跑
- 无质量兜底

#### 设计
1. **默认开启规则版 Reflection**：`pipeline.reflection_enabled` 默认值改 `true`（单次 LLM ~800 token，成本可控）
2. **自纠错循环**（新）：Reflection 后检查 `confidence_adjustment`，低于阈值则重跑质量最差的专家
   - 新增配置 `pipeline.reflection_self_correct_enabled`（默认 true）
   - 新增配置 `pipeline.reflection_confidence_threshold`（默认 -0.2）
   - 逻辑：`confidence_adjustment < threshold` 且 `missing_perspectives` 命中已调用专家 → 重跑该专家（注入反思反馈到 prompt）
   - 防死循环：最多重跑 1 次，重跑结果覆盖 blackboard 对应 entry
3. **置信度写答案 metadata**：`confidence_adjustment` 写入 `phase4_result`，前端可展示

#### 改造点（[pipeline.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/pipeline.py)）
- 主流程 Phase 3.5 Reflection 后插入自纠错逻辑
- 新增 `_rerun_specialist_with_reflection(state, agent_key, query, reflection_result, blackboard, trace_id)` 函数
- `_phase_synthesis` 返回值增加 `confidence` 字段

#### 流程
```
Phase 3 执行 → Phase 3.5 Reflection
  → 检查 confidence_adjustment < -0.2?
    → 是：重跑 missing_perspectives 命中的专家（带反思反馈，最多 1 次）
    → 否：继续
→ Phase 3.7 Debate → Phase 4 综合（含 confidence）
```

### 2.2 建议二：工具结果校验 + 幻觉前置阻断 + 合规过滤

#### 现状
- `execute_tool`（[tools/__init__.py:712](file:///Users/xiaoyuer/projects/investment-analyzer/backend/tools/__init__.py)）返回结果不校验数值合理性
- 幻觉检测 `verify_fund_codes_in_response` 事后异步，用户已看到错误
- 无合规过滤（保本/稳赚等违规表述）

#### 设计

##### A. 工具结果校验（[tools/__init__.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/tools/__init__.py)）
- 新增 `_validate_tool_result(name, result_str) -> (ok, msg)` 函数
- 校验规则（轻量、零 LLM 成本）：
  - `query_valuation`/`get_valuation_list`：PE 范围 0-500、PB 范围 0-100、percentile 0-1
  - 基金代码 6 位数字格式
  - 空结果检测（返回 `{"error": ...}` 或空 data）
- execute_tool 出口调用：异常时标记 `result = {"warning": msg, "data": ...}`，并触发 1 次重试
- 重试仍异常则保留 warning 标记返回（不阻断，让 LLM 看到 warning 自行判断）
- 新增配置 `tools.validate_result_enabled`（默认 true）

##### B. 合规过滤（[pipeline.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/pipeline.py)）
- 新增 `_COMPLIANCE_BLOCKLIST` 正则数组：保本/稳赚/ guaranteed /无风险/零风险/必涨/稳赢
- Phase 4 综合答案生成后、`EVENT_ANSWER` 前同步扫描
- 命中则：在答案末尾追加合规风险提示 + 记录日志
- 新增配置 `pipeline.compliance_filter_enabled`（默认 true）

##### C. 幻觉前置标注（[pipeline.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/pipeline.py)）
- Phase 4 综合答案后同步调 `quick_check_fund_codes(answer)`（已有函数，纯正则，零成本）
- 若提取到基金代码：在答案末尾追加"⚠️ 本回复含基金代码，请以工具验证结果为准"提示
- 完整 verify 保持事后异步（不阻塞主流程）
- 新增配置 `pipeline.fund_code_warning_enabled`（默认 true）

#### 改造点
- `tools/__init__.py`：新增 `_validate_tool_result` + execute_tool 出口调用 + 重试
- `pipeline.py`：`_phase_synthesis` 返回前加合规扫描 + 基金代码提示
- `db/config.py`：注册 3 个新配置项
