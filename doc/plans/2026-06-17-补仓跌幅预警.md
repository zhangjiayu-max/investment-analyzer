# 补仓后跌幅预警 — 实现计划

## Context

用户希望在每次补仓后，如果基金净值下跌超过买入价的 4%（可配置），系统自动生成预警。这帮助用户识别连续下跌的基金，也为后续对话提供重要上下文。

当前系统已有 5 类巡检预警（估值/回撤/集中度/现金/数据过期），但缺少"相对买入价的跌幅"这一维度。现有的回撤预警是对比 30 天高点，与用户需求不同。

## 方案概述

在 `portfolio_holdings` 表新增 `last_buy_price` 和 `last_buy_date` 两个字段，跟踪最近一次买入价格。在现有巡检扫描中新增第 6 类预警，并在净值自动刷新后自动触发扫描。

**无需前端改动** — 现有预警面板 UI 已支持按类型和严重级别展示所有预警。

### 1. `backend/db/__init__.py` — 新增列 + 历史数据回填

在 line 403（`has_base_position` 列迁移）之后添加：

```python

_add_column_if_not_exists(conn, "portfolio_holdings", "last_buy_date", "TEXT")
```

紧接着添加一次性回填 SQL，为已有持仓从交易记录中取最近一次买入价：

```python
conn.execute("""
    UPDATE portfolio_holdings SET
        last_buy_price = (
            SELECT price FROM portfolio_transactions
            WHERE holding_id = portfolio_holdings.id
              AND transaction_type = 'buy' AND status IN ('confirmed','settled') AND price > 0
            ORDER BY id DESC LIMIT 1
        ),
        last_buy_date = (
            SELECT transaction_date FROM portfolio_transactions
            WHERE holding_id = portfolio_holdings.id
              AND transaction_type = 'buy' AND status IN ('confirmed','settled') AND price > 0
            ORDER BY id DESC LIMIT 1
        )
    WHERE last_buy_price IS NULL
""")
```

## 验证方式

1. **数据库迁移**：重启后端，检查 `portfolio_holdings` 表是否新增了 `last_buy_price` 和 `last_buy_date` 列，且已有持仓被正确回填
2. **买入后即时预警**：创建一笔买入交易（确认），如果当前净值已低于买入价 4%+，检查是否生成了 `buy_drop_alert`
3. **巡检扫描**：手动触发 `POST /api/portfolio/alerts/scan`，检查新的补仓跌幅预警是否出现
4. **自动触发**：等待每日 20:00 净值刷新，或手动验证 `_auto_refresh_nav` 逻辑是否正确调用了扫描
5. **前端展示**：在持仓管理页面的"风险预警"面板中确认新预警类型正常显示
