## 二、根因分析

### O-1：portfolio_alerts title「近5日近5日」重复

**调用链**：
- `alert_scanner.scan_valuation_thresholds()` 检测低估 → 调 `daily_position_advisor._calc_trend_score(fund_code, 5)` → 返回 `(score, reason)`，其中 `reason` 已含「近5日 ...」前缀（如 `"近5日 +1.36%，止跌回升"`）
- `alert_scanner.py:250/253/256` 又拼接 `f"（近5日{tr}，上车信号强）"` → 最终 `trend_hint = "（近5日近5日 +1.36%，止跌回升，上车信号强）"`

**根因**：alert_scanner 拼接时重复加了「近5日」前缀，而 `_calc_trend_score` 返回的 `tr` 已经自带前缀。

**影响范围**：SQL 查询显示 76 条 valuation_low alert 受影响（[id 354-644] 区间）。

**实测数据**：5 条最新事件 expected_impact_pct 全 None。

**根因**：`event_radar.scan_forward_events()` 中：

### O-6：market_events direction 全 neutral

**实测数据**：5 条最新事件 direction 全 neutral。

**根因**：
1. LLM 提取 direction 时倾向给 neutral（保守）
2. `_calibrate_direction()` 只能将高错误率板块方向压成 neutral，**不会反向提升**（即 LLM 给 neutral 时永远保持 neutral）
3. 没有基于事件标题/摘要关键词推断方向的兜底逻辑

**影响范围**：所有 direction=neutral 的事件（占绝大多数）。

### O-2：market_events sources backfill（重新过滤历史事件）

**新增函数**：`backend/services/market/event_radar.py`

```python
def backfill_event_sources(max_events: int = 100) -> dict:
    """对历史 market_events 重新跑 _filter_sources_by_relevance。

    场景：事件 42/41 等创建于 P0-E 修复之前，sources 用旧 news[:3] 生成，
    与事件主题无关。本函数重新采集最近 24h 新闻并按主题过滤，覆盖旧 sources。

    Args:
        max_events: 最多 backfill 多少条（按 rowid DESC 倒序）
    Returns:
        {"processed": N, "updated": M, "skipped": K}
    """
    from db.market_events import list_market_events, update_market_event_fields
    events = list_market_events(limit=max_events)
    # 重新采集最近 24h 新闻（用于过滤）
    all_news = _collect_news()
    updated = 0
    for ev in events:
        new_sources = _filter_sources_by_relevance(ev, all_news, max_per_event=3)
        if new_sources:
            try:
                update_market_event_fields(ev["event_id"], {"sources": new_sources})
                updated += 1
            except Exception as e:
                logger.warning(f"[event_radar] backfill sources 失败 {ev['event_id']}: {e}")
    return {"processed": len(events), "updated": updated, "skipped": len(events) - updated}
```

**调用时机**：
1. 后端启动时自动调用一次（受开关控制）
2. 提供 API `/api/alerts/event-radar/backfill-sources` 手动触发

### O-4：watchlist current_percentile fallback

**修改文件**：`backend/services/advisor/watchlist_refresher.py`（或类似文件）

**修改点**：当 watchlist 项无 `index_code` 时，使用 `fund_code` 关联 `fund_metadata.tracking_index` 获取指数代码；若仍无，则用基金净值回撤估算「相对分位」。

```python
def _refresh_watchlist_percentile(item):
    fund_code = item["fund_code"]
    index_code = item.get("index_code")
    if not index_code:
        # fallback 1: 从 fund_metadata 查 tracking_index
        fm = lookup_fund_info(fund_code)
        index_code = fm.get("tracking_index") if fm else None
    if index_code:
        val = get_best_valuation(index_code, ...)
        if val:
            return val.get("percentile")
    # fallback 2: 用基金净值回撤估算
    nav_hist = get_fund_nav_history_from_cache(fund_code, days=365)
    if nav_hist and len(nav_hist) > 30:
        current = nav_hist[-1]["nav"]
        max_nav = max(n["nav"] for n in nav_hist)
        drawdown = (current - max_nav) / max_nav
        # 简单线性映射：drawdown 0% → 100% 分位；drawdown -50% → 0% 分位
        percentile = max(0, min(100, 100 + drawdown * 200))
        return round(percentile, 2)
    return None
```

**开关控制**：`watchlist.fallback_percentile_enabled` 默认 true。

### O-6：market_events direction 推断兜底

**新增函数**：`backend/services/market/event_radar.py`

```python
def _infer_direction_by_keywords(event: dict) -> str:
    """基于事件标题/摘要关键词推断方向（LLM 给 neutral 时的兜底）。

    Returns:
        "positive" / "negative" / "neutral"
    """
    text = (event.get("title", "") + " " + event.get("summary", "")).lower()
    positive_kw = ["利好", "支持", "出台", "发布", "增长", "上升", "上涨",
                   "批准", "获批", "落地", "推进", "加速", "突破"]
    negative_kw = ["利空", "下跌", "下滑", "风险", "警告", "违约", "退市",
                  "亏损", "减持", "解禁", "处罚", "不及预期", "下调"]
    pos_hits = sum(1 for kw in positive_kw if kw in text)
    neg_hits = sum(1 for kw in negative_kw if kw in text)
    if pos_hits > neg_hits:
        return "positive"
    if neg_hits > pos_hits:
        return "negative"
    return "neutral"
```

**集成点**：`scan_forward_events()` 在 `_calibrate_direction` 之后调用：
```python
calibrated_dir, _ = _calibrate_direction(ev.get("direction", "neutral"), ev.get("affected_sectors", []))
# O-6: LLM 给 neutral 时，基于关键词推断兜底
if calibrated_dir == "neutral":
    calibrated_dir = _infer_direction_by_keywords(ev)
```

**开关控制**：`alerts.event_direction_infer_enabled` 默认 true。

**历史数据 backfill**：`backfill_event_direction()` 对所有 direction=neutral 的事件重新跑关键词推断。

### O-7：market_events confidence 动态调整

**修改函数**：`_calibrate_confidence(original_confidence, sectors)` → `_calibrate_confidence_v2(original_confidence, sectors, event, sources_count, matched_count)`

```python
def _calibrate_confidence_v2(original, sectors, event, sources_count=0, matched_count=0):
    conf = original
    # 1. 基础校准（保留原逻辑）
    # ... 原有板块匹配加成 ...
    
    # 2. O-7 新增：基于数据质量动态调整
    # sources 数量：0 → -0.15, 1-2 → -0.05, 3+ → +0
    if sources_count == 0:
        conf -= 0.15
    elif sources_count <= 2:
        conf -= 0.05
    
    # matched_holdings 数量：0 → -0.1（与用户无关），1+ → +0.05
    if matched_count == 0:
        conf -= 0.1
    else:
        conf += 0.05
    
    # 3. 时效性：detected_date > 7天 → -0.1
    detected = event.get("detected_date", "")
    if detected:
        try:
            dt = datetime.strptime(detected[:10], "%Y-%m-%d")
            age_days = (datetime.now() - dt).days
            if age_days > 7:
                conf -= 0.1
        except Exception:
            pass
    
    # 4. 上限约束：不超过 0.95（避免「100% 把握」误导）
    return max(0.3, min(0.95, conf))
```

**开关控制**：`alerts.confidence_dynamic_adjust_enabled` 默认 true。
