## 后端设计

### 新增端点（在 `routers/capabilities.py` 扩展）

#### 1. GET /api/capabilities/stats — 工具调用统计

**查询参数**：
- `days`: 时间窗口（默认 7，可选 1/7/30/90）
- `tool_name`: 指定工具名（空=全部）

**返回结构**：
```python
{
    "total_calls": 533,
    "total_success": 510,
    "total_failed": 23,
    "overall_success_rate": 0.957,
    "avg_duration_ms": 285.4,
    "by_tool": [
        {
            "tool_name": "search_knowledge",
            "calls": 150,
            "success": 148,
            "failed": 2,
            "success_rate": 0.987,
            "avg_duration_ms": 240.3,
            "max_duration_ms": 5821,
            "p95_duration_ms": 850.0,
            "error_categories": {"none": 148, "tool_error": 2}
        },
        ...
    ],
    "by_day": [  # 近 N 天趋势
        {"date": "2026-07-03", "calls": 80, "success": 78},
        ...
    ],
    "by_error_category": {
        "none": 510, "tool_error": 15, "timeout": 5, "validation_failed": 3
    }
}
```

#### 2. POST /api/capabilities/debug — 工具调试执行

**请求体**：
```json
{
    "tool_name": "query_valuation",
    "arguments": {"index_name": "沪深300"},
    "trace_id": "debug_xxxxxx"  // 可选，自动生成
}
```

**返回**：
```python
{
    "tool_name": "query_valuation",
    "arguments": {"index_name": "沪深300"},
    "result": "{...}",  # 工具返回的 JSON 字符串
    "parsed_result": {...},  # 解析后的 dict（若可解析）
    "duration_ms": 124,
    "success": true,
    "error_category": "none",
    "trace_id": "debug_abc123",
    "audit_id": 3142  # 写入 tool_audit_logs 的 ID
}
```

**安全约束**：
- 仅允许已暴露（TOOLS 数组中）的工具调试
- 超时保护 30s（复用 execute_tool 的 timeout）
- 自动写入 tool_audit_logs（trace_id 前缀 `debug_`）
- 禁用列表：`web_search`（外部请求，避免滥用）可选加入黑名单配置

#### 3. GET /api/capabilities/stats/recent — 最近调用记录

**查询参数**：
- `tool_name`: 指定工具（空=全部）
- `limit`: 默认 20，最大 100
- `success`: 筛选成功/失败（空=全部）

**返回**：最近 N 条调用记录（含参数、结果预览、耗时、时间）

#### Tab 3: 调试台（新增）

**布局**：
```
┌─────────────────────────────────────────────┐

│ 描述: 查询指定指数的估值数据...              │

│ [参数输入区]（根据 parameters schema 自动生成)│

├─────────────────────────────────────────────┤
│ [执行按钮]  ▶ 运行                           │
├─────────────────────────────────────────────┤
│ [结果区]                                     │
│ ┌─ 执行信息 ──────────────────────────────┐ │
│ │ 耗时: 124ms  状态: ✅成功  trace: debug_ │ │
│ ├─ 返回结果 ──────────────────────────────┤ │
│ │ {                                        │ │
│ │   "index_code": "000300",               │ │
│ │   "pe": 12.5,                           │ │
│ │   "percentile": 35.2,                   │ │
│ │   ...                                    │ │
│ │ }                                        │ │
│ └────────────────────────────────────────────┘ │
│ [复制结果] [重新运行]                        │
├─────────────────────────────────────────────┤
│ [历史调试记录] 最近 5 次（本会话内）          │

│ 14:28 search_knowledge(白酒) 240ms ✅       │
└─────────────────────────────────────────────┘
```

**参数表单自动生成**：
- 根据 `parameters.properties` 动态渲染输入框
- `type: string` → 文本输入
- `type: number/integer` → 数字输入
- `type: boolean` → 开关
- `type: array` → 逗号分隔输入（转数组）
- `enum` → 下拉选择
- `required` 字段标记必填

**交互**：
- 选择工具 → 自动加载参数 schema → 渲染表单
- 点击「运行」→ POST /api/capabilities/debug → 展示结果
- 结果区支持 JSON 折叠/展开 + 复制
- 历史记录存在前端 localStorage（仅本浏览器）

## 安全考虑

1. **调试权限**：初版不做鉴权（本地工具），生产环境需加 admin token
2. **频率限制**：调试接口建议 10 次/分钟（防滥用）
3. **黑名单**：`web_search` 等外部请求工具可配置禁用调试
4. **结果截断**：调试结果最大 5000 字符，超出截断并提示
