# 对话质量评估系统设计

## 背景

当前系统已有基础的反馈机制（点赞/点踩 + 专家级反馈），但缺少针对**复杂多Agent对话**的系统化质量评估。需要一个能够自动评估对话质量、并展示评估结果的功能。

## 实现状态

✅ **已完成** (2026-06-12)
- 后端评估逻辑：`backend/agent/conversation_evaluator.py`
- 数据库表：`conversation_evaluations`, `conversation_eval_details`
- API 路由：5个新端点
- 前端组件：`ConversationEvalCard.vue` 集成到 ChatView
- 质量仪表盘：QualityDashboard 增加对话质量 Tab

## 评估维度设计

### 一、自动评估维度（系统自动计算）

#### 1. 执行效率 (Execution Efficiency)
| 指标 | 计算方式 | 权重 |
|------|----------|------|
| 总耗时 | duration_ms | 15% |
| 专家并行度 | 实际并行执行的专家数 / 总专家数 | 10% |
| 重复调用检测 | 相同 agent_key + 相似 query 的调用次数 | 10% (扣分项) |
| Token 效率 | 有效 token / 总 token | 5% |

#### 2. 数据利用 (Data Utilization)
| 指标 | 计算方式 | 权重 |
|------|----------|------|
| RAG 命中率 | 引用知识库的专家数 / 总专家数 | 10% |
| 工具调用成功率 | 成功的工具调用 / 总工具调用 | 10% |
| 持仓数据利用 | 是否引用用户持仓数据 | 5% |
| 估值数据利用 | 是否引用估值数据 | 5% |

#### 3. 专家协作 (Specialist Collaboration)
| 指标 | 计算方式 | 权重 |
|------|----------|------|
| 专家覆盖度 | 实际调用专家数 / 推荐专家数 | 10% |
| 交叉审阅 | 是否触发交叉审阅 | 5% (加分项) |
| 仲裁质量 | 仲裁结果是否采纳了多数专家意见 | 5% |
| 分歧识别 | 是否正确识别专家分歧 | 5% |

#### 4. 响应质量 (Response Quality)
| 指标 | 计算方式 | 权重 |
|------|----------|------|
| 结构化程度 | 是否有清晰的章节结构 | 5% |
| 数据引用 | 是否引用具体数字 | 5% |
| 风险提示 | 是否包含风险提示 | 5% |
| 可操作性 | 是否给出具体建议 | 5% |

### 二、用户评估维度（用户打分）

复用现有的 QualityFeedback 组件，支持 5 星评分：
- **数据准确性**：数据是否准确、来源是否清晰
- **逻辑一致性**：分析逻辑是否自洽、结论是否合理
- **可执行性**：建议是否具体、是否可操作
- **整体满意度**：对本次对话的整体评价

## 数据模型

### 新增表：conversation_evaluations

```sql
CREATE TABLE IF NOT EXISTS conversation_evaluations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    conversation_id INTEGER NOT NULL,
    message_id INTEGER,
    
    -- 自动评估分数（0-100）
    auto_score REAL,
    auto_score_breakdown TEXT,  -- JSON: {"execution": 85, "data": 90, "collaboration": 80, "response": 88}
    
    -- 用户评估分数（0-5）
    user_score REAL,
    user_score_breakdown TEXT,  -- JSON: {"accuracy": 4, "logic": 5, "actionability": 4}
    user_comment TEXT,
    
    -- 评估元数据
    complexity TEXT,
    specialist_count INTEGER,
    duration_ms INTEGER,
    has_cross_review INTEGER DEFAULT 0,
    has_arbitration INTEGER DEFAULT 0,
    duplicate_calls INTEGER DEFAULT 0,
    
    created_at TEXT DEFAULT (datetime('now','localtime')),
    updated_at TEXT DEFAULT (datetime('now','localtime'))
);
```

### 新增表：conversation_eval_details

```sql
CREATE TABLE IF NOT EXISTS conversation_eval_details (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    eval_id INTEGER REFERENCES conversation_evaluations(id),
    dimension TEXT NOT NULL,  -- execution/data/collaboration/response
    metric TEXT NOT NULL,
    value REAL,
    detail TEXT,
    created_at TEXT DEFAULT (datetime('now','localtime'))
);
```

## API 设计

### 1. 自动评估对话
```
POST /api/eval/conversation/{conversation_id}
返回: { ok: true, evaluation: {...} }
```

### 2. 获取对话评估结果
```
GET /api/eval/conversation/{conversation_id}
返回: { ok: true, evaluation: {...} }
```

### 3. 提交用户评分
```
POST /api/eval/conversation/{conversation_id}/user-score
Body: { score: 4.5, breakdown: {...}, comment: "..." }
返回: { ok: true }
```

### 4. 获取评估统计
```
GET /api/eval/conversation-stats
返回: { ok: true, stats: {...} }
```
