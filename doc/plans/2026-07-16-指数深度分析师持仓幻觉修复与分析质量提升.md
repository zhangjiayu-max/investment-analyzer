# 指数深度分析师持仓幻觉修复 与 分析质量提升设计稿

> 日期：2026-07-16
> 触发：用户反馈"指数深度分析师"分析电力指数时，把关注列表基金误当持仓，出现幻觉
> 范围：①修复持仓幻觉（P0，用户明确要求"要修复"）②分析质量提升（P1，用户要求"如何提升"）

---

## 一、问题诊断

### 1.1 持仓幻觉根因（已确认 3 个）

**根因 A — Prompt "禁止否定"规则强制幻觉**（[db/analysis.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/db/analysis.py) L824-829）

```
# ⚠️ 重要规则
1. 必须先检查 <user_portfolio> 标签是否存在。如果有该标签，说明用户持有相关基金，你必须在分析中引用这些持仓数据。
2. 绝对不能说"用户未持仓"，除非 <user_portfolio> 标签确实不存在。
3. 投资建议必须基于实际持仓状态：有持仓就给出持有/加仓/减仓建议，无持仓才给建仓/观望建议。
```

问题：规则 2 用"绝对不能"强行禁止 LLM 如实说明未持仓。当 `<user_portfolio>` 标签为空但 `## 组合约束` JSON 块里的 `recent_decisions` / `market.indices` 含有基金/指数名称时，LLM 被迫把这些"当成持仓"来引用，直接制造幻觉。

**根因 B — facts_block 注入无字段语义标注**（[index_analysis.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/routers/analysis/index_analysis.py) L213-214）

```python
if facts_block:
    full_prompt += f"\n\n## 组合约束（系统注入，优先级最高）\n```json\n{facts_block}\n```"
```

`build_portfolio_facts()` 返回的 facts_block 含 6 个字段，语义混杂：
| 字段 | 语义 | 是否持仓 |
|------|------|----------|
| snapshot | 持仓快照（total_value/bond_pct/overconcentrated） | ✅ 真实持仓 |
| constraints | 约束规则（债市温度/禁加目标） | ❌ 规则 |
| market.indices | 主要指数估值摘要 | ❌ 市场数据 |
| recent_analyses | 最近 24h 分析记录 | ❌ 历史分析 |
| market_state | 市场状态 regime/sentiment | ❌ 市场数据 |
| recent_decisions.last_3_days | 最近 3 天决策（含 target_subject 基金名） | ❌ 历史决策 |

LLM 看到 `recent_decisions.last_3_days` 里的 `target_subject`（基金名称）和 `market.indices` 里的 `index_name`，误判为"用户持有这些基金"。JSON 一锅炖，没有任何字段说明哪些是持仓、哪些是参考。

**根因 C — fuzzy 匹配空 keywords 边界 bug**（[index_analysis.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/routers/analysis/index_analysis.py) L125-128）

```python
keywords = req.index_name.replace("指数", "").strip()
fuzzy_matched = [h for h in all_holdings
                 if h.get("fund_name") and keywords in h["fund_name"]
                 and h not in matched]
```

当 `req.index_name` 为"指数"或"XX指数"被 replace 后变空字符串时，`"" in h["fund_name"]` 恒为 True，会匹配所有持仓。虽然不会把关注列表拉进来（list_holdings 只查 portfolio_holdings），但会把无关持仓错误归入"该指数相关基金"，制造另一种幻觉。

### 1.2 最近 10 条分析记录质量评估结果

| 维度 | 现状 | 问题 |
|------|------|------|
| 交叉审阅触发率 | 0/10 = 0% | should_run_cross_review 门槛过高/配置未开 |
| 专家数量 | 1-2 个/条，平均 1.5 | 复杂问题专家不足（目标 ≥3） |
| 报告格式 | 40% 不统一 | 3/10 内嵌 ```json 代码块 |
| 绝对化措辞 | 40%（"一定/必然/绝对"） | prompt 约束缺失 + 无后处理 |
| 评估系统触发 | has_eval 0/12 = 0% | conversation_evaluator 未接入 |
| agent 体系对齐 | analysis_agents.agent_key 全空 | 与 metadata 中 agent_key 脱节 |

## 二、修复方案

### P0-A：修复持仓幻觉（3 处改动）

#### A1. 重写 Prompt 规则（[db/analysis.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/db/analysis.py) L824-829）

把"绝对不能说未持仓"改为"只以 `<user_portfolio>` 标签为唯一持仓真相源"：

```
# ⚠️ 持仓数据真实性规则（最高优先级）

1. **唯一真相源**：用户是否持有该指数相关基金，**只以 <user_portfolio> 标签为准**。
   - 标签存在且有内容 → 用户持有，按持仓数据给建议
   - 标签不存在或为空 → 明确说明"当前未持有该指数相关基金"，给建仓/观望建议
2. **禁止幻觉持仓**：不得将 `## 组合约束` JSON 中的 recent_decisions、market.indices、
   recent_analyses 等字段误认为用户持仓。这些字段只是历史决策/市场参考/历史分析，
   不代表用户当前持有。
3. **禁止编造**：不得虚构基金代码、市值、盈亏数字。所有持仓数字必须来自 <user_portfolio> 标签。
```

同步修改 L851-859"持仓关联分析"和 L882-883"我的持仓"章节描述，与新规则对齐。

#### A2. facts_block 注入加字段语义标注（[index_analysis.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/routers/analysis/index_analysis.py) L213-214）

不再整块 JSON 注入，改为按字段语义分段标注，明确区分"持仓"vs"参考数据"：

```python
if facts_block:
    try:
        facts = json.loads(facts_block)
        # 只注入真实持仓相关字段，剔除易混淆的市场/历史字段
        portfolio_relevant = {
            "持仓快照": facts.get("snapshot", {}),
            "组合约束": facts.get("constraints", {}),
        }
        # 市场参考数据单独标注，明确不是持仓
        market_ref = {
            "市场指数估值参考": facts.get("market", {}).get("indices", []),
            "市场状态": facts.get("market_state", {}),
        }
        full_prompt += (
            f"\n\n## 组合约束（系统注入，仅约束加仓红线）\n```json\n"
            f"{json.dumps(portfolio_relevant, ensure_ascii=False, indent=2, default=str)}\n```"
            f"\n\n## 市场参考数据（仅作参考，**不是用户持仓**，禁止当作持仓引用）\n```json\n"
            f"{json.dumps(market_ref, ensure_ascii=False, indent=2, default=str)}\n```"
        )
        # recent_decisions / recent_analyses 不注入（历史数据易致幻觉）
    except Exception:
        # 降级：原整块注入但加显著警告
        full_prompt += f"\n\n## 组合约束（系统注入）\n```json\n{facts_block}\n```\n⚠️ 注意：以下 JSON 中的 recent_decisions/market 字段不是用户持仓，禁止当作持仓引用。"
```

关键点：
- 剔除 `recent_decisions` 和 `recent_analyses`（历史数据，对单指数深度分析无价值且致幻）
- `market.indices` 保留但显式标注"不是用户持仓"
- 持仓快照和组合约束作为"加仓红线约束"保留

#### A3. 修复 fuzzy 匹配空 keywords 边界 bug（[index_analysis.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/routers/analysis/index_analysis.py) L125-128）

```python
if req.index_name:
    keywords = req.index_name.replace("指数", "").strip()
    if keywords:  # 修复：空 keywords 会匹配所有持仓
        fuzzy_matched = [h for h in all_holdings
                         if h.get("fund_name") and keywords in h["fund_name"]
                         and h not in matched]
        matched.extend(fuzzy_matched)
```

### P1-B：分析质量提升（4 处改动）

#### B1. 降低交叉审阅触发门槛 + 默认开启（[orchestrator.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/core/orchestrator.py) should_run_cross_review）

当前 `should_run_cross_review` 要求 min_specialists=2 且冲突 severity≥medium，导致 0/10 触发。调整：
- `cross_review_min_specialists` 默认从 2 → 1（只要有专家就审）
- 复杂度 complex/medium 时强制交叉审阅（不再仅依赖冲突检测）
- 检查 `agent.cross_review_enabled` 配置默认值是否为 true

#### B2. 强制复杂度→专家数量配额（[orchestrator.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/core/orchestrator.py) 路由阶段）

- complex ≥ 3 个专家
- medium ≥ 2 个专家
- 在路由后检查专家数不足时自动补一个相关专家

#### B3. 统一最终报告为纯 markdown（[orchestrator.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/core/orchestrator.py) _build_final_synthesis_prompt）

在综合 prompt 中明确：
- "最终输出必须是纯 Markdown 叙述式报告，禁止使用 ```json 代码块"
- "如需结构化数据，用 Markdown 表格或列表呈现"

#### B4. 绝对化措辞后处理过滤（[orchestrator.py](file:///Users/xiaoyuer/projects/investment-analyzer/backend/agent/core/orchestrator.py) 最终输出后）

在最终回答生成后加正则后处理（可选开关 `agent.absolutism_filter_enabled` 默认 true）：
```python
import re
ABSOLUTISM_MAP = {
    "一定会": "大概率会",
    "必然": "预计",
    "绝对": "较可能",
    "肯定会": "可能会",
    "肯定会涨": "大概率上涨",
    "肯定会跌": "大概率下跌",
}
def _filter_absolutism(text: str) -> str:
    for k, v in ABSOLUTISM_MAP.items():
        text = text.replace(k, v)
    return text
```

注：仅过滤"投资结论"类绝对化措辞，保留"必须遵守红线"等规则性表述。为避免误伤，只在最终输出末尾段落（投资建议部分）做替换，或整文替换但加白名单。
