## 一、问题诊断

### 1.1 现状数据

| 维度 | 数据 | 评价 |
|---|---|---|
| 蒸馏知识总量 | 8,726 条（36 本投资书 + 4 个技术笔记） | 充足 |
| 对话系统 RAG 覆盖率 | 92.3%（60/65 对话触发检索） | 良好 |
| RAG 总命中率 | 96.58%（7 月达 100%） | 良好 |
| **分析任务 RAG 引用率** | **2.2%~3.4%** | **严重不足** |
| **diversification 路由** | **死代码**（导入未调用） | **断层** |
| **宏观策略师** | **20 次运行 0 调用 search_knowledge** | **引导失效** |

| 持仓基金知识盲区 | 6 只主动基金 + 6 只指数基金无知识 | 盲区 |

### 1.2 根因分类

**断层 A — 双轨制架构断层**（最严重）：
- 对话专家走 `run_specialist()`（multi_agent.py:522），有 tools + ReAct + 主动检索指令
- 分析专家走路由直调 `_call_llm()`（无 tools 参数），prompt 纯文本拼装
- analysis_agents 表 12 个专家 tools 字段全空，且路由代码不传 tools → LLM 无法 function calling

**断层 B — 路由级 RAG 漏写**：
- 5 个路由手动写了 `build_rag_context_with_details()`（panorama/portfolio_ai/deep_dive/index_analysis/market_intel）
- 1 个路由漏写（diversification.py:25 仅 import 未调用）

**断层 C — Prompt 引导冲突**：
- 宏观策略师 prompt 强引导 yingmi_search_news，挤占 search_knowledge 轮次预算
- 检索类工具共享 `_SEARCH_TOOLS` 计数（multi_agent.py:606-609），轮次耗尽后无法再调

**断层 D — 查询预处理缺失**：
- 无 URL 剥离（mp.weixin 类查询 FTS 失效）
- 无基金名/代码识别（基金名被 jieba 切碎）
- jieba 无自定义词典

**断层 E — 索引污染与时效过严**：
- 579 条 AI/LLM 技术笔记（note）与投资无关，污染检索结果
- 时效过滤对"历史/规律"类查询过严（3 个月清空书籍知识）

## 二、修复方案

### 设计原则
1. **独立开关控制**：所有新功能默认关闭，符合硬约束
2. **最小侵入**：优先复用已有机制（run_agent_analysis、_ACTIVE_RETRIEVAL_INSTRUCTION）
3. **分层修复**：断层 A/B/C 是"知识不被引用"的根因，优先级最高；D/E 是"检索质量"优化，次之
4. **不新增表**：复用现有 analysis_agents、fund_metadata、knowledge_base 表

### R-2: 分析路由 RAG 调用统一化（核心修复）

**公共函数**：
```python
# backend/routers/analysis/_common.py
from services.rag import build_rag_context_with_details, log_rag_search
import logging

logger = logging.getLogger(__name__)

def inject_rag_context(
    base_query: str,
    extra_keywords: str = "",
    limit: int = 5,
    max_chars: int = 1500,
    caller: str = "",
) -> str:
    """分析路由统一 RAG 注入入口。
    
    Args:
        base_query: 基础检索词（如"持仓分散度 集中度"）
        extra_keywords: 附加关键词（如基金名、指数名）
        limit: 最大返回条数
        max_chars: 注入上下文最大字符数
        caller: 调用方标识，用于日志
    Returns:
        RAG 上下文文本（含 ## 知识库参考 标题），空则返回空字符串
    """
    try:
        rag_query = f"{base_query} {extra_keywords}".strip()
        rag_result = build_rag_context_with_details(query=rag_query, limit=limit)
        context = rag_result.get("context", "")
        if context:
            log_rag_search(
                conversation_id=0, message_id=0, query=rag_query,
                keywords=rag_result.get("keywords", []),
                results=rag_result.get("results", []),
                fts_count=rag_result.get("fts_count", 0),
                chroma_count=rag_result.get("chroma_count", 0),
                freshness_filtered=rag_result.get("freshness_filtered", 0),
            )
            return f"\n\n## 知识库参考\n{context[:max_chars]}"
        return ""
    except Exception as e:
        logger.warning(f"[{caller}] RAG 检索失败: {e}")
        return ""
```

**6 个路由改造**：
| 路由 | 当前状态 | 改造方式 |
|---|---|---|
| panorama.py | 手写调用 | 替换为 `inject_rag_context("全景诊断 持仓健康", fund_names, caller="panorama")` |
| portfolio_ai.py | 手写调用 | 替换为 `inject_rag_context("组合分析 基金投资", fund_names, caller="portfolio_ai")` |
| deep_dive.py | 手写调用 | 替换为 `inject_rag_context("基金深度分析", fund_name, caller="deep_dive")` |
| index_analysis.py | 手写调用 | 替换为 `inject_rag_context("指数分析 估值", index_name, caller="index_analysis")` |
| market_intel.py | 手写调用 | 替换为 `inject_rag_context("市场情报 行业分析", "", caller="market_intel")` |

#### R-4-1: URL 剥离
在 `build_rag_context_with_details()` 入口（rag.py:2029）增加：
```python
import re
_URL_PATTERN = re.compile(r'https?://[^\s\u4e00-\u9fff]+')

def _strip_url_from_query(query: str) -> tuple[str, str | None]:
    """剥离 URL，返回 (清洗后query, url或None)。"""
    url_match = _URL_PATTERN.search(query)
    if not url_match:
        return query, None
    url = url_match.group(0)
    cleaned = _URL_PATTERN.sub('', query).strip()
    return cleaned, url
```
入口处：`query, url = _strip_url_from_query(query)`，若 url 存在则走文章解读流程而非标准检索。

#### R-4-2: 基金名识别 + 映射
新增 `_detect_fund_names()` 函数（rag.py，参照 `_detect_index_names()`）：

def _detect_fund_names(query: str) -> list[dict]:
    """识别查询中的基金名，返回 [{fund_name, fund_code, tracking_index}]。"""
    from db.fund_metadata import list_fund_metadata
    funds = list_fund_metadata()  # 21 条 fund_metadata

_jieba_initialized = False

def _get_jieba():
    global _jieba_initialized
    import jieba
    if not _jieba_initialized:
        # 加载基金名

            from db.fund_metadata import list_fund_metadata
