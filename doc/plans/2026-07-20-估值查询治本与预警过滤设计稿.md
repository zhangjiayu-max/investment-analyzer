## 二、设计目标

1. **用户感知**：右上角铃铛不再出现"估值数据缺失"类预警
2. **数据完整性**：持仓中的指数能查到本地或在线估值（任意 metric_type 都算"已查到"）
3. **去重正确**：同指数 24 小时内不重复生成 alert
4. **不破坏现有逻辑**：所有改动通过开关或可选参数控制，默认行为兼容

---

## 三、修复方案（5 个改动）

### 改动 1：`scan_valuation_failures` 改为只写日志不创建 alert（用户直接诉求）

**位置**：[alert_scanner.py:944-1007](file:///Users/xiaoyuer/projects/investment-analyzer/backend/services/advisor/alert_scanner.py#L944)

**改动**：
- 函数内不再调用 `create_alert()`
- 改为 `logger.warning()` 写日志，便于开发侧排查
- 保留 `alerts.valuation_failure_scan_enabled` 开关（默认 true，仅控制日志）

**理由**：用户明确说"估值没查到就别预警提醒了"。开发侧监控不应该暴露给终端用户。

```python
# 改动前
create_alert(
    alert_type="valuation_query_failed",
    title=f"估值数据缺失：{name}",
    ...
)

# 改动后
logger.warning(f"[alert_scanner] 估值查询失败 {code} ({name})：{content}")
```

### 改动 2：`get_best_valuation` 增加 metric_type 自动 fallback

**位置**：[valuations.py:546-680](file:///Users/xiaoyuer/projects/investment-analyzer/backend/db/valuations.py#L546)

**改动**：新增参数 `allow_metric_fallback: bool = False`，默认 False（不破坏现有调用）

- 当指定 metric_type 本地查不到时，按预设顺序自动尝试其他 metric_type
- fallback 顺序：市盈率 → 市净率 → 市销率 → 股息率
- 返回结果中新增 `fallback_metric_type` 字段，标注实际命中的指标类型

**调用方改造**：
- `scan_valuation_thresholds`（alert_scanner.py:181）调用时传 `allow_metric_fallback=True`
- `query_valuation` 工具（专家调用）可选传 True
- 其他批量场景保持 False（避免误触发在线兜底）

```python
def get_best_valuation(
    index_code: str,
    metric_type: str = "市盈率",
    query_source: str = "unknown",
    trace_id: str = None,
    enable_online: bool = True,
    allow_metric_fallback: bool = False,  # 新增
) -> dict | None:
    ...
    # 1-3 阶段：本地查询
    detailed = _try_local_valuation(index_code, metric_type, ...)
    if not detailed and allow_metric_fallback:
        # 自动 fallback：尝试本地其他 metric_type
        for fallback_type in ["市净率", "市销率", "股息率"]:
            if fallback_type == metric_type:
                continue
            fallback_data = _try_local_valuation(index_code, fallback_type, ...)
            if fallback_data:
                fallback_data["fallback_metric_type"] = fallback_type
                return fallback_data
    # 4 阶段：在线兜底（保持原逻辑）
    ...
```

### 改动 4：扩展 `get_index_valuation` 对国证/Wind/港股代码的支持

**位置**：[market_data.py:251-318](file:///Users/xiaoyuer/projects/investment-analyzer/backend/services/market/market_data.py#L251)

**改动**：在 `ak.stock_zh_index_value_csindex` 失败后，新增 fallback 渠道：
- 国证 H 系（H30094/H30217/H30590）→ 用 `ak.stock_zh_index_value_em`（东方财富）
- Wind 88xxxx（882011）→ 同上
- 港股 HSTECH/HSI/HSCEI → 用 `ak.stock_hk_index_daily_em` 或保持 ttfund 兜底

**实现**：
```python
def get_index_valuation(index_code: str = "000300") -> dict:
    result = {"index_name": ..., "pe": None, ...}
    
    # 渠道1：中证官方（原逻辑，仅对 000xxx/399xxx/931xxx 有效）
    if index_code.startswith(("000", "399", "931")):
        result = _query_csindex_valuation(index_code, result)
    
    # 渠道2（新增）：东方财富（对国证 H 系、Wind 88xxxx 兜底）
    if result["pe"] is None and result["pb"] is None:
        result = _query_em_valuation(index_code, result)
    
    # 渠道3（新增）：港股指数
    if index_code.startswith(("H", "HSI", "HSCEI")):
        result = _query_hk_index_valuation(index_code, result)
    
    return result
```

> **注意**：渠道2/3 实现时需验证 akshare 接口可用性。若 akshare 不支持，则保留 ttfund 兜底（见改动 5）。

## 七、风险评估

| 风险 | 影响 | 缓解 |
|---|---|---|
| `allow_metric_fallback=True` 误触发在线兜底导致批量超时 | 智能补仓等批量场景变慢 | 默认 False，仅 alert_scanner 传 True |
| 东方财富接口变更 | 改动 4 渠道2 失效 | 加 try-except，失败则保留 ttfund 兜底 |
| `_CODE_NAME_MAP` 名称与 ttfund 实际名称不一致 | ttfund 名称重试失败 | 实现时逐一验证 ttfund 返回 |
| 清理旧预警后用户看不到历史问题 | 用户可能再次反馈 | 仅删 `valuation_query_failed` 类型，保留其他预警 |

## 八、回滚方案

所有改动通过开关或可选参数控制：
- `alerts.valuation_failure_scan_enabled=false`：关闭失败扫描
- `allow_metric_fallback=False`：恢复原 metric_type 行为
- `_CODE_NAME_MAP` 扩展不影响原 7 个代码
- 表名修复纯 bug fix，无需回滚

如需回滚，`git revert` 对应 commit 即可。
