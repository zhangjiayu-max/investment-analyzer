# 系统级功能闭环与导航重组设计稿

> 日期：2026-07-11
> 目标：系统性排查所有功能模块，优化闭环联动、导航结构、移动端同步、未用功能激活、模块说明标准化、样式优化

## 一、当前问题诊断

### 1.1 功能闭环断点

| 闭环 | 现状 | 断点 |
|------|------|------|
| 持仓管理 | CRUD→交易→AI分析→风险预警 | ✅ 较完整 |
| 决策闭环 | 对话→建议→决策→执行→复盘 | ⚠️ 建议采纳率低，复盘结果未反哺画像 |
| 机会雷达 | 发现→跟踪→上车信号→落地验证 | ⚠️ 候选基金→关注→买入→持仓链路刚打通，买入后未自动标记"已上车" |
| 质量评估 | 反馈→BadCase→Eval→Prompt优化 | ⚠️ Shadow Mode 未启用，进化闭环未转起来 |

### 1.2 高价值未用功能（10项）

| 功能 | 位置 | 价值 | 现状 |
|------|------|------|------|
| skill_document.md | backend/ | 114篇文章蒸馏的投资框架 | ⚠️ 未确认是否注入RAG |
| 机构动向 | services/institutional_flow.py | 北向资金/龙虎榜辅助信号 | ⚠️ 仅辅助标记，未驱动决策 |
| Shadow Mode | ShadowModePage.vue | Prompt 变更对比验证 | ⚠️ 页面存在但可能未启用 |

| 滚动收益分析 | api/index.js | 滚动年化收益分析 | ⚠️ 前端入口不明 |
| 情景推演 | runWhatIf | "如果加仓X万"推演 | ⚠️ 使用场景不明 |
| 对比分析 | runCompareDiff | 基金/指数对比 | ⚠️ 入口不明 |

### 1.3 导航结构问题

当前 Sidebar 有 35 个导航项 + 5 个分组，存在以下问题：
- **持仓相关功能分散**：持仓管理、智能补仓、风险与提示分属不同位置
- **市场类功能分散**：市场热点、事件雷达、债市分析分属不同位置
- **家庭财务缺失分组**：财务总览、资金桶藏在"理财决策"分组里
- **导航项无说明**：用户不清楚每个模块有什么用
- **分组命名不直观**："实验室""进化系统"不够明确

### 1.4 移动端缺失

移动端缺失 8 个页面：smart-add、allocation-dashboard、attribution、accuracy、strategy-sandbox、strategy-backtest、data-health、search

### 1.5 其他问题
- AppToast.vue 未挂载
- 双端默认页不一致（Web=articles，Mobile=chat）
- 移动端无 TickerBar、KYC入口、Token预算指示器

## 二、设计方案

### 2.1 左侧功能栏重组（需求3）

**当前结构**（5分组 + 扁平项，35项）：
```
每日看板 / 市场热点 / 事件雷达 / AI对话
理财决策(8子项)
文章管理 / 估值数据 / 估值图片 / 持仓管理 / 智能补仓 / 风险与提示
知识库(5子项) / 债券分析(1子项)
数据健康 / Agent管理 / Token用量 / 系统配置
进化系统(3子项) / 实验室(4子项)
```

**新结构**（8分组，35项，层级更清晰）：
```
每日看板

📊 市场雷达（新增分组）
  ├─ 市场热点
  ├─ 机会雷达（事件雷达+关注列表+落地验证整合，重命名）
  └─ 债市分析

💬 AI 对话

💰 持仓管理（新增分组）
  ├─ 持仓管理
  ├─ 智能补仓
  └─ 风险与提示

📋 理财决策
  ├─ 决策档案
  ├─ 收益归因
  ├─ 行为诊断
  ├─ 决策准确率
  ├─ 配置偏离
  └─ 策略沙盒

🏠 家庭财务（新增分组）
  ├─ 财务总览
  └─ 资金桶

📚 知识中心（重命名）
  ├─ 文章管理
  ├─ 估值数据
  ├─ 估值图片
  ├─ 蒸馏知识
  ├─ 作者文章
  ├─ 个人文档
  └─ RAG 分析

⚙️ 系统与进化（合并）
  ├─ Agent管理
  ├─ Token用量
  ├─ 系统配置
  ├─ 数据健康
  ├─ 质量仪表盘
  ├─ Bad Case
  ├─ 评测集
  ├─ 健康分
  ├─ Shadow Mode
  ├─ 策略回测
  └─ 能力中心
```

**核心变化：**
1. 新增「市场雷达」「持仓管理」「家庭财务」3个分组
2. 「事件雷达」重命名为「机会雷达」（整合事件+关注+验证）
3. 「知识中心」合并原文章+估值+知识库+RAG
4. 「系统与进化」合并管理+进化+实验室
5. 移除「债券分析」单独分组（移入市场雷达）
6. RAG 命中测试合并到 RAG 分析页面（减少导航项）

### 2.2 模块说明标准化（需求5）

**导航项 tooltip**：每个导航项加 `desc` 字段，Sidebar 渲染 `title` tooltip

```javascript
{ key: 'event-radar', label: '机会雷达', icon: 'satellite', hot: true,
  desc: '前瞻事件扫描 + 关注基金上车信号 + 落地验证' },
```

**页面顶部说明**：每个页面顶部已有的 subtitle 统一保留，缺失的补充

**新手引导**：首次访问时弹出功能引导（OnboardingWelcome.vue 已存在，需激活）

### 2.3 功能闭环打通（需求1）

#### P0：机会雷达买入闭环
- 候选基金 → 关注列表 → 上车信号 → 买入 → **自动标记 bought**
- 实现：`markWatchlistBought` API 已存在，在持仓交易创建时自动调用

#### P0：skill_document 注入 RAG
- 确认 skill_document.md 是否已索引到 ChromaDB
- 若未索引，执行 `scripts/distill.py` 或手动索引

#### P1：健康分驱动预警
- 健康分低于阈值时生成 alert
- 在 `alert_scanner.py` 新增 `scan_health_score` 函数

#### P1：家庭财务驱动配置
- 家庭财务数据（收入/支出/资产）驱动持仓配置建议
- 在 `allocation_advisor` 专家中注入家庭财务上下文

#### P2：机构动向驱动预警
- 北向资金大幅流入/流出时生成 alert
- 在 `alert_scanner.py` 新增 `scan_institutional_flow` 函数

#### P2：策略回测联动决策
- 决策创建前可选先回测策略历史表现
- 在决策创建流程中加入"回测验证"步骤
