# AI 对话系统升级方案

## 现状分析

### 当前架构

```
用户消息 → 选择 Agent → RAG 检索(FTS5) → 拼接 system_prompt + RAG + 历史 → LLM → 回复
```

### 现有问题

| 维度 | 现状 | 问题 |
|------|------|------|
| Agent 数量 | 2 个（估值分析师、投资研究助手） | 覆盖面窄，缺少风险管理、资产配置等方向 |
| 提示词 | 简单的 10 行 prompt | 缺少专业分析框架、输出格式约束、思维链引导 |
| Agent 路由 | 单 Agent，创建对话时绑定 | 无法根据问题自动匹配最佳 Agent |
| 记忆系统 | 20 条历史硬截断 | 长对话丢失上下文，无摘要压缩，无跨对话记忆 |
| RAG 接入 | 已接入，但 knowledge_scope 未生效 | 所有 Agent 检索范围一样，无法区分专长 |
| Skill 蒸馏 | 设计文档存在，未实现 | 作者文章的知识未融入 Agent |

## 二、提示词专业化方案

### 2.1 提示词来源

| 来源 | 内容 | 获取方式 |
|------|------|----------|
| **Skill 蒸馏** | 从 122 篇文章提取的认知框架、言行模式 | LLM 批量分析（见 skill-distillation-design.md） |
| **专业模板** | CFA/CFP 等金融分析框架 | 从公开资料整理 |
| **行业最佳实践** | 量化投资、基本面分析的标准流程 | 参考专业机构研报 |

### 2.2 提示词层次设计

```
┌─────────────────────────────────────────┐
│  Layer 1: 基础人设                       │
│  "你是xxx，擅长xxx"                       │
├─────────────────────────────────────────┤
│  Layer 2: 分析框架                       │
│  专业的分析方法论（估值分析、风险评估等）   │
├─────────────────────────────────────────┤
│  Layer 3: 输出规范                       │
│  结构化输出、数据引用、风险提示            │
├─────────────────────────────────────────┤
│  Layer 4: 思维链                         │
│  引导 AI 按步骤思考，而不是直接给答案      │
├─────────────────────────────────────────┤
│  Layer 5: 知识边界                       │
│  明确能力范围，超出时诚实说明              │
└─────────────────────────────────────────┘
```

### 2.3 专业化提示词示例（估值分析师）

```
## 人设
你是一位拥有 10 年经验的指数估值分析师，专注于 A 股和港股指数的估值研究。

## 分析框架
### 估值水平判断
- 使用历史分位点作为核心指标
- 分位点 <20%：深度低估，建议分批建仓
- 分位点 20-40%：相对低估，可适度配置
- 分位点 40-60%：合理区间，持有观望
- 分位点 60-80%：相对高估，考虑减仓
- 分位点 >80%：深度高估，建议止盈

### 趋势分析
- 观察近 3-6 个月估值走势
- 结合宏观经济环境判断趋势持续性
- 关注资金流向和市场情绪

### 风险收益比
- 计算 z-score 衡量偏离程度
- 对比历史最大回撤
- 评估当前买入的潜在下行风险

## 输出规范
### 结构要求
1. **结论先行**：先给出明确的判断（低估/合理/高估）
2. **数据支撑**：引用具体分位点、PE、PB 等数据
3. **趋势判断**：说明近期估值变化趋势
4. **风险提示**：指出主要风险因素
5. **操作建议**：给出具体但非时点性的建议

### 引用规则
- 必须引用具体数字，不要泛泛而谈
- 数据来源要明确（如"根据最新估值数据"）
- 区分确定性判断和推测性判断

## 思维链
收到问题后，请按以下步骤思考：
1. 理解用户的核心诉求（是问估值？问操作？问趋势？）
2. 检索相关指数的最新估值数据
3. 用分析框架进行系统分析
4. 综合判断给出结论
5. 标注置信度和主要风险

## 知识边界
- 擅长：指数估值分析、定投策略、行业轮动
- 不擅长：个股深度分析、宏观经济预测、政策解读
- 超出范围时说明："这超出了我的专业范围，建议咨询..."
```

### 3.2 记忆系统升级

#### 当前问题

| 问题 | 影响 |
|------|------|
| 20 条历史硬截断 | 长对话丢失上下文 |
| 无摘要压缩 | 无法回顾早期对话内容 |
| 无跨对话记忆 | 每次对话都是全新开始 |
| metadata 字段未使用 | 无法存储结构化信息 |

#### 升级方案

**Level 1: 对话内摘要压缩**

```python
def compress_history(messages: list, max_tokens: int = 4000) -> list:
    """当历史超过阈值时，压缩早期对话为摘要"""
    
    if count_tokens(messages) <= max_tokens:
        return messages
    
    # 保留最近 10 条完整消息
    recent = messages[-10:]
    old = messages[:-10]
    
    # 将早期对话压缩为摘要
    summary = summarize(old)
    
    return [
        {"role": "system", "content": f"之前的对话摘要：{summary}"},
        *recent
    ]
```

**Level 2: 跨对话记忆**

新增 `user_memory` 表：

```sql
CREATE TABLE IF NOT EXISTS user_memory (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    memory_type TEXT NOT NULL,  -- preference, fact, context
    content TEXT NOT NULL,
    source_conversation_id INTEGER,
    importance REAL DEFAULT 0.5,
    created_at TEXT DEFAULT (datetime('now','localtime')),
    last_accessed TEXT
)
```

记忆类型：
- `preference`：用户偏好（如"喜欢简洁回答"、"关注港股"）
- `fact`：用户事实（如"持有沪深300"、"定投中证500"）
- `context`：上下文信息（如"上周讨论过恒生科技"）

**Level 3: 记忆检索与注入**

```python
def build_memory_context(user_message: str, conversation_id: int) -> str:
    """检索相关记忆注入对话"""
    
    memories = []
    
    # 1. 当前对话的历史摘要
    conv_summary = get_conversation_summary(conversation_id)
    if conv_summary:
        memories.append(f"当前对话摘要：{conv_summary}")
    
    # 2. 跨对话的相关记忆
    related = search_user_memory(user_message, limit=3)
    if related:
        memories.append("用户相关记忆：" + "\n".join([m["content"] for m in related]))
    
    return "\n\n".join(memories)
```

#### 记忆系统架构

```
用户消息
    ↓
┌─────────────────────────────────┐
│  记忆检索                        │
│  1. 当前对话摘要                 │
│  2. 跨对话相关记忆               │
│  3. 用户偏好/事实                │
└─────────────────────────────────┘
    ↓
┌─────────────────────────────────┐
│  上下文组装                      │
│  system_prompt + RAG + 记忆 + 历史│
└─────────────────────────────────┘
    ↓
LLM → 回复
    ↓
┌─────────────────────────────────┐
│  记忆更新                        │
│  1. 保存用户消息                 │
│  2. 提取新记忆（偏好/事实）       │
│  3. 更新对话摘要（每 N 轮）       │
└─────────────────────────────────┘
```

## 四、知识库深度集成

### 4.1 当前 RAG 的问题

| 问题 | 说明 |
|------|------|
| knowledge_scope 未生效 | 所有 Agent 检索范围一样 |
| 只用 FTS5 全文检索 | 语义理解能力弱 |
| 检索结果固定 top 5 | 无法根据相关性动态调整 |
| 无重排序 | 检索结果可能不相关 |

### 4.2 升级方案

**Phase 1: 修复 knowledge_scope 过滤**

```python
# app.py 中调用 RAG 时传入 agent 的 knowledge_scope
rag_types = agent.get("knowledge_scope", {}).get("rag_types", [])
rag_context = build_rag_context(req.content, content_types=rag_types)
```

**Phase 2: 增强检索**

```python
def build_rag_context(query: str, content_types: list = None, top_k: int = 5) -> str:
    """增强的 RAG 检索"""
    
    # 1. FTS5 关键词检索（现有）
    fts_results = fts_search(query, content_types, top_k=top_k)
    
    # 2. 语义检索（新增，需要 Embedding）
    # semantic_results = embedding_search(query, content_types, top_k=top_k)
    
    # 3. 结果去重与重排序
    # merged = merge_and_rerank(fts_results, semantic_results)
    
    # 4. 格式化输出
    return format_rag_context(fts_results)
```

**Phase 3: 知识图谱（长期）**

构建投资知识图谱：
- 指数 → 成分股 → 行业 → 宏观经济
- 投资策略 → 风险因素 → 历史表现
- 作者观点 → 论据 → 数据来源

### 4.3 RAG 对对话的提升

| 场景 | 无 RAG | 有 RAG |
|------|--------|--------|
| 问估值 | "沪深300的估值需要查看最新数据" | "沪深300当前PE 11.5，分位点23%，处于低估区间" |
| 问策略 | "定投是一种常见的投资方式" | "根据研究员雷牛牛的观点，恐慌指数是定投的好时机" |
| 问风险 | "投资有风险，需谨慎" | "恒生科技近一年最大回撤32%，当前已反弹15%" |

## 六、验证方式

### Agent 扩展验证

- [ ] 新 Agent 能正确回答其专长领域的问题
- [ ] 不同 Agent 对同一问题的回答风格有明显差异
- [ ] 用户可以手动选择或自动路由到合适的 Agent

### 提示词验证

- [ ] 回复结构符合输出规范（结论先行、数据支撑等）
- [ ] 能引用具体数字和来源
- [ ] 超出能力范围时诚实说明

### 记忆系统验证

- [ ] 长对话（>20轮）不丢失关键上下文
- [ ] 跨对话能记住用户偏好
- [ ] 记忆检索不影响响应速度（<200ms）

### RAG 验证

- [ ] 不同 Agent 的检索范围不同
- [ ] 检索结果与问题高度相关
- [ ] 引用的知识库内容准确

## 七、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 提示词过长 | Token 成本增加 | 分层加载，按需注入 |

| 路由不准确 | 用户体验差 | 支持手动覆盖路由结果 |
| Skill 过时 | 回答不准确 | 定期增量更新 Skill |
