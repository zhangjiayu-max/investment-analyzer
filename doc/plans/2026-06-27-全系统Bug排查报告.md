## 一、排查方法论

```

  2. 异常处理缺陷      → grep 裸 except / conn.close 不在 finally
  3. API 参数校验      → 检查 data: dict 接收的无校验端点

```

## 二、发现的问题

### 🔴 严重 (3个)

#### Bug 1: Router 字典缓存无限增长

**位置:** `agent/router.py:169-178`  
**问题:** 60s TTL 但过期条目从未清理。每次新查询往 `_cache` 加一条，服务运行一周后可能有上千个过期条目。

**影响:** 内存泄漏，长期运行后 OOM。  
**修复:**
```python
# 在 route() 方法中，缓存写入前加清理
with self._lock:
    # 清理过期条目
    expired = [k for k, (_, ts) in self._cache.items() if now - ts > self._cache_ttl * 2]
    for k in expired:
        del self._cache[k]
    self._cache[cache_key] = (result, now)
```

---

#### Bug 2: RAG Query Rewrite 参数错误

**位置:** `rag.py:403`  
**问题:** `_call_llm(prompt, temperature=..., max_tokens=...)` — `prompt` 被当作 `caller` 参数传入，`messages` 缺失。每次 RAG 查询都走失败链路（重试 4 次 + 被 try/except 吞掉）。

**影响:** 每次 RAG 查询有 4 次无效重试，虽然 token 可能没消耗（请求未发出），但 caller 字段被长文本污染，token 审计数据失真。

**修复:** 降本增效第二轮文档已标记为"推荐直接关闭 query rewrite"。

---

#### Bug 3: `db/portfolio.py:1190` 裸 except

**位置:** `db/portfolio.py:1190`  
**问题:** 上次修复漏了这一处，`except:` 捕获了 KeyboardInterrupt/SystemExit。

**修复:** `except:` → `except Exception:`

### 🟡 中等 (5个)

#### Bug 4: Dashboard 启动时并行 9 个 API 调用

**位置:** `frontend/src/components/Dashboard.vue onMounted()`  
**问题:**
```javascript
onMounted(() => {
    loadDashboard(),      // API 1
    loadTodayDecisions(), // API 2
    loadDecisionReviews(),// API 3
    loadHotTopics(),      // API 4
    loadOpportunities(),  // API 5
    loadDailyReport(),    // API 6
    loadRecHistory(),     // API 7
    loadBondTemperature(),// API 8
    loadDailyAdvice(),    // API 9
})
```

9 个 API 并行调用，后端 SQLite 单写锁可能触发 `database is locked`。

**影响:** 首页偶尔加载失败，用户看到 error 卡片。

**修复:** 前 3 个关键 API 先发，其余延迟 500ms 分批发：
```javascript
loadDashboard()
loadDailyAdvice()
setTimeout(() => {
    loadHotTopics(); loadTodayDecisions(); loadDecisionReviews()
}, 500)
setTimeout(() => {
    loadDailyReport(); loadBondTemperature(); loadOpportunities(); loadRecHistory()
}, 1000)
```

#### Bug 5: Validator 配置开关缺失

**位置:** `agent/validator.py`  
**问题:** `LightValidator` 本身的规则检查是零 LLM 的，没问题。但 LLM 质检部分只在 `_llm_check` 开头检查 `validator.llm_check_enabled`。**规则检查部分没有总开关**——如果将来想完全关闭 Validator（比如怀疑它误判），没有配置项。

**修复:** 加 `validator.enabled` 开关，默认 `true`。

---

#### Bug 6: Fund Data Service 缓存缺失时静默失败

**位置:** `fund_data_service.py`  
**问题:** 多层降级（内存→SQLite→akshare→东方财富→返回 None），但调用方可能没处理 None：

```python
# db/portfolio.py:1229
from fund_data_service import get_or_refresh_fund_nav_history
nav = get_or_refresh_fund_nav_history(fund_code)
# 如果 nav 是 None，后续 _safe_float(nav[0]["净值"]) 会报错
```

**修复:** 调用方加 `if nav is None: return default_value`。
