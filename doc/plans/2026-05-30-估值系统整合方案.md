# 估值系统整合方案

## 一、背景与目标

### 1.1 现状分析

系统目前有**两套独立的估值数据体系**：

| 体系 | 数据表 | 数据来源 | 特点 |
|------|--------|----------|------|
| **指数估值** | `index_valuations` | AI 解析单指数图片 | 细粒度、支持历史趋势、指标丰富（PE/PB/股息率/风险溢价等） |
| **螺丝钉估值** | `dd_valuations` | AI 解析螺丝钉估值表 | 粗粒度、包含市场温度、仅 PE/PB 百分位 |

**问题：**
1. 两套系统数据不互通，指数代码不统一
2. 无定时自动更新，完全依赖手动操作
3. 螺丝钉数据存储在 JSON 中，无法直接用于历史趋势分析
4. 前端展示分离在两个 Tab，用户体验割裂

### 1.2 整合目标

1. **数据互补**：当指数估值数据过期时，自动参考螺丝钉估值数据
2. **统一查询**：提供智能查询接口，自动返回最佳可用数据
3. **独立存储**：保持两张表独立，减少数据冗余
4. **智能降级**：优先使用详细数据，缺失时自动降级到螺丝钉数据

---

## 二、整合方案设计

### 2.1 数据层设计（独立存储 + 查询时合并）

**核心思路**：保持两张表独立存储，查询时根据时效性智能合并返回。

**数据时效性规则**：
- 详细数据（index_valuations）：7 天内有效
- 螺丝钉数据（dd_valuations）：30 天内有效
- 超过有效期的数据标注"过期"但仍可查看

**指数代码标准化**：
- 建立指数代码映射表，统一代码格式
- 螺丝钉数据查询时自动映射到标准代码

### 2.2 API 层设计

#### 新增接口

1. **`GET /api/valuations/unified`** - 统一估值查询
   ```python
   # 参数
   - index_code: 指数代码（可选）
   - metric_type: 指标类型（可选，默认市盈率）
   - source: 数据来源筛选（可选：manual/akshare/螺丝钉/all）
   - max_days: 最大有效天数（可选，默认 7）
   
   # 返回
   {
     "indexes": [
       {
         "index_code": "000300",
         "index_name": "沪深300",
         "metric_type": "市盈率",
         "current_value": 14.56,
         "percentile": 91.37,
         "data_source": "manual",  # 数据来源
         "is_expired": false,      # 是否过期
         "days_old": 2,            # 数据天数
         "market_temperature": null  # 市场温度（仅螺丝钉数据有）
       }
     ],
     "market_temperature": 25.5,  # 最新市场温度
     "summary": {
       "total": 50,
       "valid": 45,
       "expired": 5
     }
   }
   ```

2. **`GET /api/valuations/market-temperature`** - 市场温度查询

   ```python
   }
   ```

### 2.3 前端展示设计（保持独立 Tab）

**保持现有两个 Tab 分离**，但在每个 Tab 内增加智能提示：

#### Tab 1: 指数估值（主 Tab）

- 显示 `index_valuations` 表的数据
- 在指数选择器下方添加提示：
  - "该指数最近 7 天无详细数据，显示螺丝钉估值（X天前）"
  - "数据来源：螺丝钉（点击查看详情）"
- 在页面顶部添加市场温度卡片（来自螺丝钉数据）

#### Tab 2: 螺丝钉估值（辅助 Tab）

- 显示 `dd_valuations` 表的数据
- 保持现有的表格展示
- 添加"同步到指数估值"按钮（手动触发）

### 2.4 智能降级逻辑

```python
def get_best_valuation(index_code: str, metric_type: str = "市盈率") -> dict:
    """获取最佳估值数据（智能降级）。"""
    # 1. 优先使用最近 7 天内的详细数据
    detailed = get_latest_valuation(index_code, metric_type, max_days=7)
    if detailed:
        detailed["data_source"] = "manual"
        detailed["is_expired"] = False
        return detailed
    
    # 2. 降级到螺丝钉数据（最近 30 天内）
    dd_data = get_latest_dd_valuation(index_code, metric_type, max_days=30)
    if dd_data:
        dd_data["data_source"] = "螺丝钉"
        dd_data["is_expired"] = False
        dd_data["degraded"] = True
        return dd_data
    
    # 3. 使用过期数据（如有）
    detailed_expired = get_latest_valuation(index_code, metric_type, max_days=365)
    if detailed_expired:
        detailed_expired["data_source"] = "manual"
        detailed_expired["is_expired"] = True
        return detailed_expired
    
    # 4. 无可用数据
    return None
```

## 四、风险与注意事项

### 4.1 数据质量风险

- **风险**：螺丝钉数据精度可能不如详细图片（仅 PE/PB，无 zscore 等）
- **缓解**：在 UI 上明确标注数据来源，让用户知道数据精度差异

### 4.2 指数代码映射风险

- **风险**：AI 识别的指数代码可能不准确
- **缓解**：建立映射表，支持手动修正；使用名称模糊匹配作为兜底

### 4.3 数据一致性风险

- **风险**：同一指数在不同来源的数据可能不一致
- **缓解**：优先使用最新数据，在 UI 上显示多个来源的数据供参考
