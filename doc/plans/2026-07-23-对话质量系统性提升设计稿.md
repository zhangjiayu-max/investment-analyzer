## 二、修复方案

### G-1：文章解读专家深度分析能力

**目标**：文章解读从"概要式总结"升级为"深度逻辑拆解"。

**修改**：`backend/db/agents.py` 中 article_expert 的 system_prompt（行 602-644）

在 `## 分析框架` 下新增 3 个深度分析维度（在"文章内容提取"之后）：

```
### 历史对比维度（必做）
- 识别文章引用的历史事件（如"2007五朵金花""2015互联网+""2021核心资产"）
- 对比当前事件与历史事件的异同点（驱动因素/市场环境/估值水位/资金结构）
- 指出历史规律在当前场景下的适用性边界（哪些条件已变化）

### 机制拆解维度（必做）
- 拆解文章核心论点的传导链路：A → B → C（如"抱团→估值泡沫→资金虹吸→瓦解"）
- 识别链路中的关键假设和触发条件
- 标注每个环节的不确定性和潜在证伪点

### 量化验证维度（必做）
- 文章提到的每个数据点（如"暴涨翻倍""阴跌"）必须尝试量化
- 对涉及标的/板块，必须调用 query_valuation 工具查询当前估值分位
- 文章未提供的数据（如 PE/PB 百分位）必须在"数据缺口"中明确标注
- 禁止在无估值数据的情况下给出"减仓/清仓"操作建议
```

**修改**：`## 负面约束` 下追加：
```
- 禁止仅基于"文章观点"+"已亏损"就建议减仓/清仓，必须叠加估值分位或基本面恶化证据
- 涉及亏损标的的操作建议，必须先查询当前估值分位，标注"现在是否适合卖"
```

**复活深度约束**：`agent/core/multi_agent.py`
- 将 `_RISK_AND_DEPTH_CONSTRAINT`（行 230-252）追加到 article_expert 的 prompt 加载流程
- 或新增 `ARTICLE_DEPTH_CONSTRAINT` 专门针对文章解读

#### G-3b：区分止盈 vs 止损 + 拦截止损

新增 `_classify_sell_type(answer, specialist_results)` 函数：

```python
def _classify_sell_type(answer: str) -> str:
    """区分止盈（盈利卖出）vs 止损（亏损清仓/减仓）。
    
    Returns:
        "take_profit" | "stop_loss" | "unknown"
    """
    # 检测亏损关键词
    loss_kws = ["亏损", "浮亏", "亏损中", "深套", "套牢", "-1", "-2", "跌幅"]
    has_loss = any(kw in answer for kw in loss_kws)
    
    # 检测卖出操作词
    sell_kws = ["减仓", "清仓", "卖出", "止损", "割肉"]
    has_sell = any(kw in answer for kw in sell_kws)
    
    if has_sell and has_loss:
        return "stop_loss"
    if has_sell:
        return "take_profit"
    return "unknown"
```

```python
def _extract_viewpoints_for_rag(article_context: str) -> list[str]:
    """从文章内容提取核心论点用于 RAG 检索。
    
    复用 services/content/article_reader.py 的 extract_article_structure，
    取 core_viewpoints 前3个，每个扩展为检索 query。
    """
    try:
        from services.content.article_reader import extract_article_structure
        structure = extract_article_structure(article_context)
        viewpoints = structure.get("core_viewpoints", [])
        mentioned_targets = structure.get("mentioned_targets", [])
        
        queries = []
        for vp in viewpoints[:3]:
            # 拼接：核心论点 + 提及标的
            query = vp.get("viewpoint", "") if isinstance(vp, dict) else str(vp)
            if mentioned_targets:
                query += " " + " ".join(t.get("name", "") for t in mentioned_targets[:2] if isinstance(t, dict))
            if query.strip():
                queries.append(query.strip())
        return queries
    except Exception:
        return []
```

## 五、风险与回滚

- **G-1 风险**：prompt 加长可能增加 token 消耗。回滚：关闭 `article_depth_analysis_enabled` 开关。
- **G-2 风险**：sell 裁决降级可能导致该 sell 时未 sell。缓解：降级为 hold 时在报告中明确提示"估值数据缺失，建议补充后再决策"。
- **G-3 风险**：止损拦截可能误伤合理止损场景。缓解：满足估值高位或基本面恶化时仍允许止损。
- **G-4 风险**：extract_article_structure 可能失败。兜底：用原固定 suffix。
