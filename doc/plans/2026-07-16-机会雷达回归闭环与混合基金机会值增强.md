# 机会雷达回归闭环增强 + 混合基金机会值分析

> 日期：2026-07-16
> 范围：事件雷达回归验证闭环增强 + 关注基金（混合型）机会值分析补全
> 原则对照：②回归闭环 ⑥前瞻性 ①多维度数据

## 一、问题诊断

### 问题1：事件雷达回归闭环薄弱

**现有闭环**（`event_radar.py`）：
```
scan_forward_events → 创建事件(含_calibrate_confidence校准)
→ 事件状态流转 upcoming→imminent→materialized
→ verify_materialized_events → _verify_single_event
→ update_event_verification 写入 verification_result
→ get_sector_accuracy_stats 聚合板块准确率
→ _calibrate_confidence 在新事件创建时校准 confidence
```

**5 个薄弱点**：

| # | 薄弱点 | 影响 | 现状代码 |
|---|--------|------|---------|
| 1 | 只校准 confidence，不影响 direction | 方向预测错误不会被惩罚，同样的方向错误会重复 | event_radar.py L840 |
| 2 | 趋势事件（is_trend）跳过校准 | 趋势类事件置信度恒为原始值，不学习 | event_radar.py L843 |
| 3 | 仅新事件创建时校准，已存在事件不回溯 | 早期的 upcoming 事件即使板块准确率已更新，也不会被重新校准 | 缺失回溯机制 |
| 4 | 前端不展示"原始 vs 校准后"对比 | 用户无法判断置信度是 LLM 原始判断还是已校准 | EventRadarPage.vue L1273 |
| 5 | 验证只取 affected_sectors[0] 第一个指数 | 多板块事件只用一个板块验证，其他板块方向错误不可见 | _verify_single_event L1081 |

### 问题2：混合基金机会值分析空白

**现有逻辑**（`watchlist.py watchlist_patrol_api`）：
```
混合基金无 tracking_index → index_name 为空
→ 三级估值查询全跳过（本地DB/天天基金/akshare 都依赖 index_name）
→ current_pct = None
→ _calculate_suggested_buy_price 返回 None
→ _compute_signal_status 返回 "gray"
→ 前端显示"无数据"/"估值数据不足"
```

**已有但未接入的能力**：

| 能力 | 文件 | 适合混合基金？ |
|------|------|---------------|
| `calculate_drawdown_analysis(fund_code)` | services/fund/fund_analysis.py L418 | ✅ 返回 drawdown_percentile（回撤分位），可作为机会值替代 |
| `yingmi_fund_diagnosis` 工具 | tools/__init__.py L563 | ✅ 含基金级估值参考 |
| `eastmoney_fund_diagnosis` 工具 | tools/__init__.py L519 | ✅ 综合诊断 |

## 二、增强方案

### 增强1：事件雷达回归闭环强化

#### 1.1 验证多板块（修复薄弱点5）

`_verify_single_event` 改为遍历所有 `affected_sectors`，对每个有指数映射的板块分别验证，取**加权平均**作为最终结果（按置信度权重）。

```python
# 修改前：只取第一个板块
for s in sectors:
    key = _normalize_sector(s)
    if key and key in SECTOR_TO_INDEX:
        index_code = SECTOR_TO_INDEX[key][0]
        break

# 修改后：遍历所有板块，取加权平均
sector_results = []
for s in sectors:
    key = _normalize_sector(s)
    if key and key in SECTOR_TO_INDEX:
        index_code = SECTOR_TO_INDEX[key][0]
        prices = _fetch_index_close_prices(index_code, mat_date, end_date)
        if prices:
            change_pct = ...
            sector_results.append({"sector": s, "index_code": index_code, "change_pct": change_pct, "status": ...})

# 加权平均（等权，未来可按板块置信度加权）
if sector_results:
    avg_change = sum(r["change_pct"] for r in sector_results) / len(sector_results)
    # 方向判定用多数投票
    correct_count = sum(1 for r in sector_results if r["status"] == "correct")
    wrong_count = sum(1 for r in sector_results if r["status"] == "wrong")
    if correct_count > wrong_count: final_status = "correct"
    elif wrong_count > correct_count: final_status = "wrong"
    else: final_status = "flat"
```

verification_result 结构扩展：
```json
{
  "status": "correct",
  "change_pct": 2.5,
  "sector_results": [
    {"sector": "医药", "index_code": "000808", "change_pct": 3.2, "status": "correct"},
    {"sector": "新能源", "index_code": "399808", "change_pct": 1.8, "status": "correct"}
  ],
  ...
}
```

#### 1.2 回溯校准已存在事件（修复薄弱点3）

新增 `recalibrate_existing_events()` 函数，在 `verify_materialized_events` 完成后调用：
- 查询所有 `status IN ('upcoming','imminent')` 且 `affected_sectors` 对应板块有 >=3 个验证样本的事件
- 用 `_calibrate_confidence` 重新计算 confidence
- 若新 confidence 与原 confidence 差异 > 0.1，更新事件并追加 timeline "回溯校准：{old}→{new}"

#### 1.3 趋势事件也参与校准（修复薄弱点2）

移除 `scan_forward_events` L843 的 `if not is_trend` 条件，趋势事件同样校准。趋势事件通常置信度较高，校准后会更真实反映历史准确率。

#### 1.4 方向错误惩罚机制（修复薄弱点1）

在 `_calibrate_confidence` 基础上新增 `_calibrate_direction`：
- 若某板块近 10 个验证事件中，方向错误率 > 40%，后续该板块的新事件 direction 降级（positive→neutral, negative→neutral）
- 记录到 timeline "方向降级：{old}→{new}（板块方向错误率 {rate}%）"

#### 1.5 前端展示原始 vs 校准后置信度（修复薄弱点4）

EventRadarPage.vue 事件卡片的 confidence-tag 改造：
- 若 `calibrated_confidence` 字段存在且与 `confidence` 不同，显示两个值：`原始 75% → 校准 60%`
- 若无校准（字段相同或不存在），维持原显示

后端 `list_market_events` / `get_market_event` 返回时追加 `original_confidence` 字段（从 timeline 解析或新增字段）。

#### 1.6 落地验证 Tab 增强

Tab 3「落地验证」当前只展示准确率统计 + 引导跳转。增强为：
- **准确率趋势图**：近 30 天每日准确率折线（correct/total）
- **板块准确率热力图**：各板块准确率 + 样本数，颜色深浅反映准确率
- **最近验证事件列表**：最近 10 条已验证事件，含标题/方向/预测/实际涨跌/验证结果
- **校准日志**：最近 10 条回溯校准记录（事件标题 + 原始置信度 + 校准后 + 原因）

### 增强2：混合基金机会值分析

#### 2.1 巡检流程增强（watchlist_patrol_api）

对 `index_name` 为空的基金（混合型/其他无跟踪指数基金），启用**三级替代分析**：

```
第一级：回撤分位分析（calculate_drawdown_analysis）
  → drawdown_percentile >= 80% → 信号灯 green（接近历史大底）
  → drawdown_percentile >= 60% → 信号灯 yellow（回撤较深）
  → drawdown_percentile < 60% → 信号灯 red（未到机会区）
  → 数据不足 → 进入第二级

第二级：盈米基金诊断（yingmi_fund_diagnosis）
  → 解析估值字段（若有）→ 转化为百分位 → 信号灯
  → 无估值字段 → 进入第三级

第三级：净值历史分位（fund_nav_history 表）
  → 当前净值在近 1000 天净值序列中的分位
  → 分位 <= 20% → green
  → 分位 <= 40% → yellow
  → 分位 > 40% → red
```

#### 2.2 上车价推算增强（_calculate_suggested_buy_price）

新增分支：当 `current_pct` 为 None 但有 `drawdown_percentile` 时：
```python
if current_pct is None and drawdown_percentile is not None:
    if drawdown_percentile >= 80:
        return round(current_nav, 4), "历史大底区·当前净值"
    elif drawdown_percentile >= 60:
        return round(current_nav * 0.97, 4), "深度回撤区·-3%"
    elif drawdown_percentile >= 40:
        return round(current_nav * 0.93, 4), "中度回撤区·-7%"
    else:
        return round(current_nav * 0.90, 4), "等待更大回撤·-10%"
```

#### 2.3 信号灯计算增强（_compute_signal_status）

新增分支：当 `current_pct` 为 None 但有 `drawdown_percentile` 时：
```python
if current_pct is None:
    if drawdown_percentile is not None:
        if drawdown_percentile >= 80: return "green", f"处于历史深度回撤区（回撤分位{drawdown_percentile}%），接近大底", None
        elif drawdown_percentile >= 60: return "yellow", f"回撤较深（分位{drawdown_percentile}%），接近机会区", None
        else: return "red", f"回撤不足（分位{drawdown_percentile}%），等待更深回调", None
    return "gray", "估值数据缺失，请巡检刷新", None
```

#### 2.4 watchlist 表字段扩展

新增字段存储替代分析结果：
```sql
ALTER TABLE watchlist ADD COLUMN analysis_method TEXT DEFAULT '';  -- 分析方法：index_valuation/drawdown/yingmi/nav_percentile
ALTER TABLE watchlist ADD COLUMN drawdown_percentile REAL;          -- 回撤分位（混合基金用）
ALTER TABLE watchlist ADD COLUMN nav_percentile REAL;               -- 净值历史分位（兜底）
```

#### 2.5 前端展示增强

EventRadarPage.vue 关注基金卡片：
- 估值分位列：若 `analysis_method` 不是 `index_valuation`，显示标签（如"回撤分位 85%"而非"估值分位"）
- 信号灯理由：展示具体分析方法（"基于回撤分位分析"）
- 折叠详情区：增加"机会值分析维度"区块，展示：
  - 跟踪指数估值（若有）
  - 回撤分位（若用回撤分析）
  - 净值历史分位（若用净值分位）
  - 盈米诊断估值（若有）

#### 2.6 买入评分增强（_calculate_buy_score）

当前 4 维度评分全部依赖 `current_pct`，混合基金得分为 0。增强为：
- 若 `current_pct` 为 None 但有 `drawdown_percentile`，用 `drawdown_percentile` 替代 `current_pct` 计算 v_score
- v_score = max(0, 100 - max(0, 80 - drawdown_percentile) * 1.2)（回撤分位越高分越高）
