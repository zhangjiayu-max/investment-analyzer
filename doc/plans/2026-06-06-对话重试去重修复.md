# 对话重试去重修复方案

## 问题背景

对话ID 32 出现 33 条 agent_runs（5个专家 × ~6次重复执行），根因是前端页面切换时的恢复/重试机制存在竞态条件，导致同一条用户消息被重复发送到后端。

## 根因分析

| 问题 | 严重度 | 位置 |
|------|--------|------|
| `sendMessageAndTrack` 无防重复调用守卫 | 高 | `autoRecoverIfNeeded` 场景2 |
| `recoverFromDisconnect` 和 `autoRecoverIfNeeded` 可竞态执行 | 高 | 无共享互斥锁 |
| `startStream` 静默覆盖已有流状态（丢失 AbortController） | 中 | `useStreamingState.js` |
| `tryResumeConversation` 失败后回退到 `sendMessageAndTrack` | 中 | ChatView.vue |
| 后端无消息级去重 | 中 | `send_message_stream` |

## 修复方案

### 1. 添加恢复互斥锁（ChatView.vue）

用 `isRecovering` ref 防止 `recoverFromDisconnect` 和 `autoRecoverIfNeeded` 并发执行：

```js
const isRecovering = ref(false)

async function autoRecoverIfNeeded(convId) {
  if (isRecovering.value) return
  isRecovering.value = true
  try {
    // ... 原有逻辑
  } finally {
    isRecovering.value = false
  }
}

async function recoverFromDisconnect() {
  if (isRecovering.value) return
  isRecovering.value = true
  try {
    // ... 原有逻辑
  } finally {
    isRecovering.value = false
  }
}
```

### 2. `sendMessageAndTrack` 添加 sending 守卫（ChatView.vue）

```js
function sendMessageAndTrack(convId, text) {
  const state = getStreamState(convId)
  if (state?.sending) {
    console.warn(`对话 ${convId} 已有进行中的请求，跳过重复发送`)
    return
  }
  // ... 原有逻辑
}
```

### 3. `startStream` 前清理已有状态（useStreamingState.js）

```js
function startStream(convId, controller) {
  // 如果已有流，先中止
  const existing = streamStates.get(convId)
  if (existing?.controller) {
    existing.controller.abort()
  }
  // ... 原有逻辑
}
```

### 4. `tryResumeConversation` 失败后先检查后端状态（ChatView.vue）

```js
onError: (cid, errorData) => {
  if (errorData.code === 'RESUME_FAILED') {
    finishStream(cid)
    removeTask(cid)
    // 先刷新消息，看后端是否已完成
    loadMessages(cid).then(() => {
      const lastAssistant = [...messages.value].reverse().find(m => m.role === 'assistant')
      if (lastAssistant && lastAssistant.execution_status !== 'streaming') {
        // 后端已完成，无需重发
        showToast('任务已完成', 'success')
        return
      }
      // 确实需要重发
      const lastUserMsg = [...messages.value].reverse().find(m => m.role === 'user')
      if (lastUserMsg) {
        showToast('正在重新执行...', 'info')
        sendMessageAndTrack(cid, lastUserMsg.content)
      }
    })
    return
  }
  // ...
}
```

### 5. 后端消息级去重（conversations.py）

在 `send_message_stream` 中，检查是否已有进行中的 agent_runs：

```python
# 检查是否有进行中的任务（防前端重复发送）
from db.agents import get_running_agent_count
running = get_running_agent_count(conv_id)
if running > 0:
    yield _sse_event("error", {"message": "该对话有进行中的任务，请等待完成", "code": "DUPLICATE_REQUEST"})
    return
```

## 验证方式

1. 启动后端，发送一条复杂问题（触发多专家）
2. 立即切换页面，再切回
3. 检查 agent_runs 表，确认没有重复记录
4. 快速多次切换页面，确认只触发一次执行
