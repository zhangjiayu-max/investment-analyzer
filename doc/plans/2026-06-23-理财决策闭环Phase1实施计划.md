# 理财决策闭环 Phase 1 实施计划

> 基于设计稿：`doc/plans/2026-06-23-理财决策闭环与AI对话增强设计.md`

**目标：** 打通闭环地基，让 AI 对话不丢上下文，AI 建议能沉淀为决策草案，决策能经过执行前检查并生成待确认交易草稿。

**架构：** Phase 1 不引入外部队列或新状态服务，沿用 SQLite、FastAPI、Vue 3。后端新增聚合型上下文构建器和交易草稿服务，复用现有 `decision_records`、`decision_actions`、`portfolio_transactions` 和对话表。前端先以最小交互接入保存决策与交易草稿，完整行动中心留到 Phase 2。

**Tech Stack:** Python 3.11+ / FastAPI / sqlite3 / pytest / Vue 3 Composition API / axios / Vitest

---

## 文件结构

- Create `backend/conversation_context.py`  
  统一构建对话上下文包，聚合摘要、最近消息、持仓、估值、决策、关注列表和 RAG。

- Create `backend/tests/test_conversation_context.py`  
  验证上下文包包含持仓、现金、决策、摘要和最近消息，且按 token 预算裁剪。

- Modify `backend/db/decisions.py`  
  增强 `build_decision_precheck()`，加入持仓集中度、现金余额、数据新鲜度、重复未完成交易检查；新增 `create_transaction_draft_from_decision()`。

- Create `backend/tests/test_db_decisions_phase1.py`  
  覆盖执行前检查和从决策生成交易草稿。

- Modify `backend/routers/decisions.py`  
  新增 `POST /api/decisions/{decision_id}/transaction-draft`，返回交易草稿和更新后的决策。

- Modify `backend/routers/conversations.py`  
  在 `chat/simple/complex` 分支使用统一上下文包；优先保持现有接口行为，减少一次性重构风险。

- Modify `frontend/src/api/index.js`  
  新增生成交易草稿 API；清理重复 `return`。

- Modify `frontend/src/components/ChatView.vue` and `frontend/src/components/chat/ChatMessage.vue`  
  确保 assistant 消息能保存为决策草案，并在保存成功后提示用户。

- Modify `frontend/src/components/DecisionRecordsPage.vue`  
  在决策卡片增加“生成交易草稿”入口，展示 precheck 风险。
