# 估值图片重新识别功能设计

## 背景

当前估值图片识别后，已解析的卡片上没有"重新识别"按钮。如果识别结果有误（如"未知指数"），用户只能删除图片再重新上传，体验差。需要让已解析的图片也能一键重新识别并更新数据。

## 当前状态

### 雷牛牛估值 Tab（gallery）
- 已解析区（`records`）：展示 index_name、metric_type、current_value、percentile
- **无"重新识别"按钮**
- 后端 API：`POST /api/valuation/parse` → `parse_single_valuation()` → `ImageParser.parse()` → `save_valuation()`
- `save_valuation()` 用 `ON CONFLICT(index_code, snapshot_date, metric_type) DO UPDATE` —— 重复解析会覆盖，无需改后端

### 螺丝钉估值 Tab（dd）
- 已解析的图片显示绿色"已识别"徽章
- 图片上已有"识别估值"按钮，点击可重新解析
- 异步任务去重：`find_running_task()` 只检查 `pending/parsing` 状态，`done/error` 可重新触发
- `save_dd_valuation()` 用 `ON CONFLICT(image_path) DO UPDATE` —— 重复解析自动覆盖
- **不需要改动**

## 修改方案

只改**前端的雷牛牛已解析区**，后端不需改动（解析和存储逻辑已有覆盖能力）。

### 文件：`frontend/src/components/ImageGallery.vue`

#### 改动 1：已解析记录卡片加"重新识别"按钮

**位置**：`records` 模板循环（约第 845-933 行），在每张记录卡片的操作区加按钮

**UI**：每个已解析卡片右下角加一个 `btn-reparse` 按钮，内容为 🤖 重新识别

**交互流程**：
1. 点击"重新识别" → 弹出 `ConfirmDialog` 确认（遵循项目规范，所有操作类按钮必须二次确认）
2. 用户确认 → 调用已有的 `parseAndSaveValuation(r.image_path, r.model_type)` API
3. 按钮进入 loading 状态（disabled + 显示"识别中..."）
4. 解析完成 → 刷新 records 列表
5. 解析失败 → toast 提示错误

#### 改动 2：确认函数 `confirmReParse(record)`

新增函数，逻辑：
```javascript
const confirmReParse = async (record) => {
  // ConfirmDialog 已由模板的 @confirm 触发
  const parseFn = isDDImage({ path: record.image_path }) ? parseDDImage : parseAndSaveValuation
  record._reparsing = true  // 局部 loading 状态
  try {
    await parseFn(record.image_path, record.model_type || '')
    showToast('识别完成', 'success')
    await loadRecords()  // 刷新已解析列表
  } catch (e) {
    showToast('识别失败: ' + (e.response?.data?.detail || e.message), 'error')
  } finally {
    record._reparsing = false
  }
}
```
