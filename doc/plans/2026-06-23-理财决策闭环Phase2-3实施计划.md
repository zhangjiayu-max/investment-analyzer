# 理财决策闭环 Phase 2-3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把持仓 AI 分析输出沉淀为可处理的“建议候选池”，并把决策复盘统计变成可查询、可展示、可被后续 AI 对话引用的个人理财学习闭环。

**Architecture:** Phase 2 在现有 `db.decisions` 中新增 `recommendation_candidates` 表，AI 分析完成后通过确定性抽取器生成候选建议，用户再显式保存为决策或忽略。Phase 3 在同一决策数据层提供聚合统计，复用已有 `decision_reviews`、知识沉淀和坏例评测生成逻辑，前端在 `DecisionRecordsPage.vue` 中展示建议候选和复盘洞察。

**Tech Stack:** Python 3.11、FastAPI、sqlite3、pytest、Vue 3 `<script setup>`、axios、Vite。
