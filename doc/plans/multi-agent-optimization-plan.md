# 多 Agent 项目优化设计稿

## 背景

对照 AI 大模型工程化面试标准，从 4 个维度审视项目不足，制定优化方案。

---

## 1. Agent 逻辑透明度

### 现状
- Function Calling 机制完整：8 工具 + 4 专家 Agent + Orchestrator 编排
- 有界循环（MAX_TURNS=5/6），工具子集隔离

### 不足与优化

#### 1.1 JSON 解析失败静默回退
- 现状：`args={}` 无日志无重试
- 优化：记录 malformed JSON，让 LLM 重试一次

#### 1.2 Prompt 无结构化标签
- 现状：纯 Markdown 文本
- 优化：用 `<context>`、`<instructions>`、`<examples>` 标签分区

#### 1.3 无 Few-shot 示例
- 优化：高频场景加 1-2 个示例

#### 1.4 Lost-in-the-Middle
- 现状：RAG 上下文直接拼接
- 优化：最相关片段放首尾，次要信息放中间

---

## 2. RAG 质量

### 现状
- FTS5 + ChromaDB 双路召回
- RAG 日志 + 统计仪表盘
- 6 类知识源分类

### 不足与优化

#### 2.1 分段策略粗糙
- 现状：固定 500 字符切分
- 优化：按段落/句子边界切分，保留上下文

#### 2.2 无 Rerank
- 优化：加 `bge-reranker-small` 对 Top-K 重排序

#### 2.3 无 HyDE
- 优化：先让 LLM 生成假设答案，再用它检索

#### 2.4 分数合并不可靠
- 现状：FTS5 rank 和 cosine distance 独立归一化
- 优化：统一用加权 RRF（Reciprocal Rank Fusion）

#### 2.5 文档截断过短
- 玼状：文章 5000 字截断
- 优化：延长到 15000+，或全文索引

---

## 3. 系统可靠性

### 现状
- SSE 流式输出
- Vision API 3 次重试 + Semaphore 并发

### 不足与优化

#### 3.1 LLM 限流无重试
- 优化：加指数退避重试，检测 429 状态码

#### 3.2 无内容安全过滤
- 优化：输入长度限制 + 输出敏感词检测

#### 3.3 无语义缓存
- 优化：相似查询复用结果

#### 3.4 无幻觉检测
- 优化：LLM 回答标注数据来源，与知识库交叉验证

#### 3.5 无 Token 成本追踪
- 优化：从 `response.usage` 记录 token 数，建成本仪表盘

#### 3.6 LLM 超时未配置
- 优化：显式设置 `timeout=60s`

---

## 4. 工程成熟度

### 不足与优化

#### 4.1 测试覆盖率 ~0%
- 优化：给 `tools.py`、`rag.py` 写单元测试

#### 4.2 无 Bad Case 分析
- 优化：记录用户追问/否定的对话

#### 4.3 无错误聚合
- 优化：接入错误日志表

---

## 优先级排序

1. LLM 调用加指数退避重试
2. Prompt XML 标签隔离
3. RAG 加 Rerank
4. Token 用量记录
5. JSON 解析失败重试
6. RAG 分段策略优化（按句子边界）
7. RAG 分数合并优化（RRF）
8. 输入输出安全过滤
9. 单元测试
