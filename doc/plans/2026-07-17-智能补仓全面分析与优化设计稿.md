## 二、成熟补仓策略调研

### 2.1 经典策略对比

| 策略 | 核心思想 | 适用场景 | 优势 | 劣势 |
|------|----------|----------|------|------|
| **等额定投 (DCA)** | 固定金额定期买入 | 长期储蓄型投资 | 简单、无脑执行 | 摊薄效率低 |
| **价值平均法 (VA)** | 目标市值路径，偏离时调整 | 有明确目标金额 | 自动低买高卖 | 需较大现金储备 |
| **金字塔补仓** | 越跌越买，跌幅越大仓位越大 | 确定性高的标的 | 摊薄效率高 | 接飞刀风险 |
| **网格交易** | 预设价格区间，机械买卖 | 震荡市 | 震荡市收益稳定 | 单边市失效 |
| **凯利公式** | 基于胜率/盈亏比定仓位 | 有历史统计的标的 | 数学最优 | 参数敏感 |
| **均线补仓** | 价格站上/跌破均线触发 | 趋势跟踪/均值回归 | 有技术面支撑 | 滞后性 |
| **市盈率分位补仓** | 仅在低估时分位加仓 | 指数基金 | 安全边际高 | 可能长期等不到 |
| **最大回撤补仓** | 从高点回撤N%后加仓 | 趋势向上的标的 | 利用回调 | 无法区分回调与反转 |

### 2.2 价值平均法 (Value Averaging) 详解

由 Michael Edleson 提出，核心公式：

```
目标市值_t = 初始市值 + t × 每期目标增长额
本期投入 = 目标市值_t - 实际市值_t
```

- 当实际市值 > 目标市值 → 投入为负（卖出或减少投入）
- 当实际市值 < 目标市值 → 投入为正（加大买入）
- 自然实现"低买高卖"的纪律性

**与当前DCA的关键区别**：VA 是"市值驱动"（目标导向），DCA 是"金额驱动"（投入固定）。VA 在深度下跌时会自动加大投入，在上涨时会自动减少甚至卖出。

### 2.3 网格交易详解

```
设定价格区间 [P_low, P_high]，划分为 N 格
每格间距 = (P_high - P_low) / N
每格仓位 = 总资金 / N

价格触及格线 → 机械执行买入/卖出
```

**与本系统的关系**：网格交易更适合"震荡市"中的指数基金，可以作为金字塔补仓的补充策略。当估值处于历史中位数附近（30-70%分位）时，网格交易比金字塔更有效。

### 2.4 机构常用补仓框架

| 机构做法 | 说明 |
|----------|------|
| **分批建仓** | 首次建仓30%，后续分3-4批在回调中完成 |
| **动态再平衡** | 偏离目标配置±5%时触发再平衡 |
| **风险预算** | 按波动率分配仓位，高波动低仓位 |
| **最大回撤止损** | 从最高点回撤20%强制止损 |
| **时间止损** | 持仓超过N个月未盈利则减仓 |
| **基本面恶化退出** | 基金经理变更/规模暴增/风格漂移则退出 |

### 3.2 当前仍存在的问题

#### P0 — 严重设计缺陷

**问题1：只有"买"信号，没有"卖/减/停"信号**

当前系统三个信号（A金字塔/B趋势/C大跌）全部是"买入"建议。一个完整的投资策略必须包含退出机制：
- 何时止盈？亏损修复后是否继续持有？
- 何时止损？跌到多少应该认输出局？
- 何时暂停？资金池耗尽后怎么办？

**影响**：用户可能陷入"无限补仓"陷阱，越补越深。

**问题2：信号A与信号C的互斥逻辑不够清晰**

- 亏损-9%时触发信号C（大跌定投），建议金额 = 市值×8%×1.0×1.0×仓位余量
- 亏损-10%时触发信号A（金字塔），建议金额 = 市值×add_ratio%
- 边界处建议金额可能跳变，用户困惑

**问题3：信号B（趋势加仓）与价值投资理念矛盾**

"涨势好时加仓"本质是追涨，与"估值低时加仓"的价值投资理念相悖。虽然系统标注了"短期波段"，但普通用户可能不理解这种策略切换的逻辑。

**影响**：用户可能在趋势信号触发时追高买入，随后趋势反转被套。

#### P1 — 功能缺失

**问题4：缺少"价值平均法"（Value Averaging）作为可选策略**

VA 是 DCA 的最重要升级方向之一。当前引擎1是固定金额定投（金额驱动），VA 是目标市值驱动，更符合"低买高卖"的纪律性。

**问题5：缺少"网格交易"（Grid Trading）策略**

对于估值处于合理区间（30-70%分位）的标的，金字塔补仓可能长期不触发，网格交易可以在此期间产生收益。

**问题6：缺少"基本面恶化"检测**

- 基金经理变更：新经理风格可能不同
- 规模暴增：基金规模过大影响灵活性
- 跟踪误差扩大：指数基金偏离基准
- 费率变化：管理费上调

这些因素可能导致"越补越错"。

**问题7：缺少"仓位再平衡"建议**

当前系统只关注单标的补仓，没有考虑组合整体配置偏离。例如某个标的因跌幅过大导致仓位占比下降，是否需要补回目标配置？

**问题8：摊薄模拟器过于简陋**

当前只支持"再跌X%补Y元"的单一场景，无法对比不同策略（如"等额定投 vs 金字塔 vs 价值平均"）的效果差异。

#### P2 — 体验问题

**问题9：冷却期提示不够直观**

前端只显示"已拦截"和原因文字，用户无法直观看到冷却期剩余天数和已用次数。

**问题10：金字塔档位配置复杂**

当前配置格式 `10:15:5,20:25:10,30:30:15,40:20:20,50:10:25` 对普通用户不友好，含义不直观。

**问题11：引擎1和引擎2的结果展示割裂**

引擎1（月定投）和引擎2（金字塔）分开展示，用户需要自行理解两者的关系。实际上它们是同一策略的不同层次。

**问题12：资金池"平均配额预警"含义模糊**

"超出平均配额"这个提示用户可能不理解是什么意思、应该怎么做。

## 四、优化方案

### 4.1 总体原则

1. **不重复造轮子**：复用现有双引擎+L2-L5+三信号框架，在之上做增强
2. **先补"退出"短板**：P0 优先解决"只有买没有卖"的问题
3. **策略可选**：新增策略作为可选引擎，默认关闭，用户自行开启
4. **向后兼容**：现有API字段不删不改，只新增
5. **开关控制**：所有新功能默认关闭，通过配置项开启

### 4.2 方案一：退出机制（P0）

新增"退出信号"层，与现有"进入信号"对称：

```
触发信号
├── 进入信号（现有）
│   ├── 信号A: 金字塔补仓（深跌摊薄）
│   ├── 信号B: 趋势加仓（涨势跟进）
│   └── 信号C: 大跌定投（分批接刀）
│
└── 退出信号（新增）
    ├── 信号D: 止盈退出（盈利达目标）
    ├── 信号E: 止损退出（跌破容忍线）
    └── 信号F: 暂停观望（估值过高/资金耗尽）
```

**信号D：止盈退出**

| 条件 | 建议 |
|------|------|
| 盈利 > 20%（宽基）/ 30%（行业/主题） | 建议减仓50% |
| 盈利 > 30%（宽基）/ 50%（行业/主题） | 建议清仓 |
| 估值分位 > 80% | 建议减仓（无论盈亏） |

**信号E：止损退出**

| 条件 | 建议 |
|------|------|
| 亏损 > 30% AND 估值分位 > 50%（非低估） | 建议止损50% |
| 亏损 > 40% AND 基本面恶化（经理变更/规模暴增） | 建议清仓 |
| 从最高点回撤 > 25% | 建议止损 |

**信号F：暂停观望**

| 条件 | 建议 |
|------|------|
| 资金池已耗尽 | 暂停所有补仓，等待修复 |
| 估值分位 > 70% | 暂停金字塔和定投 |
| 连续3次补仓后仍下跌 | 暂停该标的补仓，重新评估 |

**实施方式**：
- 新增 `_detect_exit_signals()` 函数
- 计划卡新增"退出信号"区块（红色高亮）
- 退出信号不自动转决策候选（需用户确认）
- 配置开关：`smart_add.exit_signal_enabled`（默认false）

### 4.3 方案二：价值平均法引擎（P1）

作为引擎1的替代选项，用户可在"DCA"和"VA"之间切换。

**核心逻辑**：

```python
def _calc_value_averaging(holding, target_monthly_growth, current_month):
    """价值平均法：计算本期目标投入。
    
    target_value = initial_value + current_month × target_monthly_growth
    actual_value = holding.current_value
    required_investment = target_value - actual_value
    
    Returns:
        > 0: 需要买入
        < 0: 建议卖出/减少
        = 0: 维持不变
    """
```

**参数**：
- `target_monthly_growth`：每月目标市值增长额（默认 = 基础月投）
- `max_monthly_investment`：单月最大投入（防止极端行情超额投入）
- `max_monthly_withdrawal`：单月最大撤出（防止过度卖出，默认0即不卖）

**与当前DCA的对比展示**：

| 场景 | DCA投入 | VA投入 | 说明 |
|------|---------|--------|------|
| 市值等于目标 | ¥2,760 | ¥2,760 | 无差异 |
| 市值低于目标10% | ¥2,760 | ¥5,520 | VA自动加大 |
| 市值高于目标10% | ¥2,760 | ¥0 | VA自动暂停 |
| 市值高于目标20% | ¥2,760 | -¥2,760 | VA建议卖出 |

**配置项**：
- `smart_add.va_enabled`：VA引擎开关（默认false）
- `smart_add.va_target_growth_pct`：目标月增长%（默认=base_dca_pct/12）
- `smart_add.va_max_monthly_mult`：单月最大投入倍数（默认3.0，即最多3倍月投）
- `smart_add.va_allow_sell`：是否允许卖出建议（默认false，保守）

### 4.4 方案三：网格交易策略（P1）

作为金字塔补仓的补充，适用于估值处于合理区间（30-70%分位）的标的。

**核心逻辑**：

```python
def _generate_grid_plan(holding, valuation, cfg):
    """生成网格交易计划。
    
    1. 确定网格区间：[当前价×0.8, 当前价×1.2]
    2. 划分N格（默认5格）
    3. 每格买入/卖出金额 = 标的市值 / N
    4. 当前价在网格中的位置 → 建议操作
    """
```

**触发条件**：
- 估值分位 30-70%（合理区间）
- 非深度亏损（亏损 > -15%则不适合网格，应优先金字塔）
- 非债券基金

**配置项**：
- `smart_add.grid_enabled`：网格策略开关（默认false）
- `smart_add.grid_count`：网格数量（默认5）
- `smart_add.grid_range_pct`：网格区间%（默认±20%）
