# LLM 评估 Agent 与进化系统设计

> 日期：2026-06-13 | 状态：✅ 已完成

## 一、目标

设计一个**通用的 LLM 评估 Agent**，用于：
1. 评估所有 LLM 输出的质量（对话、日报、热点、持仓分析等）
2. 结合进化系统，让 Agent 更好地理解用户偏好
3. 形成"评估 → 学习 → 优化"的闭环

### 3.1 LLM 评估 Agent 设计

#### Agent 定义

```python
# backend/agent/llm_evaluator_agent.py

LLM_EVALUATOR_SYSTEM_PROMPT = """
你是专业的 LLM 输出质量评估专家。你的任务是评估 AI 生成的投资分析内容的质量。

## 评估维度

### 1. 数据准确性 (25%)
- 数据来源是否清晰？
- 数字是否准确、是否有单位？
- 是否引用了具体的估值指标（PE/PB/百分位）？

### 2. 逻辑一致性 (25%)
- 分析逻辑是否自洽？
- 结论是否与论据匹配？
- 是否存在矛盾或跳跃？

### 3. 可执行性 (25%)
- 建议是否具体？
- 是否有明确的操作方向（买入/卖出/持有/等待）？
- 是否有时间/比例/仓位建议？

### 4. 用户理解 (25%)
- 是否回答了用户的核心问题？
- 是否考虑了用户的持仓情况？
- 是否考虑了用户的风险偏好？

## 输出格式

{
    "total_score": 0-100,
    "dimensions": {
        "data_accuracy": {"score": 0-100, "evidence": "具体证据", "issues": ["问题1"]},
        "logic_consistency": {"score": 0-100, "evidence": "具体证据", "issues": ["问题1"]},
        "actionability": {"score": 0-100, "evidence": "具体证据", "issues": ["问题1"]},
        "user_understanding": {"score": 0-100, "evidence": "具体证据", "issues": ["问题1"]}
    },
    "strengths": ["优点1", "优点2"],
    "weaknesses": ["问题1", "问题2"],
    "suggestions": ["建议1", "建议2"],
    "user_preference_hints": ["用户可能偏好...", "用户可能不喜欢..."]
}
"""
```

### 3.2 进化系统设计

#### 数据流

```
┌─────────────────────────────────────────────────────────────┐
│                     LLM 评估与进化闭环                       │
└─────────────────────────────────────────────────────────────┘

用户交互
    │
    ├─→ LLM 输出 ─→ LLM 评估 Agent ─→ 评估结果
    │                                      │
    │                                      ├─→ 低分(<60) → Bad Case + 反馈学习
    │                                      ├─→ 中分(60-80) → 记录 + 分析模式
    │                                      └─→ 高分(≥80) → 正面模式提取
    │
    └─→ 用户反馈 ─→ 反馈学习 ─→ 用户画像更新
                                      │
                                      ↓
                              下次 LLM 调用时注入
                              - 用户偏好
                              - 历史问题模式
                              - 成功案例参考
```

#### 学习机制

**1. 用户偏好学习**

```python
# 从评估结果中提取用户偏好
def extract_user_preferences(evaluation: dict) -> dict:
    """从评估结果中提取用户偏好信号"""
    preferences = {}

    # 用户理解维度低分 → 用户可能有特殊需求
    if evaluation["dimensions"]["user_understanding"]["score"] < 60:
        issues = evaluation["dimensions"]["user_understanding"]["issues"]
        for issue in issues:
            if "持仓" in issue:
                preferences["关注持仓"] = True
            if "风险" in issue:
                preferences["风险敏感"] = True

    # 可执行性维度低分 → 用户可能需要更具体的建议
    if evaluation["dimensions"]["actionability"]["score"] < 60:
        preferences["需要具体建议"] = True

    return preferences
```

**2. 失败模式学习**

```python
# 从低分评估中提取失败模式
def extract_failure_patterns(evaluations: list) -> list:
    """从多次低分评估中提取系统性失败模式"""
    patterns = []

    # 统计常见问题
    issue_counts = {}
    for eval in evaluations:
        for dim in eval["dimensions"].values():
            for issue in dim.get("issues", []):
                issue_counts[issue] = issue_counts.get(issue, 0) + 1

    # 提取高频问题
    for issue, count in sorted(issue_counts.items(), key=lambda x: -x[1]):
        if count >= 3:  # 出现3次以上
            patterns.append({
                "pattern": issue,
                "frequency": count,
                "suggestion": f"需要改进：{issue}"
            })

    return patterns
```

**3. 成功案例学习**

```python
# 从高分评估中提取成功模式
def extract_success_patterns(evaluations: list) -> list:
    """从高分评估中提取成功模式"""
    patterns = []

    for eval in evaluations:
        if eval["total_score"] >= 80:
            # 提取优点
            for strength in eval.get("strengths", []):
                patterns.append({
                    "pattern": strength,
                    "score": eval["total_score"],
                    "context": eval.get("context", "")
                })

    return patterns
```

### 3.3 Prompt 注入机制

```python
def build_eval_aware_prompt(base_prompt: str, user_id: str = "default") -> str:
    """构建评估感知的 prompt"""

    # 1. 获取用户偏好
    from agent.feedback_learner import get_preference_context
    pref_ctx = get_preference_context(user_id)

    # 2. 获取历史失败模式
    failure_patterns = get_recent_failure_patterns(user_id, limit=5)
    failure_ctx = ""
    if failure_patterns:
        failure_ctx = "## 历史问题（请避免）\n"
        for p in failure_patterns[:3]:
            failure_ctx += f"- {p['pattern']}\n"

    # 3. 获取成功案例
    success_patterns = get_recent_success_patterns(user_id, limit=3)
    success_ctx = ""
    if success_patterns:
        success_ctx = "## 成功案例（请参考）\n"
        for p in success_patterns[:2]:
            success_ctx += f"- {p['pattern']}\n"

    # 4. 组合 prompt
    return f"""{base_prompt}

{pref_ctx}

{failure_ctx}

{success_ctx}

请特别注意以上用户偏好和历史问题，提供更符合用户需求的分析。"""
```

### 3.4 数据库设计

```sql
-- LLM 评估记录表
CREATE TABLE IF NOT EXISTS llm_evaluations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    -- 评估目标
    target_type TEXT NOT NULL,  -- conversation / daily_report / hot_topics / portfolio
    target_id INTEGER,          -- 目标 ID（conversation_id 或其他）
    message_id INTEGER,         -- 消息 ID（对话场景）

    -- 评估结果
    total_score REAL,
    dimensions_json TEXT,       -- JSON: {data_accuracy: {score, evidence, issues}, ...}
    strengths_json TEXT,        -- JSON: ["优点1", "优点2"]
    weaknesses_json TEXT,       -- JSON: ["问题1", "问题2"]
    suggestions_json TEXT,      -- JSON: ["建议1", "建议2"]

    -- 用户理解洞察
    user_preference_hints TEXT, -- JSON: ["用户可能偏好..."]

    -- 元数据
    evaluator_version TEXT DEFAULT 'v1',
    duration_ms INTEGER,
    token_usage INTEGER,

    created_at TEXT DEFAULT (datetime('now','localtime'))
);

-- 用户偏好学习表
CREATE TABLE IF NOT EXISTS user_preference_learnings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id TEXT DEFAULT 'default',
    preference_key TEXT NOT NULL,
    preference_value TEXT,
    source TEXT,                -- 评估来源
    confidence REAL DEFAULT 0.5,
    created_at TEXT DEFAULT (datetime('now','localtime')),
    updated_at TEXT DEFAULT (datetime('now','localtime'))
);

-- 失败模式表
CREATE TABLE IF NOT EXISTS failure_patterns (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    pattern TEXT NOT NULL,
    frequency INTEGER DEFAULT 1,
    last_seen TEXT DEFAULT (datetime('now','localtime')),
    suggestion TEXT,
    created_at TEXT DEFAULT (datetime('now','localtime'))
);
```
