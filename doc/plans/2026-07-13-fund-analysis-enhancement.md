# 基金深度分析增强 — 段永平投资理念映射

> 基于段永平"买股票就是买公司、长期持有好公司、拐点加仓"理念，从单一估值判断升级为多维决策体系。

## 设计理念

### 段永平核心原则 → 系统映射

| 段永平原则 | 传统做法 | 系统增强后 |
|-----------|---------|-----------|
| 买股票就是买公司 | 只看估值分位 | + 基金质量评分（经理/费率/规模/跟踪误差） |
| 长期持有好公司 | 只看当前涨跌 | + 趋势均线系统（120/250/500日趋势状态） |
| 拐点加仓 | 只看"低估" | + 回撤恢复周期（历史回撤分位+恢复能力） |
| 别人恐惧我贪婪 | 无情绪判断 | + 情绪温度计（恐贪指数+拥挤度） |
| 看懂生意模式 | 无资金面验证 | + 资金流向信号（融资融券+ETF份额） |
| 综合判断 | 单维度决策 | + 持仓健康度总评（六维体检报告） |

### 决策升级前后对比

```
旧决策链：估值低 → 买 / 估值高 → 卖

新决策链：
  基金质量好？──否──→ 跳过，不持有
       │是
  趋势方向？──下行──→ 等待拐点信号
       │上行/震荡
  回撤位置？──高位──→ 不加仓
       │中低位
  资金信号？──流出──→ 谨慎，缩减仓位
       │流入/中性
  情绪温度？──极贪──→ 分批止盈
       │极恐/中性
  → 执行加仓/持有/减仓
```

## P0-B: 回撤恢复周期分析

### 目标

回答"现在是不是拐点"——段永平的"即使跌也会在拐点加仓"。

### 评分模型（满分100）

| 指标 | 权重 | 分数计算 | 数据源 |
|------|------|---------|--------|
| 当前回撤深度 | 30 | 距高点>40%:30 / >25%:22 / >15%:15 / >5%:8 / <5%:3 | `fund_nav_history` |
| 回撤分位 | 25 | 处于历史最大95%+:25 / 80%+:18 / 50%+:12 / 20%+:6 / <20%:2 | `fund_nav_history` |
| 恢复能力 | 25 | 历史平均恢复<60天:25 / <120天:18 / <250天:12 / >250天:5 | `fund_nav_history` |
| 回撤速度 | 10 | 急跌(月跌>15%):10 / 缓跌:5 / 未跌:3 | `fund_nav_history` |
| 底部信号 | 10 | 净值企稳(近5日波动<1%):10 / 仍在跌:3 / 创新高:0 | `fund_nav_history` |

### 核心算法

```python
def calculate_drawdown_analysis(nav_history: list[dict]) -> dict:
    """
    输入: [{date, nav}, ...] 按日期升序
    输出: {
        current_drawdown: 0.18,        # 当前回撤深度
        max_drawdown: 0.45,            # 历史最大回撤
        drawdown_percentile: 0.85,     # 当前回撤在历史中的分位
        recovery_periods: [60, 120, 90], # 历史每次回撤的恢复天数
        avg_recovery_days: 90,         # 平均恢复天数
        is_bottoming: False,           # 是否在企底
        recent_volatility: 0.023,      # 近5日波动率
    }
    """
    # 1. 计算每个时点的回撤: drawdown[i] = 1 - nav[i] / max(nav[0..i])
    # 2. 当前回撤 = 最后一个时点的回撤
    # 3. 历史最大回撤 = max(drawdown)
    # 4. 回撤分位 = 当前回撤在所有回撤值中的排名
    # 5. 恢复周期: 找到每次回撤的起点→恢复到前高的天数
    # 6. 企底判断: 近5日净值波动率 < 1%
```

### 回撤恢复周期识别算法

```
回撤周期定义：
  高点 → 回撤谷底 → 恢复到前高

算法：
  1. 遍历净值序列，记录每个高点（局部最大值）
  2. 对每个高点，找到后续的最低点（谷底）
  3. 计算高点到谷底的天数（下跌周期）
  4. 计算谷底恢复到高点净值的天数（恢复周期）
  5. 如果未恢复到前高，标记为"未恢复"

示例：
  净值: 1.0 → 0.6 → 1.0 → 0.8 → 1.2 → 0.9
  周期1: 高点1.0 → 谷底0.6(跌40%) → 恢复1.0(恢复90天)
  周期2: 高点1.2 → 谷底0.9(跌25%) → 未恢复
```

### 输出格式

```json
{
  "fund_code": "161725",
  "fund_name": "招商中证白酒指数C",
  "drawdown_score": 68,
  "rating": "good",
  "dimensions": {
    "current_drawdown": {"score": 22, "weight": 0.30, "reason": "距高点-28%,中等回撤"},
    "drawdown_percentile": {"score": 18, "weight": 0.25, "reason": "处于历史80%分位"},

    "bottoming_signal": {"score": 5, "weight": 0.10, "reason": "仍在下跌,未企稳"}
  },
  "detail": {
    "current_drawdown": -0.28,
```

## P2-A: 情绪温度计

### 目标

回答"市场情绪如何"——段永平的"别人恐惧我贪婪"。

### 评分模型（满分100）

| 指标 | 权重 | 分数计算 | 数据源 |
|------|------|---------|--------|
| 换手率分位 | 25 | 处于1年低位(恐慌):25 / 中位:12 / 高位(过热):5 | `ak.stock_zh_a_spot_em` |
| 涨跌家数比 | 20 | 跌停>50家:20 / 上涨<30%:15 / 均衡:10 / 涨停>50:3 | `get_market_overview` |
| 波动率分位 | 20 | 波动率1年高位(恐慌):20 / 中位:10 / 低位:5 | 指数历史 |
| 热度词频 | 15 | 负面词频上升:15 / 正常:8 / 正面词频上升:3 | 盈米热点+东方财富 |
| 恐贪综合 | 20 | 极恐(0-20):20 / 恐(20-40):15 / 中性(40-60):10 / 贪(60-80):5 / 极贪(80+):0 | 综合计算 |

### 恐贪指数算法

```python
def calculate_fear_greed_index(market_data: dict) -> int:
    """
    恐贪指数 = 加权平均(换手率分位反转 + 涨跌比反转 + 波动率分位 + 新闻情绪)
    返回 0-100，0=极度恐惧，100=极度贪婪

    逻辑（逆向指标）：
    - 换手率越低 → 越恐惧 → 分数越低

    - 负面新闻越多 → 越恐惧 → 分数越低
    """
```

## P2-B: 持仓健康度总评

### 目标

把前5个维度 + 现有估值系统整合成"基金体检报告"，形成最终决策建议。

### 评分模型

```
综合评分 = 质量评分 × 0.20
         + 回撤评分 × 0.20
         + 趋势评分 × 0.15
         + 资金评分 × 0.15
         + 情绪评分 × 0.10
         + 估值评分 × 0.20   ← 现有估值系统
```

### 段永平决策矩阵

```
                    估值低分位    估值中分位    估值高分位
质量优秀+趋势向上      强烈加仓      持有          持有
质量优秀+趋势不明      定投加仓      持有          减仓
质量优秀+趋势向下      等待拐点      持有          减仓
质量一般+任何趋势      观望          观望          减仓
```

### 输出格式（体检报告）

```json
{
  "fund_code": "161725",
  "fund_name": "招商中证白酒指数C",
  "total_score": 68,
  "rating": "good",
  "report": {
    "quality": {"score": 75, "rating": "good", "label": "基金质量"},
    "drawdown": {"score": 68, "rating": "good", "label": "回撤恢复"},
    "trend": {"score": 52, "rating": "fair", "label": "趋势均线"},
    "capital": {"score": 65, "rating": "good", "label": "资金流向"},
    "sentiment": {"score": 35, "rating": "fair", "label": "情绪温度"},
    "valuation": {"score": 82, "rating": "excellent", "label": "估值水位"}
  },
  "decision_matrix": {
    "quality_rating": "good",
    "trend_direction": "tangled",
    "valuation_level": "low",
    "action": "dca",           // strong_buy / dca / hold / reduce / wait
    "action_label": "定投加仓",
    "reason": "质量良好+估值偏低+趋势不明,适合定投等待趋势确立"
  },
  "duan_yongping_view": "好生意(消费龙头)+好价格(低估)+趋势待确立,适合分批建仓",
  "advice": "基金质量良好,估值偏低,但趋势尚未确立。建议定投方式加仓,等待均线多头排列后加大仓位。"
}
```

### action 映射

| action | 含义 | 触发条件 |
|--------|------|---------|
| strong_buy | 强烈加仓 | 质量≥good + 趋势上行 + 估值低 + 情绪偏恐 |
| dca | 定投加仓 | 质量≥good + 估值低 + (趋势不明 OR 回撤高位) |
| hold | 持有 | 质量≥good + 估值中 OR 趋势上行 + 估值中 |
| reduce | 减仓 | 估值高 OR 质量差 OR 趋势下行+估值中 |
| wait | 等待 | 趋势下行 + 回撤未企稳 + 估值不低 |

## API 设计

### 获取单基金六维分析

```
GET /api/analysis/fund-quality/{fund_code}
```

**参数：**
- `force_refresh`: bool, 是否强制重新计算（默认false，用缓存）

**响应：** 体检报告JSON（见P2-B输出格式）

### 批量获取持仓基金分析

```
POST /api/analysis/fund-quality/batch
Body: {"fund_codes": ["161725", "005827", ...]}
```

**响应：** `{"results": [{fund_code, total_score, rating, action, ...}, ...]}`

### 刷新评分

```
POST /api/analysis/fund-quality/refresh
Body: {"fund_codes": ["161725"]}  # 可选,不传则刷新所有持仓+关注
```

## 风险与降级

| 风险 | 降级方案 |
|------|---------|
| akshare 接口限流 | 5分钟缓存 + 跳过该维度 + 权重重分配 |
| ETF份额接口不可用 | 跳过，权重重分配到融资余额 |
| 北向资金不可用 | 不使用北向资金，改用融资融券 |
| 净值历史不足250天 | MA250/MA500返回None，趋势评分降级 |
| 情绪数据源不足 | 返回中性分50，标记"数据不足" |
| 龙虎榜无数据 | 跳过，权重重分配 |

## 与现有系统的关系

| 现有功能 | 关系 |
|---------|------|
| 估值系统（valuation.py） | 估值评分作为第6维度输入，不修改现有逻辑 |
| 健康分引擎（health_score.py） | 组合级总评复用现有框架，基金级是新功能 |
| 买入评分（buy-score） | 买入评分关注"上车时机"，六维评分关注"基金质量"，互补关系 |
| 事件雷达 | 事件影响短期波动，六维评分关注长期质量，时间维度互补 |
| 关注列表 | 六维评分结果展示在关注卡片上，增强决策信息 |
| 预警系统 | 六维评分异常（如质量下降、趋势反转）可触发预警 |
