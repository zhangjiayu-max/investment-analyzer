### 2.1 健康分系统（health_score.py）

#### 问题 1：选品质量评分逻辑错误 🔴

**位置：** `calc_quality_score()`  
**现状：**
```python
# 用 PE/PB 分位判断"选品质量"
if avg_pct < 30:    # PE分位 < 30%
    total += 180    # 高分 → "好选品"
elif avg_pct < 90:
    total += 80
else:
    total += 50     # 低分 → "坏选品"
```

**问题分析：**  
这个逻辑把"买入时机"和"选品质量"混淆了。PE 分位低只能说明"你现在买入的时机好"，不能说明"这只基金本身质地好"。一只跟踪误差 5% 的烂指数基金在 PE 分位 20% 时买入，评分比一只跟踪误差 0.1% 的优质基金在 PE 分位 50% 时买入更高——这不合理。

**正确做法：**  
选品质量应该评估基金本身的质地：
- 同类排名分位（akshare 有 `fund_rank_em` 接口）
- 基金经理任职稳定性（>3年 = 高分）
- 跟踪误差（指数基金 < 2% = 高分）
- 信息比率 / Sharpe（如果有数据）
- 费率水平（同类最低 25% = 高分）

**修复方案：**
```python
def calc_quality_score(holdings_with_val):

    # 2. 基金经理稳定性（40分）
    # 3. 跟踪误差/信息比率（40分）
    # 4. 费率水平（40分）
```

#### 问题 2：持有行为评分使用硬编码假数据 🔴

**位置：** `calc_behavior_score()` 第3项  
**现状：**
```python
# 3. 平均持有时长（60分）
avg_days = 180  # 默认假设半年 ← 硬编码！
if avg_days >= 365:
    score += 60
elif avg_days >= 180:
    score += 45  # ← 永远走这个分支
```

**问题分析：**  
`avg_days = 180` 是硬编码的，意味着每次计算都会给 45 分（因为 180 >= 180），不管用户真实持有时长是多少。这 60 分的权重形同虚设。

**正确做法：**  
从 `portfolio_snapshots` 表或交易记录推算真实持有时长：
```python
# 方案A：从持仓快照推算
snapshots = list_portfolio_snapshots(limit=365)
# 计算每只基金从首次出现到现在的天数，取中位数

# 方案B：从交易记录推算
trades = list_transactions()
# 买入日期 → 现在（或卖出日期）的天数
```

**工作量：** 2小时

#### 问题 6：北向资金公式敏感度极低 🟡

**位置：** `calc_fear_greed_index()`  
**现状：**
```python
factors["north_flow"] = max(0, min(100, (net_flow / 100 + 50)))
```

**问题分析：**  
如果近 5 日北向净流入 50 亿，结果 = 50.5（中性）。如果净流出 50 亿，结果 = 49.5（还是中性）。这个公式对北向资金几乎不敏感。实际上北向单日净流入/流出 50 亿是很大的信号。

**正确做法：**  
用标准差归一化：
```python
# 获取近60日北向资金数据
df = ak.stock_hsgt_north_net_flow_in_em(symbol="北上")
flows = [_safe_float(r.get("净流入", 0)) for _, r in df.tail(60).iterrows()]
mean_flow = sum(flows) / len(flows)
std_flow = (sum((f - mean_flow)**2 for f in flows) / len(flows)) ** 0.5

# 近5日净流入的 z-score
recent_flow = sum(flows[-5:])
z_score = (recent_flow - mean_flow * 5) / (std_flow * 5**0.5) if std_flow > 0 else 0
# z-score 映射到 0-100：-2→0, 0→50, +2→100
factors["north_flow"] = max(0, min(100, (z_score / 4 + 0.5) * 100))
```

**工作量：** 1小时

### 2.3 滚动收益分析（rolling_return.py）

#### 问题 8：组合净值模拟是假的 🔴🔴

**位置：** `_get_portfolio_nav_simulated()`  
**现状：**
```python
def _get_portfolio_nav_simulated(holdings, days=1825):
    """模拟组合净值（按持仓加权）。简化版：用第一个持仓基金的净值。"""
    for h in holdings:
        code = h.get("fund_code", "")
        if code:
            nav = _get_fund_nav_from_akshare(code, days)
            if nav:
                return nav  # ← 只取第一只基金！
    return []
```

**问题分析：**  
这是整个系统最严重的问题。函数名叫"组合净值模拟"，注释说"按持仓加权"，实际只取第一只基金的净值。如果用户持有 10 只基金，分析结果只反映其中一只的历史表现。**用户看到的"我的持仓组合滚动收益"是假的。**

**正确做法：**
```python
def _get_portfolio_nav_weighted(holdings, days=1825):
    """按持仓权重加权计算组合净值。"""
    # 1. 获取每只基金的净值序列
    fund_navs = {}
    for h in holdings:
        code = h.get("fund_code", "")
        if code:
            nav = _get_fund_nav_from_akshare(code, days)
            if nav:
                fund_navs[code] = nav
    
    # 2. 对齐日期（取所有基金共有的日期范围）
    all_dates = set()
    for nav in fund_navs.values():
        all_dates.update(d[0] for d in nav)
    all_dates = sorted(all_dates)
    
    # 3. 按权重加权计算每日组合净值
    total_value = sum(h.get("current_value", 0) or 0 for h in holdings)
    weights = {h.get("fund_code"): (h.get("current_value", 0) or 0) / total_value 
               for h in holdings if h.get("fund_code") in fund_navs}
    
    portfolio_nav = []
    for date in all_dates:
        daily_value = 0
        for code, nav_series in fund_navs.items():
            # 找到该日期对应的净值（或最近的前一个净值）
            matching = [n for n in nav_series if n[0] <= date]
            if matching:
                daily_value += matching[-1][1] * weights.get(code, 0)
        if daily_value > 0:
            portfolio_nav.append((date, daily_value))
    
    return portfolio_nav
```

**工作量：** 4小时

#### 问题 16：Token 估算可能低估中文 🟢

**位置：** `estimate_text_tokens()`  
**现状：**
```python
non_space_chars = len(re.sub(r"\s+", "", text))
return max(ascii_words, non_space_chars // 2)
```

**问题分析：**  
中文约 1.5 字符/token，`non_space_chars // 2` 对纯中文文本会低估约 25%。这意味着 token 预算可能在中文场景下超限。

**修复方案：**
```python
cn_chars = sum(1 for c in text if '\u4e00' <= c <= '\u9fff')
en_chars = len(text) - cn_chars
return int(cn_chars / 1.5 + en_chars / 4)
```

### 2.7 相关性矩阵（correlation.py）

#### 问题 17：不同基金的净值日期未对齐 🟡

**位置：** `_build_correlation_matrix()`  
**现状：**
```python
returns = {}
for code, series in nav_data.items():
    returns[code] = _calc_daily_returns(series)
```

**问题分析：**  
不同基金的净值日期可能不同（QDII 基金 T+1、A 股基金 T 日）。直接对各自的日收益率序列算相关系数，日期没有对齐，结果可能偏差。

**修复方案：**
```python
# 先对齐日期，只保留所有基金都有数据的交易日
all_dates = set.intersection(*[set(d[0] for d in series) for series in nav_data.values()])
aligned_returns = {}
for code, series in nav_data.items():
    date_nav = {d[0]: d[1] for d in series}
    aligned = [(date_nav[d], date_nav.get(prev_date, date_nav[d])) 
               for d, prev_date in zip(sorted(all_dates)[1:], sorted(all_dates)[:-1])]
    aligned_returns[code] = [(curr - prev) / prev for curr, prev in aligned if prev > 0]
```

**工作量：** 2小时

### 2.9 估值分析（valuation.py）

#### 问题 19：估值分析以个股为主，基金分析较弱 🟡

**位置：** `analyze_stock()` vs `analyze_fund()`  
**现状：** `analyze_stock` 有完整的价格统计、PE/PB 评分、融合层。`analyze_fund` 主要依赖净值数据，没有和指数估值关联。

**问题分析：**  
对于基金投资者，更有用的是"这只基金跟踪的指数估值如何"，而不是"这只基金净值走势如何"。当前基金分析没有充分利用 `index_valuations` 表的数据。

**修复方案：**  
在 `analyze_fund` 中增加：
