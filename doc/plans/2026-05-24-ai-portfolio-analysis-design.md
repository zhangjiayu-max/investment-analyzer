# AI 持仓分析模块重新设计

## 背景

当前 AI 分析模块只有一个文本输入框 + 一段文字回答，用户不知道能问什么，AI 输出也没有结构。
持仓页已有的"分散度分析"和"交易行为分析"是固定规则计算，缺乏 AI 的深度解读和个性化建议。

## 目标

将 AI 分析从"一个输入框"升级为**四个分析模式 + 主动事件驱动**，让用户能：
- 一键了解组合整体健康状况
- 深入分析单只基金的投资价值
- 复盘自己的交易操作质量
- 推演不同市场情景下的组合变化

## 一、模式 1：全景诊断

### 入口
持仓页顶部大按钮「🔍 全景诊断」，点击即触发，无需输入问题。

### 数据输入
| 数据源 | 内容 |
|--------|------|
| 持仓数据 | 所有持仓的基金代码、份额、成本、市值、盈亏 |
| 账户信息 | 花无缺/小鱼儿各自的持仓分布 |
| 分散度分析 | 基金类型分布、指数分布、仓位集中度 |
| 估值数据 | 各持仓跟踪指数的 PE/PB 百分位、z-score |
| MCP 数据 | 资产穿透分析、基金相关性、行业配置、市场行情 |
| 新闻热点 | 近期的财经新闻和政策（通过 SearchHotTopic / SearchFinancialNews） |

### AI 分析内容
1. **整体风险评估**
   - 组合集中度评分（前 3 大持仓占比是否过高）
   - 基金相关性风险（是否同涨同跌）
   - 估值水位风险（是否整体处于高估区域）
   - 账户对比（花无缺 vs 小鱼儿的风险差异）

2. **当前市场环境适配度**
   - 当前市场风格（成长/价值、大盘/小盘）
   - 组合是否适应当前市场
   - 需要关注的 3 件最重要的事

3. **加减仓建议**
   - 按基金列出建议（增配/持有/减配/清仓）
   - 带具体理由（估值原因/集中度原因/相关性原因）
   - 建议的优先级排序

### 输出结构
```
┌─────────────────────────────────────────┐
│  组合健康评分: 72/100                    │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐  │
│  │ 风险 │ │ 估值 │ │ 分散 │ │ 适配 │  │
│  │ 中 ⚠️│ │ 偏低 │ │ 良好 │ │ 一般 │  │
│  └──────┘ └──────┘ └──────┘ └──────┘  │
├─────────────────────────────────────────┤
│  ## 当前市场环境                        │
│  (AI 生成的简要市场分析)                │
│                                         │
│  ## 组合风险评估                        │
│  - 集中度：前3大持仓占比XX%，...        │
│  - 相关性：XX 和 XX 高度相关（0.9+）   │
│  - 估值：整体处于XX分位                │
│                                         │
│  ## 加减仓建议                          │
│  | 基金 | 操作 | 紧急度 | 理由 |       │
│  | 白酒 | 暂持 | 低   | 估值极低 |    │
│  | 医疗 | 减仓 | 中   | 超配+集中 |   │
└─────────────────────────────────────────┘
```

## 二、模式 2：单基金深度分析

### 入口
持仓列表每行右侧新增「分析」按钮，点击打开单基金分析弹窗/面板。

### 数据输入
| 数据源 | 内容 |
|--------|------|
| 该基金持仓数据 | 份额、成本、市值、盈亏、持有时间 |
| 该基金交易记录 | 所有买入/卖出历史 |
| 基金基本面 | 通过 akshare 查询的重仓股、行业配置、资产配置 |
| 估值数据 | 跟踪指数的 PE/PB 历史分位 |
| MCP 数据 | 该基金的行业穿透数据、诊断结果 |
| 同类基金对比 | 同类型基金的收益排名（可选） |

### AI 分析内容
1. **角色定位**
   - 该基金在组合中的角色（核心/卫星/战术配置）
   - 是否与其他基金功能重叠

2. **持有期收益质量分析**
   - 总持有时间
   - 年化收益率（简单年化，非 IRR）
   - 最大浮亏 / 最大浮盈
   - 买入时估值分位 vs 当前估值分位（买贵了还是买便宜了）

3. **操作质量评估**
   - 各笔买入的估值位置评估
   - 各笔卖出的估值位置评估（卖飞了还是及时止盈？）
   - 整体操作评分（A/B/C/D）

4. **建议**
   - 继续持有 / 加仓 / 减仓 / 清仓
   - 带具体估值和仓位理由
   - 如果建议卖出，给出参考止盈/止损线

### 输出结构
```
┌─────────────────────────────────────────┐
│  招商中证白酒指数A  评分: 78/100        │
│  (建议: 暂持不追)                       │
├─────────────────────────────────────────┤
│  角色: 核心持仓 · 持有 180天            │
│  成本: 1.2000 · 现价: 0.8234           │
│  盈亏: -31.4% · 年化: -48.2%           │
├─────────────────────────────────────────┤
│  买入质量                   ★★★☆☆      │
│  - 首笔买入时估值分位 45%（适中）       │
│  - 后续补仓时估值分位持续走低 ✅        │
│  - 平均成本逐步降低，定投效果显著       │
│                                         │
│  当前估值                                 │
│  - PE 分位: 8.5%（极度低估）            │
│  - 此时减仓可能卖在底部                 │
│                                         │
│  建议: 暂停定投但不减仓，等待估值修复    │
└─────────────────────────────────────────┘
```

## 三、模式 3：交易复盘

### 入口
分析面板内的「交易复盘」tab，或交易历史旁边的"复盘分析"入口。

### 数据输入
| 数据源 | 内容 |
|--------|------|
| 交易记录 | 所有买入/卖出流水（含 pending/confirmed） |
| 交易标签 | 用户为每笔交易打的标签（追涨/抄底/定投等） |
| 估值历史 | 各指数在交易时点的估值分位 |
| 持仓变化 | 交易前后的持仓变化 |

### AI 分析内容
1. **买入时机评估**
   - 去重统计各基金的买入操作
   - 每笔买入时对应指数的估值分位
   - 买入时机评分（低估区域买入 = 好，高估区域买入 = 差）
   - 买入频率评估（太频繁说明可能情绪化）

2. **卖出时机评估**
   - 每笔卖出时对应指数的估值分位
   - 卖出后该基金走势（卖对了还是卖飞了？）
   - 止盈/止损纪律执行情况

3. **操作模式识别**
   - 是否存在追涨杀跌行为
   - 定投纪律评分（是否按期执行）
   - 是否频繁操作（交易成本影响）
   - 情绪化交易识别（大涨后追买、大跌后恐慌卖）

4. **改进建议**
   - 具体的行为改进建议
   - 可执行的纪律规则

### 输出结构
```
┌─────────────────────────────────────────┐
│  交易复盘报告  2026-05                 │
├─────────────────────────────────────────┤
│  操作总览                               │
│  买入 8 笔 · 卖出 3 笔 · 净投入 XX     │
│                                         │
│  买入质量                ★★★★☆          │
│  - 80% 的买入在估值适中/偏低区域 ✅     │
│  - 20% 的买入在偏高区域（白酒 2月）⚠️  │
│  - 建议：高估区域暂停定投               │
│                                         │
│  卖出质量                ★★☆☆☆          │
│  - 医药减仓后反弹 12%（卖早了）         │
│  - 互联网减仓时机较好 ✅                │
│                                         │
│  行为模式                               │
│  - 偏好行业主题基金，集中度偏高         │
│  - 下跌时敢于加仓 ✅                    │
│  - 上涨时持有耐心不足 ⚠️               │
├─────────────────────────────────────────┤
│  改进计划                               │
│  1. 设定止盈目标线（如 +20%）           │
│  2. 高估区域执行分批减仓而非一次性      │
└─────────────────────────────────────────┘
```

## 四、模式 4：情景推演

### 入口
分析面板内的「情景推演」tab。

### 数据输入
| 数据源 | 内容 |
|--------|------|
| 持仓数据 | 当前所有持仓的份额、成本 |
| 估值数据 | 各指数当前点位、分位、历史极值 |
| MCP 行情 | 市场最新行情数据 |

### 用户可选择的场景
1. **市场下跌 X%** — 用户输入跌幅，如 10%、20%
2. **估值修复到中位数** — AI 自动计算各指数修复到 PE 中位数的涨幅
3. **估值修复到机会值** — AI 自动计算修复到机会值的涨幅
4. **行业政策利好/利空** — 选择受影响的行业，模拟冲击

### AI 分析内容
1. **市场下跌场景**
   - 组合整体预估亏损
   - 受影响最大的前 3 只基金
   - 是否有对冲手段

2. **估值修复场景**
   - 各基金潜在涨幅
   - 组合整体预期收益
   - 如果修复到中位数，哪些弹性最大

3. **应对建议**
   - 当前仓位是否适合当前市场
   - 是否需要提前做一些准备
   - 不同情景下的最优应对策略

### 输出结构

┌─────────────────────────────────────────┐

│                                         │
│  受影响最大的基金:                      │
│  | 基金 | 预估亏损 | 占比 |            │

│  | 医疗 | -¥X,XXX | 20% |              │
│  | 互联网 | -¥X,XXX | 15% |            │
│                                         │
│  # 场景: 估值修复到历史中位数           │
│                                         │
│  组合预估收益: +¥XX,XXX (+X.X%)        │
│                                         │
│  弹性最大的基金:                        │
│  | 基金 | 当前分位 | 目标涨幅 |         │
│  | 白酒 | 8% | +35% |                  │

## 五、主动事件驱动

除了用户手动分析，系统在以下事件自动触发分析并生成预警：

| 事件 | 触发条件 | 分析内容 | 输出目标 |
|------|---------|---------|---------|
| 净值异动 | 单只基金单日跌 >3% 或涨 >5% | 异动归因 + 该基金当前估值 + 建议操作 | 预警列表 + Toast |
| 估值触发 | 持仓基金跟踪指数进入极度低估/高估区域 | 该基金在组合中的角色 + 历史操作回顾 + 建议 | 预警列表 |
| 月度回顾 | 每月 1 号 | 上月操作总结 + 本月计划 + 组合变化 | 分析面板历史 |
| 定投提醒 | 到达预设定投日 | 该基金当前估值 + 建议投/不投 + 金额建议 | 预警列表 |
| 持仓偏离 | 单只基金占比超过预设阈值（如 25%） | 集中度预警 + 减仓建议 | 预警列表 |

## 六、技术实现要点

### 后端

1. **新增 endpoint**
   - `POST /api/portfolio/analysis/panorama` — 全景诊断
   - `POST /api/portfolio/analysis/{holding_id}/deep-dive` — 单基金深度分析
   - `POST /api/portfolio/analysis/trade-review` — 交易复盘（可选日期范围）
   - `POST /api/portfolio/analysis/what-if` — 情景推演

2. **每个 endpoint 的逻辑**
   - 收集所需数据（持仓/交易/估值/MCP）
   - 构造结构化 prompt（不同模式用不同的 system prompt）
   - 调用 LLM
   - 保存到 `portfolio_analysis_records`（带 analysis_type 区分）
   - 返回结构化结果

3. **prompt 管理**
   - 在 `analysis_agents` 表中新增对应 agent（如 `id=3 为 全景诊断分析师`）
   - 每个模式的 prompt 可在后台配置和迭代

4. **估值历史查询优化**
   - 交易复盘需要查询交易时点的估值分位，需要按日期查询历史估值
   - `get_valuation_history()` 已支持，但按日期查询性能需关注

### 前端

1. **全景诊断**
   - 持仓页顶部新增「全景诊断」按钮
   - 加载时显示进度条（数据准备中 → AI 分析中 → 完成）
   - 结果以评分卡片 + 详细分析布局呈现

2. **单基金分析**
   - 持仓列表每行新增「分析」按钮
   - 打开侧边面板或弹窗
   - 评分卡 + 购买质量时间线 + 建议

3. **交易复盘**
   - 分析面板新增「交易复盘」tab
   - 可选的日期范围选择器
   - 操作评分 + 行为模式分析

4. **情景推演**
   - 分析面板新增「情景推演」tab
   - 场景选择按钮组 + 自定义参数
   - 结果以对比表格展示

5. **共用组件**
   - `AiScoreCard.vue` — 评分卡组件（标题 + 分数 + 颜色）
   - `AiSuggestionTable.vue` — 建议表格（基金 + 操作 + 理由）
   - `AiLoadingProgress.vue` — 分析进度条

## 七、实施步骤建议

### 第一阶段：核心能力
1. 全景诊断（整合现有分散度 + MCP + 估值 + 新闻数据）
2. 单基金分析（结合交易记录 + 估值历史）
3. `analysis_agents` 表中新增对应 agent

### 第二阶段：深度分析
4. 交易复盘（买入/卖出时机评估 + 行为模式识别）
5. 估值历史按日查询优化

### 第三阶段：高级功能
6. 情景推演
7. 主动事件驱动（净值异动自动分析）
8. 月度回顾自动生成

## 八、评分标准参考

为便于 AI 输出结构化的评分结果，各维度的评分规则：

| 维度 | 高分标准 | 低分标准 |
|------|---------|---------|
| 买入质量 | 80%+ 操作在估值偏低/适中区域 | 频繁在高估区域买入 |
| 卖出质量 | 止盈在合理位置，止损果断 | 卖在底部或死扛不止损 |
| 分散度 | 单只 <15%，行业分散 3+ 个 | 单只 >25%，集中于 1-2 个行业 |
| 估值健康 | 整体加权分位 20-50% | 整体加权分位 >80% 或 <5% |
| 情绪控制 | 按计划执行，无追涨杀跌 | 大涨后追买，大跌后恐慌卖 |
