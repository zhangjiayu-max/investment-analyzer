# 风险预警数据增强 — 合并 / 可靠性 / 财经新闻

> 设计稿日期：2026-07-05
> 关联前序：[2026-07-05-预警与建议逻辑增强.md](./2026-07-05-预警与建议逻辑增强.md)（8 项改进已完成）
> 用户反馈来源：用户查看风险预警数据后提出 3 个改进需求

## 一、用户需求与问题分析

### 用户原话
> 1、我看很多数据是重复的， 是否可以放在一条 然后用数量标注
> 2、预警的数据 可以标注可靠性吗， 我看很多都是按照低估来的
> 3、预警的数据， 一定要结合一些财经新闻， 和 市场走向分析

### 数据现状（已核实）

#### 问题 1：跨日重复严重
抽样近 10 天 `portfolio_alerts` 表：
| 基金 | 类型 | 10 天内条数 |
|---|---|---|
| 008929 宏利消费红利 | buy_drop_alert | 9 条 |
| 014846 博时恒乐债券A | concentration_alert | 9 条 |
| 014674 ... | buy_drop_alert | 7 条 |

**根因分析**：
- `db/portfolio.py:list_alerts` 当前 SQL：`GROUP BY title, severity`
- 但 `title` 含具体数值，例如：
  - "008929 补仓后下跌 5.2%" / "008929 补仓后下跌 6.1%"（不同日数值不同 → title 不同 → 无法合并）
- 分组后还**丢失了 `related_fund_code` / `related_fund_name`**，前端历史对比按钮拿不到 fund_code
- `routers/portfolio.py:scan_portfolio_alerts` 内的 `should_create` 同日去重有 bug：
  - 用 `a.get("created_at")` 取聚合行，但聚合行字段是 `latest_at`，永远拿不到 → 实际同日去重靠 `create_alert` 内部的同日同基金更新逻辑兜底
- AlertCenter.vue 前端已支持 `cnt`、`latest_id` 字段，但分组维度不对导致 cnt 永远=1

#### 问题 2：可靠性未标注
- 已有 `alert_accuracy_stats` 表（P2-4.3 回测），3 组数据：
  - `buy_drop_alert:danger` — 8 条样本，平均反弹 +3.68%，胜率 100%
  - `buy_drop_alert:warning` — 2 条样本，平均反弹 +1.62%，胜率 100%
  - `concentration_alert:warning` — 4 条样本，平均 -0.9%，胜率 100%（按"看跌"定义）
- 但 `list_alerts` 不返回可靠性，前端不展示
- 用户提到"很多按低估来的" — 指 `valuation_opportunity`（百分位 < 20%），历史无回测数据，可靠性确实存疑（低估值可能继续下跌）

#### 问题 3：缺少新闻与市场走向
- 现有 `news_impact` 类型预警 6 条，但来自对话内容非实时新闻
- MCP 可用工具：
  - `SearchFinancialNews(keyword, startDate, endDate, page, pageSize)` — 财经资讯
  - `SearchHotTopic(keyword, publishedDate)` — 市场热点
- 直接每次巡检调用 MCP 成本高，需缓存 + 开关

## 二、设计目标

| 目标 | 衡量标准 | 成本约束 |
|---|---|---|
| 跨日重复预警合并为一条 | 同基金同类型同 severity 只显示一条，cnt 标注次数 | 0 LLM / 0 MCP / 0 akshare |
| 标注可靠性等级 | 每条预警显示 高/中/低 + 胜率% + 样本数 | 0 LLM / 0 MCP（用现有回测数据） |
| 结合财经新闻 | danger/warning 级预警展示 2-3 条相关新闻 | LLM 默认关闭；MCP 受开关控制，30 min 缓存 |

## 三、技术方案

### P0-1：跨日重复预警合并（前端 cnt 真正生效）

#### 改动 1：`db/portfolio.py:list_alerts` 分组维度调整

**改前**：
```sql
SELECT title, severity, source, alert_type, content,
       COUNT(*) as cnt, MAX(created_at) as latest_at, MAX(id) as latest_id
FROM portfolio_alerts
WHERE ...
GROUP BY title, severity
ORDER BY latest_at DESC
```

**改后**：
```sql
SELECT
    alert_type,
    related_fund_code,
    related_fund_name,
    severity,
    source,
    -- 取最新一条的 title/content（用 MAX(id) 关联子查询）
    (SELECT title FROM portfolio_alerts p2
     WHERE p2.alert_type = p.alert_type
       AND COALESCE(p2.related_fund_code,'') = COALESCE(p.related_fund_code,'')
       AND p2.severity = p.severity
       AND p2.user_id = p.user_id
       AND (p.is_read = 0 OR p2.is_read = 0)  -- 未读分组下取最新未读
     ORDER BY p2.id DESC LIMIT 1) as title,
    (SELECT content FROM portfolio_alerts p2
     WHERE p2.alert_type = p.alert_type
       AND COALESCE(p2.related_fund_code,'') = COALESCE(p.related_fund_code,'')
       AND p2.severity = p.severity
       AND p2.user_id = p.user_id
     ORDER BY p2.id DESC LIMIT 1) as content,
    COUNT(*) as cnt,
    MAX(created_at) as latest_at,
    MAX(id) as latest_id,
    MIN(created_at) as first_at
FROM portfolio_alerts p
WHERE ...
GROUP BY alert_type, COALESCE(related_fund_code,''), severity
ORDER BY latest_at DESC
LIMIT ?
```

**收益**：
- 同基金同类型同 severity 跨日合并（如 008929 buy_drop_alert 9 条 → 1 条，cnt=9）
- 保留 `related_fund_code` / `related_fund_name`，前端历史对比按钮可用
- title 取最新一条（保留最新数值），content 同样取最新
- 新增 `first_at` 字段，前端可显示"首次 X 天前"

### P0-2：预警可靠性标注

#### 改动 1：`db/portfolio.py:list_alerts` 附加 reliability 字段

在 Python 层 join `alert_accuracy_stats`（数据量小，避免 SQL 复杂 JOIN）：

```python
def list_alerts(...):
    rows = ...  # 上面改后的 SQL
    if not rows:
        return []
    # 附加可靠性（取最近 4 周回测）
    from db._conn import _get_conn
    conn = _get_conn()
    stats_rows = conn.execute("""
        SELECT alert_type, severity,
               AVG(win_rate) as avg_win_rate,
               SUM(sample_count) as total_samples,
               AVG(avg_followup_change) as avg_change
        FROM alert_accuracy_stats
        WHERE week_start >= date('now','localtime','-28 days')
        GROUP BY alert_type, severity
    """).fetchall()
    conn.close()
    stats_map = {(r["alert_type"], r["severity"]): dict(r) for r in stats_rows}

    result = []
    for r in rows:
        r = dict(r)
        key = (r.get("alert_type"), r.get("severity"))
        s = stats_map.get(key)
        r["reliability"] = _calc_reliability(r["alert_type"], r["severity"], s)
        result.append(r)
    return result


def _calc_reliability(alert_type: str, severity: str, stats: dict | None) -> dict:
    """根据回测数据计算可靠性等级。

    规则：
      - 高：胜率 ≥ 80% 且样本 ≥ 3
      - 中：胜率 ≥ 60% 或样本 < 3（数据不足）
      - 低：胜率 < 60%
      - 无数据：返回 unknown（前端显示灰色"未回测"）
    特殊：valuation_opportunity 一律标"低"（低估值不等于买点，需结合趋势）
    """
    if alert_type == "valuation_opportunity":
        return {
            "level": "low",
            "label": "低",
            "reason": "低估值不等于买点，可能继续下跌",
            "win_rate": None, "samples": None, "avg_change": None,
        }
    if not stats or not stats.get("total_samples"):
        return {
            "level": "unknown",
            "label": "未回测",
            "reason": "暂无历史回测数据",
            "win_rate": None, "samples": 0, "avg_change": None,
        }
    win_rate = stats["avg_win_rate"] or 0
    samples = stats["total_samples"] or 0
    avg_change = stats["avg_change"]
    if win_rate >= 80 and samples >= 3:
        level, label = "high", "高"
    elif win_rate >= 60 or samples < 3:
        level, label = "medium", "中"
    else:
        level, label = "low", "低"
    return {
        "level": level, "label": label,
        "reason": f"近 4 周回测：胜率 {win_rate:.0f}% / 样本 {samples} / 平均 {avg_change:+.2f}%",
        "win_rate": round(win_rate, 1),
        "samples": samples,
        "avg_change": round(avg_change, 2) if avg_change is not None else None,
    }
```

#### 改动 2：新建 `services/alert_news_service.py`

```python
"""财经新闻检索服务 — 为预警附加相关新闻。

成本管控：
  - MCP 调用受 alert.news_integration 开关控制（默认 false）
  - 30 分钟缓存（alert.news_cache_ttl）
  - 仅 danger/warning 级预警触发检索
  - 单基金最多 3 条新闻
"""
import logging
from datetime import datetime, timedelta
from typing import Optional

from db._conn import _get_conn
from db.config import get_config as _get_config

logger = logging.getLogger(__name__)


def get_alert_news(fund_code: str, fund_name: str = "",
                   force_refresh: bool = False) -> list[dict]:
    """获取基金相关财经新闻（带缓存）。

    返回：[{news_title, news_summary, news_source, news_url, published_at}, ...]
    """
    if _get_config("alert.news_integration", "false") != "true":
        return []  # 开关关闭

    ttl_min = int(_get_config("alert.news_cache_ttl", "30"))
    per_fund = int(_get_config("alert.news_per_fund", "3"))

    # 检查缓存
    if not force_refresh:
        cached = _load_cached_news(fund_code, ttl_min)
        if cached:
            return cached[:per_fund]

    # 检索关键词：优先基金简称（去常见后缀），fallback 用基金代码
    keyword = _extract_keyword(fund_name) or fund_code
    news_list = _fetch_news_from_mcp(keyword, limit=per_fund)
    if not news_list:
        return []

    # 写缓存
    _save_news_cache(fund_code, keyword, news_list)
    return news_list[:per_fund]


def _load_cached_news(fund_code: str, ttl_min: int) -> list[dict]:
    cutoff = (datetime.now() - timedelta(minutes=ttl_min)).isoformat()
    conn = _get_conn()
    try:
        rows = conn.execute("""
            SELECT news_title, news_summary, news_source, news_url, published_at
            FROM alert_news_cache
            WHERE fund_code = ? AND fetched_at >= ?
            ORDER BY published_at DESC
        """, (fund_code, cutoff)).fetchall()
        return [dict(r) for r in rows]
    finally:
        conn.close()


def _fetch_news_from_mcp(keyword: str, limit: int = 3) -> list[dict]:
    """通过 MCP SearchFinancialNews 检索新闻。

    失败时返回空列表，不阻塞预警流程。
    """
    try:
        from mcp.yingmi_client import call_mcp_tool  # 复用现有 MCP 客户端
        today = datetime.now().strftime("%Y-%m-%d")
        start_date = (datetime.now() - timedelta(days=7)).strftime("%Y-%m-%d")
        result = call_mcp_tool("SearchFinancialNews", {
            "keyword": keyword,
            "startDate": start_date,
            "endDate": today,
            "page": 1,
            "pageSize": limit,
        })
        # MCP 返回结构需适配，这里容错
        if isinstance(result, dict):
            items = result.get("data") or result.get("items") or result.get("list") or []
        else:
            items = result if isinstance(result, list) else []
        return [
            {
                "news_title": it.get("title") or it.get("newsTitle") or "",
                "news_summary": it.get("summary") or it.get("abstract") or "",
                "news_source": it.get("source") or it.get("media") or "",
                "news_url": it.get("url") or it.get("link") or "",
                "published_at": it.get("publishTime") or it.get("publishedAt") or "",
            }
            for it in items if it.get("title") or it.get("newsTitle")
        ]
    except Exception as e:
        logger.warning(f"[alert_news] MCP 检索失败 keyword={keyword}: {e}")
        return []


def _save_news_cache(fund_code: str, keyword: str, news_list: list[dict]):
    conn = _get_conn()
    try:
        # 清除该基金旧缓存
        conn.execute("DELETE FROM alert_news_cache WHERE fund_code = ?", (fund_code,))
        for n in news_list:
            conn.execute("""
                INSERT INTO alert_news_cache
                (fund_code, keyword, news_title, news_summary, news_source, news_url, published_at)
                VALUES (?, ?, ?, ?, ?, ?, ?)
            """, (fund_code, keyword, n["news_title"], n.get("news_summary"),
                  n.get("news_source"), n.get("news_url"), n.get("published_at")))
        conn.commit()
    finally:
        conn.close()


def _extract_keyword(fund_name: str) -> str:
    """从基金名称提取检索关键词（去掉常见后缀）。"""
    if not fund_name:
        return ""
    suffixes = ["A", "C", "B", "ETF", "LOF", "联接", "份额", "类",
                "指数", "增强", "股票型", "混合型", "债券型", "基金"]
    kw = fund_name
    for s in suffixes:
        if kw.endswith(s):
            kw = kw[:-len(s)]
    return kw.strip() or fund_name


def enrich_alerts_with_news(alerts: list[dict]) -> list[dict]:
    """批量给预警列表附加 related_news 字段。

    仅 danger/warning 级触发，避免 info 级浪费 MCP 配额。
    """
    if _get_config("alert.news_integration", "false") != "true":
        for a in alerts:
            a["related_news"] = []
        return alerts

    seen_codes = set()
    news_cache: dict[str, list[dict]] = {}
    for a in alerts:
        if a.get("severity") not in ("danger", "warning"):
            a["related_news"] = []
            continue
        code = a.get("related_fund_code") or ""
        if not code:
            a["related_news"] = []
            continue
        if code not in seen_codes:
            seen_codes.add(code)
            news_cache[code] = get_alert_news(code, a.get("related_fund_name") or "")
        a["related_news"] = news_cache.get(code, [])
    return alerts
```

## 七、风险与回滚

| 风险 | 缓解措施 |
